 1. 緒論 
1.1 研究背景與動機 
自從 WINTEL 主導個人電腦的標準後，個人電腦產業面臨重大挑戰，其經營環境的激
烈多變更甚以往，因此經營模式也與以往迥然不同。然而目前個人電腦產業的趨勢已形成
跨時區、跨地域競爭的特性，專業技術廠商的崛起，IBM 共容性電腦蔚為主流，微軟的作
業系統與英戴爾的中央處理器進而完全掌握產業標準的制定與市場的脈動[12]，產品功能差
異化程度大幅下降，價格競爭取而代之，競爭基礎轉為速度、彈性、成本與服務[17]，促使
電腦產業結構轉向專業分工模式[5]。 
在世界各大品牌電腦廠競爭激烈的環境下，大廠多將原先屬於通路廠商的配銷工作，
利用 BTO 或 CTO 模式轉嫁給我國廠商，使我國廠商必須分攤過去大廠所作的存貨管理、
配銷…等動作。在此環境下，對我國個人電腦產業而言，唯有替國外品牌商提供良好的服
務來達成，亦即能在顧客要求的時間內滿足其訂單水準，才能避免被其他競爭對手所取代。
然而在面臨海外佈局，為了要滿足品牌商日益嚴苛的訂單標準，如 985（五天內完成 98%
以上的進度）甚至是 992（兩天內完成 99%以上的進度），通常是透過大量的存貨以備品牌
商的訂單需求。但個人電腦產業生命週期短，缺貨成本與存貨成本都很高，如何讓我國個
人電腦系統商在能滿足品牌商訂單要求情況下，又不必承擔太多的存貨風險，使廠商同時
維持競爭力與獲利能力，是本文主要的動機。

2.1 全球運籌管理 
「全球運籌管理」(Global Logistics Management; GLM)概念的興起與目前全球產業之發
展趨勢密不可分。由於產品的生命週期越來越短，且消費者對產品的需求朝向多樣性、個
性化發展、對交貨時間需求亦更加嚴格，全球從汽機車零件業、機械業到電子資訊業者，
為了接近市場以便就地服務，必須在全世界各地設生產基地或發貨倉庫，從各個供應鏈中
擠出利潤，藉由物流和通訊技術，整合企業內的各個活動，努力由昔日原廠委託製造(Original 
Equipment Manufacture; OEM)及原廠委託設計(Original Design Manufacture; ODM)製造導
向，轉型為製造與物流供貨同時兼顧的服務導向，以提升在全球供應鏈的附加價值，創造
競爭力。Hill 與 Jones[13]整理 Michael E. Porter 的價值鏈觀點來說明全球運籌模式價值鏈，
即價值鏈的形成並非由獨立的功能活動組合而成，而是相互依賴的價值活動所組成的系
統。因此，全球運籌管理模式可被視為一種超越傳統供應鏈管理格局的跨國界整合體系。
以供應商角度，代替顧客來進行存貨管理的服務模式在近年被發展出來，本章節將針對全
球運籌體系中供應商存貨管理模式進行探討。 
    
2.2 供應商存貨管理 
    VMI 是目前企業提升組織之間供應鏈效率的主要探討議題之一[14][11][16]，過去由於
錯誤的消費資訊、促銷活動情報、競價折價而導致需求量的增加，企業面臨日益複雜的環
境挑戰，平衡存貨成本和服務顧客需求也成為企業努力的兩大目標[11][15]。 
VMI 為一種庫存管理方案，強調由供應商直接管理並取得下游零售商的需求及存貨資訊，

4. 我國個人點腦產業 VMI 模式之資訊基礎架構 
4.1 VMI 協同機制資料格式架構之建立 
建構作業流程標準中，很重要的就是要建立傳遞的文件格式的定義，但在多對多環境
下如果無法定義標準格式的話，將會造成系統間資料相容性的問題，進而使得訊息在傳遞
的過程中發生認知偏差的問題。以 RosettaNet 標準來說，在以 PIP 為核心的基礎下，必須
制訂相關作業流程標準以及相關的文件標準，當傳送 RN 3A4 訂單需求訊息中，傳送的文
件裡必須包括哪些欄位與資料，而文件中欄位的組成是由 PIP 內所規定的數百種欄位中定
義必備的欄位以及符合各種情況的欄位。然而每個欄位中都有其定義的方式，資料內容可
能包括文字或數字，或是代碼符號…等，可藉以清楚地判斷該欄位所代表之含意。 
一般而言，RosettaNet PIP 可分為以下三個部份所組成[4]： 
1. PIP Specification 
2. XML Message Guideline 
3. XML Message Schema 
因此，在 PIP 中的 Message Guideline 中就定義了企業文件所需遵守的代碼表，而代碼
表的形成主要就是由 RosettaNet Dictionary 所組成，因此須參照 RosettaNet Dictionary 所規
範的內容。在欄位格式定義上須針對技術辭典與企業辭典兩方面的格式來制訂。 
技術辭典主要針對獨立性的定義，一般稱為 RNTD(RosettaNet Technical Dictionary)，提
供PIP用來描述產品的共通特性。而RNTD格式為了制訂與維護上的方便，因此也使用XML
的方式來建立，以求資料傳遞時的相容性與可讀性。而企業辭典則是定義了每個 PIP 在
Message Guideline 裡所需填入的值的意義，裡面會引用許多現成的代碼標準，如廠商名稱
識別碼(DUNS)、全球產品識別碼(GTIN)以及產品分類碼(UN/SPSC)等，以作為與既有標準
格式作為整合之用。另一方面，在欄位格式中也需能整合現有的標準，才能提升此一模式
之適用性與可行性。 
 
4.2 VMI 協同模式之網路通訊協定架構 
在本文所提出之 VMI 協同模式中，很重要的一點就是如何建置訊息傳送協定之機制。
因此 VMI 協同模式的資訊傳遞必須注意下列幾項特點： 
1. 必須建立在一個開放、共通的網狀架構。 
2. 明確定義所有的訊息交換協定。 
3. 明確定義企業伺服器間的資訊交換藉以溝通所採用的 HTML/XML 的語法。 
4. 交易雙方協定認同使用之文件格式定義。
