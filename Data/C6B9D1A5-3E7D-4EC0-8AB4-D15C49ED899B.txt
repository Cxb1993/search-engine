開發無遠幅界(ubiquitos)網路工作環境,建立車輛行動資訊系統平台
ABSTRACT
本研發案主要目標為導入車用行動任務資料自動蒐集平台，讓管理者可透過此一系統
進行資料稽核及分析，從通報資料中相關之人事時地物資訊，找出任務執行時狀況，進而
擬定稽核及管理建議模式，作為日後提升任務執行品質之參考依據。
本研究開發之架構為在車用環境，運用包含 RFID、GPS 模組，設計出自動化的行動任
務收集平台。配合 client - server 架構，提供應用網頁給相關使用者。在網頁設計上，
除了顯示任務執行過程的功能，搭配視覺化的地圖顯示，方便了解行動任務的路線進行，
掌握任務狀況。進一步使用系統中的稽核功能，審視任務進行或任務參與人員、物品、設
備有無異常狀況，評估員工、任務效率，進而改善缺失，發揮任務進行的最大效益。
一、 緒論與研究目的
行動任務進行時，有效且完整地記錄任務現況，對任務後的稽核和資料分析是不可或缺
的環節。任務進行過程的資訊包括: 任務狀況、任務人員、執勤時間、出任務地點、使用
裝備等…。由於資訊繁多，加上任務人員缺少對任務資訊能有效管理及記錄方法，造成資
料蒐集不足及效率差。欲有效提高任務稽核效率，資料收集自動化設計是不可或缺的。本
計劃開發之行動任務資訊系統除了車端提供記錄任務資訊平台外，後端並設計使用者可查
詢及稽核之操作介面，提供任務執行過程中相關資訊之稽核及分析功能。
行動任務資訊系統在近年來受到各界的重視，尤其在現今物流業的應用上相當普遍，
以知名企業 DHL(www.dhl.com/)為例，研發運務品管監控系統(QSMS, Quality Shipment
Monitoring System)，顯示貨件運送過程的經過，讓品管中心人員可以透過縝密的管控流
程，隨時掌握貨件狀況，如有任何突發狀況或潛在問題，可立即向客戶提出預警，為客戶
提出因應策略與解決方案。不可諱言的是，現今大多的行動任務上，由於部分環節採用舊
式的設計，在收集資料的效率和正確性上仍存在空間改善。
近年來社會上發生幾則費用報支流向的頭條新聞，曝露出事後稽核面對人為疏失或是
謊報的資訊蒐集困難問題。本研究在資訊蒐集及稽核平台開發方面，提供車用型行動任務
在員工執勤、費用流向、任務時空資訊呈現、事件還原及設備使用資訊蒐集上之協助，於
節省人力和時間的前提下將資料完整重現，定期審視任務執行狀況，對於組織內計劃擬訂、
執行控管、計畫變更、經費流用、目標達成、成果管理等情形之檢查、評估及建議有相當
大的助益。
二、背景知識及技術說明
RFID 相關研究
根據 EAN.UCC RFID 白皮書對 RFID( Radio Frequency Identification, 無線射頻辨識)
的解釋，RFID 是自動資料蒐集(Automatic Data Capture，ADC)技術的集合，其中最基本
的 RFID 系統由 Smart Tag(電子標籤)和 RFID 讀取器(RFID Reader)二者組成。Smart Tag
內建 RF 發射模組與控制電路，當 Tag 感應到 Reader 所發出之無線電波時，Tag 內的控制
電路會將此一無線電波能量轉換成電源，並趨動內建之感應器，按所設定之通訊協定進行
導稽核結果等四個階段。稽核人員若採用傳統的人工稽核技術與方法，則稽核效果與效率
將大打折扣。特別是行動任務事項稽核，需開發較自動之原始資料蒐集平台，並採用有效
的稽核技術與方法，直接驗證資訊的完整性與可靠性。蒐集資訊需涵蓋出勤工作事項之人
車事時地物及設備和經費資訊。
三、研究方法及系統設計
系統架構設計
本系統建置目標是建構車用型具有將行動任務資料自動搜集及完整記錄功能之平台，
資料隨後將通報回任務中心，配合後端系統所提供功能，將行動任務重建勾稽串聯，對不
同使用者提供有效的服務。我們採用不同硬體組合來建構一套 Client-Server 的資訊系統。
考慮行動任務團隊多樣化資訊蒐集需求，大部分的資料採自動搜集設計，部分的資料搜集
設計於 PDA 上。系統架構如下圖所示，系統架構包含數個功能模組。
系統架構
車端(Client 端)藉無線通報或可攜式記憶體媒介和 Server 連接傳送資料，Client 和
Server 二者分別負責資料搜集與資料分析的工作。系統 Server 端對不同的使用者透過 WEB
網頁提供包括稽核及資訊分析服務。
資料收集設計
系統的資料收集主要依賴 PDA 上設計 middleware 將 RFID Reader 及 GPS Receiver 二
項模組蒐集到資訊聯結。一組完整資料是同時包含 RFID 與 GPS 收集，其中 RFID 蒐集人車
物設備資訊，而 GPS 提供時空資訊。事件及經費資訊則透過計算、推論及關聯方式，於 server
端附加取得。
在 PDA 內 Middleware 設計為視窗模式，請參考下圖，
圍讀取到 tag，RFID reader 擷取標籤 ID 後，將顯示於 middleware。由於 RFID reader 於
頻率開啟到關閉時，會於範圍內的標籤持續進行偵測，故系統 middleware 的設計是將同
一標籤讀取到的資料只取第一時間點，記錄其標籤號碼。 Middleware 需內建標籤資料庫，
讀取不同型態的標籤時，顯示資料庫中對應的人員、車輛、物品、設備，便於資料識別。
GPS 資料收集
Middleware 於連接 RFID reader 時，同時開啟 GPS 模組。在 Middleware 顯示上將 GPS
功能簡化，只顯示衛星時間、經緯度座標和時速三項資料。結合二者設計，Middleware 偵
測若 RFID tag 受到讀取時，記錄當時的經緯度座標及時間，故一組完整資料是同時包含
RFID 與 GPS 收集。GPS 除了將其經緯度座標搭配 RFID，另外將任務完整的經緯度和格林威
治時間儲存，接收頻率為每秒鐘一次，接收過程獨立於 RFID 模組。
資料處理及通報作業設計
在行動任務資料收集，將資料存成文字檔，需透過通報作業上傳至 Server 端的系統
資料庫。系統採用應用程式透過 ODBC( Open DataBase Connectivity，開放資料庫連接) 存
取 Server 端資料庫系統。

經由 RFID 和 GPS 資訊透過以上步驟，重現任務，釐清是否有特殊狀況發生。過程以數
學式 h(g(f(n)))=L表示。細部說明為，t=f(n)，n:稽核的事物，f(n)=t:將 n對應在任務
中讀取的 PDA 時間。T=g(t)，T:格林威治時間，g(t):將 PDA 時間比對相同的格林威治時間。

予 RFID 電子標籤的人員、物品、設備。 費用稽核則是依據單據發生地點以地圖查詢發生
點的經緯度座標，再和任務中行經的經緯度座標比對。
五、結果與討論
本研究探討了車用型行動任務自動資料蒐集平台開發之各面向。GPS 模組提供行動任
務相關時空資訊蒐集能力；被動式中距離 RFID 模組提供成本可接受，且包括車內人、物、
設備資訊蒐集，及車對車、車對大門自動辨識能力；至於事件判別，經費稽核，及任務階
段辨識，則需依賴程式計算、法則推論、資訊關聯及時空分析來完成。
本研究亦探討了行動任務資訊自動稽核之各面向。研究中進行系列正確性的檢測，完
整確立系統資訊蒐集辨識的可信度，還檢測經由稽核功能所提供的結果。檢測將一般化稽
核和細部稽核分開討論:
檢測一般化稽核設計正確性，將一般化稽核中的人員、事件、物品、費用四種稽核於
系統網頁列出的時間、空間資料比對查核資料的真實記錄。依照稽核對象調閱存在於系統
資料庫系統的時間、空間資料表，和系統提供的資訊進行比對。四種稽核比對採統計分析，
統計正確稽核結果的正確性。
細部稽核的檢測實驗上，實驗是先建立各異常和正確條件的行動任務資料庫，藉由網
頁程式的稽核功能審查正確和異常情形的判斷是否和預期相同，將實驗結果經由統計分
析，製作統計圖表，求得錯誤偵測的效果，了解細部稽核的可信度。
在系統的使用介面上，期待系統頁面明瞭易懂、操作簡單，對行動任務出勤有所助益，並
能有效節省作業時間，有效率的彙整關聯資訊，並提供線上立即的快速任務稽核，讓通報成效與稽
核品質發揮最大效益。將來希望本系統架構在完善的設計規劃下，配合 EPC global 架構，期
待成為 Ubiquitous Network Society 的應用成果之一。
