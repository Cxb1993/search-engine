二、計畫緣由與目的 
隨著資訊科技與網際網路(Internet)高度的應用與發展，大多數資料文件已轉為數位形
式並可透過網路快速傳遞。雖然電子化確實提昇工作效率及生活品質，但同時也衍生出愈
趨複雜的網路安全與隱私保護等問題。其中，最基本且最重要的問題為如何在開放式網路
環境中鑑別(authenticate)連結端之使用者或裝置的身分，進而考慮身分鑑別後如何安全且秘
密地交換傳輸資料。 
使用者所使用的終端設備已不再僅限於傳統的個人電腦，利用手機進行資料交換、分
享、以及傳播無線商業資料也日漸普及。從前幾年所推行的 WAP 手機無線上網，到近年
來的 GPRS、i-mode 上網等，無所不在的行動生活亦日漸融入日常生活之中。「行動裝置」
(mobile device)包括常見的行動電話(GSM、GPRS、WAP)[15, 16]、個人數位助理(Personal 
Digital Assistant, PDA)、資訊家電 (使用短距離無線資料交換通訊技術，例如藍芽
(Bluetooth)[7, 8]或 IEEE 802.15 標準[19, 20])、行動通訊網路以及無線區域網路(IEEE 802.11
標準)[20]。在這些行動性強但計算能力弱的裝置上，除了裝置本身的實體安全保護外，存
取使用時的遠端身分鑑別機制與達成秘密通訊的金鑰交換(key exchange)機制更是不可或
缺。因此，在計算能力與通訊量較弱的行動裝置上如何執行遠端身分鑑別與金鑰交換功能，
是建構無所不在數位生活與行動商務的重要課題。 
為能在公開的網路容易地建立秘密通訊管道，Diffie 與 Hellman [13]於 1976 年提出公
開金鑰密碼系統(Public Key Cryptosystem, PKC)的概念，並提出一個植基於離散對數問題
(Discrete Logarithm Problem, DLP)的金鑰交換方法。此系統在通訊前，每個通訊個體，須先
產生一組私鑰與公鑰對，私鑰由通訊個體秘密保存，並將公鑰公布。系統中任意兩個個體
可使用本身的私鑰與對方的公鑰建立一把共享的交談金鑰(session key)，進而在公眾網路中
建立一個安全的溝通管道。然而使用 Diffie-Hellman 金鑰分配方法不易轉換到多個體的應用
環境中，因此，Ingemarsson 等人[21]提出 Diffie-Hellman 一般化群體金鑰協議機制。隨後陸
續有學者提出以此方法之精神所衍生的各種適用於群體的金鑰建立(key establishment)機
制。然而，如何證明群體金鑰協議之安全是過去所缺乏的。Bresson 等人[BCP01]在 2001 年
提出一個具有正規安全證明的群體鑑別金鑰協議，其正規安全證明是以Bellare等人[BPR00]
所提出的安全模型為基礎來建立。但 Bresson 等人所提出金鑰協議的延展性(scalable)差，金
鑰協議之回合數等同於通訊個體參與協定的人數，回合數為 O(n) (n 為參與金鑰協議之使用
者個數)，因此，隨著群體的使用者個數增加，其計算複雜度也隨著增加。在 2003 年，Katz
與 Yung[23]提出一個可將任何群體金鑰協議之方法轉換成為具延展性與鑑別性的方法，其
方法的回合數為常數時間，通訊個體利用數位簽章來達到身分鑑別的目的。 
然而，基於 DLP 等運算複雜度較高的金鑰協議機制並不適用於運算能力較低的行動
裝置系統中。2004 年，Bresson 等人[9]提出一個利用線上/離線簽章機制(on-line/off-line 
signature schemes)的群體金鑰協議機制，以降低使用者端運算量的複雜度。在 Bresson 等人
所提出的方法中，使用者可以將兩個模指數運算以及一個簽章運算於離線(off-line)時先行運
算，所以實際於線上(on-line)只要進行一個單向雜湊函數(one-way hash function)運算以及一
個互斥或(XOR)運算。Bresson 等人所提出的方法可適用於低資源行動裝置中，但面臨無法
達到隱含金鑰鑑別(implicit key authentication)、前推安全(forward secrecy)以及已知金鑰安全
(know key security)等安全性問題。在 2005 年，Nam 等人[28]提出另外一個適用於行動通訊

3.1.1 金鑰推導樹(Key Derivation Tree) 
    本論文所提出的方法是以第三代行動通訊系統為基礎透過延伸行動交換中心(MSC)服
務功能以協助建立金鑰協議功能，因此 MSC 需動態的為其服務範圍底下的使用者建構與維
護金鑰推導樹(Key Derivation Tree)，如圖 2 所示。本系統以 MSC 為子群體最小單位，各
MSC 自行維護虛擬節點 N 以及節點值 x，並且與其服務範圍底下用戶協議出一把子群體金
鑰 Ks，而後將 Ks 交由發起者的 MSC 進行最後的群體金鑰 Kg 計算，若遇到會議成員皆在
同一 MSC 服務範圍底下的情況，此時 Ks 即為群體金鑰 Kg。我們將金鑰推導樹(Key 
Derivation Tree)固定為四層的架構，所以每個節點 N 底下使用者數量將直接影響到金鑰更
新效率與 MSC 儲存量，當虛擬節點 N 越多時，則金鑰更新時越有效率，但相對的 MSC 就
必須儲存與維護更多的虛擬節點。當有成員加入或離開時，需進行金金鑰更新(Key 
Update)。加入新的用戶必須由會議發起者向 MSC 提出邀請才可以加入，但是為了維護通
訊安全不論是哪一種情況發生皆會進行金鑰更新，金鑰更新時只需針對有變動的節點值進
行重新計算即可。 
 
3.1.2 系統建置階段 
因為本系統是以第三代行動通訊系統為基礎透過延伸 MSC 服務功能來達到金鑰協
議，因此在現有的第三代行動通訊基礎建設完成後我們還需要額外增加單向雜湊函數運算
以及可以讓 MSC 可以動態的為其服務範圍底下的使用者建構金鑰推導樹並且維護樹中的
虛擬節點之兩個新模組。 
 
3.1.3 用戶註冊階段 
用戶 Uj需臨櫃辦理向系統申請服務，系統中心 HLR/AuC 為其產生個人的私密認證金
鑰 Kj及國際行動用戶識別碼 IMSIj，並且將之分別儲存於 USIM 卡及認證中心 AuC 內，將
USIM 卡當場交給用戶。

3.3 安全視訊會議系統 
本系統以既有即時通訊軟體架構為基礎，並加入身分鑑別與群體金鑰交換機制，以達
到身分鑑別、機密性以及隱私性等安全需求。在系統開發中，我們主要利用 Java Media 
Framework(以下簡稱 JMF)與Real-Time Transport Protocol(以下簡稱RTP)來實現影像與聲音
傳送。其中，JMF 提供多媒體資料的取得、處理與播放之統一管理架構，並且能支援多種
標準影音格式，例如：AIFF、AU、AVI、GSM、MIDI、MPEG、QuickTime、RMF 和 WAV
等。而 RTP 為在 Internet 上一個“即時”傳送聲音或影像的標準協定，以達到一邊下載一邊
播放的功能，無須等待檔案全部下載完畢後才開始播放。RTP 屬於獨立的通訊協定，提供
即時資料傳輸的點對點(end-to-end)傳送服務，並兼具單點播向(unicast) 以及群播(multicast)
的網路功能，因此適合用於多人之即時視訊會議。 
    在身分鑑別機制上，我們採用通行碼以及智慧卡之雙重保護機制，在智慧卡的使用上，
我們配合 JMF，採用可於卡上執行 Java 語言開發應用程式之 Java Card。除此之外，Java Card
具有下列優點：(1)軟體易於開發：Java Card 可使開發人員不需接觸到底層之組合語言，而
可利用擁有標準定義的程式開發介面(applet)與執行時期環境（runtime environment）的開放
式平台開發程式；(2) 安全：Java Card 中所有的函式與變數的存取權限皆被嚴格的控制，
因此攻擊者不易利用程式的弱點以偽造的指示執行惡意的程式並進一步來窺探記憶體內的
資料；(3) 硬體獨立性：由於 Java Card applets 是在 Java Card 的平台上所撰寫，因此可在不
同規格的智慧卡處理器上執行，而不用重新編譯。(4) 具有儲存與管理多個應用程式之能
力：Java Card 上可存放多個由不同服務供應商所提供之 applet，例如：電子錢包、認證、
會員及醫療保健等。由於 Java Card 上之防火牆的機制，使得 applet 之間也無法任意存取其
他 applets 的資料，因此仍可達到 applet 之間的安全性與隱私性。 
最後，整體系統的開發是以 web-based 平台為主，因此，所有的使用者皆無需再安裝
任何軟體或購置 Java Card 以外的設備即可使用本系統平台進行遠端即時視訊會議。本計
畫所設計之安全的視訊會議平台具有下列特點： 
z 透過密碼學機制，達到身分鑑別、訊息機密性、隱私保護等安全需求。 
z 達到無所不在(ubiquitous)的即時互動健康照護。利用較低運算量或通訊成本的密碼學
機制設計具鑑別性之金鑰協議(key agreement)機制，以易於實作於低資源之行動裝置
中，例如個人數位助理(Personal Digital Assistant, PDA)等。 
z 可建立群體金鑰，達到多人安全通訊之目的。 
z 可建立不同需求之密秘通訊管道。設計 3-Level KAS (Key Agreement System)，每個使
用者可分別建立 U2S 使用者與系統中心間的金鑰 (User to Server）、U2G 使用者與通
訊群體間之金鑰(User to Group）以及 U2U 個別使用者間的金鑰(User to User），因此
在群體通訊的時候，個別使用者之間亦可同時秘密通訊。 
z 具使用者親和性(good friendliness)的健康照護平台，使高齡者容易使用。 
z 便利性。採用 Web-Based 平台，因此，使用者不需下載任何的安裝程式，只需連上網
站，即可進行互動即時通訊。 
z 具動態(dynamic)群體通訊管理功能，可隨時加入或離開正在通訊的對話。 
z 具自動紀錄機制(automatic recording mechanism)，提供使用者儲存通訊內容。

4.1 適用於 3G 之金鑰交換機制之研究成果討論 
本計畫所提出之適用於 3G 之金鑰交換機制，可在只增加少數模組下快速實作於現行
之第三代行動通訊系統上，除此之外我們的金鑰協議機制所協議出來的金鑰更可以直接用
於現存第三代行動通訊系統的加密演算法，如此一來用戶端則不需要額外增加任何模組即
可達到端對端的安全傳輸。另外，我們的方法絕大部份的計算只涉及 XOR 運算、單向雜湊
函數以及第三代行動通訊系統現有的函數[1, 2, 3, 4]，因此其執行將非常有效率。我們的方
法可以擴充現有無線通訊產業的附加服務，以提昇無線通訊系統中使用者端之間的安全
性。本計畫所提出之適用於 3G 之具身分鑑別的群體金鑰交換系統具有下列主要的四項特
性如下： 
(1) 利用現有行動通訊系統函數，可以只增加少數幾個模組建構在第三代行動通訊系統之
上，增加用戶的端對端安全，同時也為第三代行動通訊帶來更多的應用，有效擴大第
三代行動通訊網路的使用範圍。 
(2) 計算過程中，主要使用 XOR 運算及單向雜湊函數等工具，因此運算量低且具有效率，
且每個 MSC 只需要為其服務範圍底下的用戶計算子群體金鑰，將可分散每個 MSC 的
運算複雜度，以利於改善行動通訊系統的整體效能。 
(3) 本計畫所提出的方法適用於群體通訊系統的環境，不管使用者是否在同一個 MSC 系統
下，皆可以很容易地建立群體會議金鑰，達到秘密通訊之目的。 
(4) 滿足下列的安全目標： 
(a) 隱含金鑰鑑別(implicit key authentication, IKA)：是指執行身分鑑別協定後，通訊的
雙方除可鑑別彼此的身分，達到相互鑑別(mutual authentication)之目的，並可確認
每次所產生的交談金鑰(session key)是否正確。本計畫所提出的方法在執行金鑰協
議之前，所有的通訊個體需執行 3GPP TS 33.102 V5.0.0 標準[1~6]的身分鑑別機
制，以達到相互鑑別之目的。除此之外，會利用 MAC 確認交談金鑰的正確性。： 
(b) 已知金鑰安全(known session key security, KSS)：是指當使用者指當通訊個體已建
立的交談金鑰被攻擊者(attacker)得知後，攻擊者無法從已知的交談金鑰去推導出
其他已建立的交談金鑰。由於每次的交談金鑰是由不同的隨機亂數所產生的，因
此，攻擊者無法從已知的交談金鑰去推導出其他已建立的交談金鑰。 
(c) 前推安全(perfect forward secrecy, PFS)：系指當通訊個體的私鑰被破解或者故意洩
密後，攻擊者無法由已知的私鑰去推導出以前已建立的交談金鑰，進而解讀相關
已加密的訊息。由金鑰推導樹中(如圖 2 所示)，由於使用者的個人加密金鑰 SKR
是由私鑰以及某一隨機亂數所產生，故即使攻擊者得知某一個使用者的私鑰，亦
無法推導出正確的 SKR，進而得知已產生的交談金鑰，故本計畫所提出的方法可

(g) 可抵抗金鑰控制(key control, KC)：不管是可信賴的第三方、合法的通訊個體還是非
法的攻擊者都無法事先預知或是控制交談金鑰的值；

(know key security)等攻擊，並提出改善的方法。然而，目前適用於 3G 系統
的群體金鑰協議機制中，除 3G 系統中既有的密碼模組外，仍需增加許多其
他的密碼模組，因此不易實作。有鑑於此，我們利用第三代行動通訊系統
現有的函數以及簡單的 XOR 運算與單向雜湊函數(one-way hash function)等
模組設計出適用於 3G 系統上的金鑰協議機制，並考量群體通訊的應用，提
出群體金鑰協議機制，如此一來用戶端則不需要額外增加任何模組即可達
到端對端(end-to-end)的安全傳輸。

六、結論 
本論文以第三代行動通訊系統為核心基礎，提出四個群體金鑰協議機
制，我們提出的方法具有下列特性： 
(1) 我們所提出的金鑰協議方法利用現存行動通訊系統的函數，因此可以在
只增加少數幾個模組的情況下，快速的建構在第三代行動通訊系統之
上，增加了用戶的端對端安全，同時也為第三代行動通訊帶來更多的應
用，有效擴大第三代行動通訊網路的使用範圍。 
(2) 我們所提出的金鑰協議方法在計算過程中，主要是使用 XOR 運算及單
向雜湊函數等工具，因此計算速度快具有效率，並分散計算量於不同伺
服器之上，每個 MSC 只需要為其服務範圍底下的用戶計算子群體金
鑰，如此設計不僅將計算量分散並且有利於整體系統效能。 
(3) 我們所提出的方法滿足會議內容機密性、會議成員位置隱密性、防止假
冒以及防止重送攻擊等四項安全目標，並且滿足非揭露性、獨立性以及
完整性等三個安全需求。

Chung-Fu Lu, Tzong-Chen Wu, and Chien-Lung Hsu 
  
Abstract— This paper will design a three-level conference key 
establishment protocol based on UMTS core network framework 
to establish a group-level, a HLR-level, and a VLR-level common 
secret keys for a group of conferees simultaneously. The 
group-level key can secure the communications for all conferees, 
the HLR-level key is for those within the same HLR domain, and 
the VLR-level key is for those within the same VLR domain. The 
proposed protocol exploits only the existing UMTS security 
functions and exclusive-or (XOR) operation. It is efficient in 
terms of lower computational complexities, cost effectiveness, 
load-amortization, scalability, and security. Our proposed 
protocol provides lawful interception as required in UMTS.

I. INTRODUCTION 
With rapid growth of mobile wireless networks, many 
mobile applications like computer, audio, video and 
telecommunication are merging together. Above all, mobile 
telecommunication systems play an ever increasing role in the 
new business opportunities. To pave the way for this 
promising future, a smooth evolution from second generation 
(2G) telecommunication systems to third generation (3G) 
systems has to be ensured. As compared with 2G systems, the 
3G systems have higher data rate and security. The 3G mobile 
systems provide data rate of up to 2 Mbps to offer mobile 
services such as wireless voice, data, and multimedia services. 
They also provide the mutual authentication mechanism to 
authenticate the mobile device and the serving network by 3G 
AKA (Authentication and Key Agreement) [1].  
The Universal Mobile Telecommunications System (UMTS) 
is a popular 3G system. In such a system, encryption of traffic 
at the air interface is optional, and the decision to employ AIE 
(Air Interface Encryption) depends on the network operator. 
The mobile users are usually unaware of whether AIE is being 
employed or not. Moreover, the UMTS only provides channel 
security instead of ends-to-ends security. It will be inefficient 
and will become a bottleneck to secure the communications 
for any two mobile users or a group of mobile users. Since the 
group communications are becoming more and more 
important, the UMTS users are in urgent need of ends-to-ends 
secure communications to protect their transmission 
information. A major problem area in group secure 
communications is group key management. Unfortunately, 
these previously proposed group key management protocols 
[2]-[3] are not efficient and fully applicable to existing UMTS. 
This paper will propose a three-level conference key 
establishment protocol on the basis of existing UMTS core 
network framework. The proposed protocol is efficient in 
terms of lower computational complexities, cost effectiveness, 
load-amortization, scalability, security, and does not change 
the UMTS architecture.

III. ANALYSIS AND DISCUSSIONS 
A secure and practical conference key establishment 
protocol for mobile communications should be executed 
efficiently on the portable devices, and implemented without 
modifying the existing UMTS. Security analysis and 
implementation discussions for the proposed protocol are 
given below based on the UMTS and its secure algorithms. 
2) Lower computational complexities and cost 
effectiveness: The security functions and operations (i.e., f1, 
f2, f3, and XOR operation) of the existing UMTS are low 
computational complexities. Since our proposed protocol 
exploits the existing UMTS functions and operations, it has 
lower computational complexities. In addition, our 
proposed protocol is practical and efficient due to O(N) 
rounds of the message transmission, O(1) completion time, 
O(1) waiting time, and O(N) communication overheads. A. Security Analysis 
We will show that the proposed protocol achieve Diffie et al.’s 
[4] security requirements below.  
3) Scalability: The number of the message transmissions 
between HLR/AuC and MSC/VLR are independent on that 
of the mobile terminals in our protocol. Scalability of the 
proposed protocol depends on that of the existing UMTS, 
since the number of the conferees depends on the number of 
the mobile terminals controlled by each MSC/VLR.  
1) Integration of entity authentication and key 
establishment: The proposed protocol achieves the entity 
authentication and the key establishment by the existing 
UMTS AKA protocol. Derivation and verification of all 
secret keys for the conference are primarily achieved by the 
shared cipher key kjiCK ,,  established by the UMTS AKA. 
4) Load-amortization: The protocol can be performed by the 
conferees in parallel. The efforts (e.g., computational 
complexities, rounds of message transmission, and etc.) for 
each conferee are the same, our protocol can hence achieve 
load-amortization. 
2) Prevention of reflection attacks: The structure of 
authentication and exchanged messages are asymmetric 
which implies it can withstand the potential reflection 
attacks [4]. That is, the proposed protocol can prevent an 
adversary from masquerading as some entity to 
communicate with an honest one.  
5) Lawful interception [5]: Our protocol allows UMTS to 
derive all secret keys for lawful interception since the 
VLR-level keys, the HLR-level keys, and the Group-level 
key are computed by the system. 3) Prevention of replay attacks: All transmitted messages are 
linked with the random number kjir ,,  and the cipher key 
kjiCK ,, . The replay attack is excluded in our protocol. 
IV. CONCLUSION 
This paper has proposed a three-level conference key 
establishment protocol to establish three types of secret keys 
shared by the conferees within the same MSC/VLR domain, 
the same HLR/AuC domain, and all conferees, respectively. 
The proposed protocol are based on the UMTS architecture 
and only uses the existing functions and operations, hence it is 
very ease of use and implementation on UMTS. It is quite 
suitable for the conference services in the existing UMTS, 
such as supplementary service MPTY (MultiParty).

Abstract 
Recently, Zhang et al. proposed location-based compromise-tolerant security mechanisms for WSNs and introduced the notion of location-based keys 
(LBKs) along with an LBK-based neighborhood authentication scheme. They also claimed that their scheme can counteract several notorious attacks 
against WSNs such as the Sybil attack, the identity replication attack, and wormhole and sinkhole attacks. In this paper, however, we will demonstrate 
that their neighborhood authentication scheme is still insecure, i.e., a malicious adversary can compromise the security information or inject bogus 
messages into legitimate nodes by taking advantage of two compromised sensor nodes which belong to different cells, respectively. Further, we also 
address a simple path detection mechanism to eliminate the drawback without increasing the complexity of their original scheme.

 public key, cryptography

3. Weakness of Zhang et al.’s Scheme 
In this section, we first demonstrate that Zhang et al.’s neighborhood authentication scheme is insecure and then amend it by adopting a simple 
path detection mechanism. 
3.1 Weakness 
Let α = {A, B, C, …} and β = {A', B', C', …} be two cells consisting of several sensor nodes. Assume that sensor nodes of the same cell can 
mutually authenticate each other while any two belonging to different cells can not. That is, the Euclidean distance between two groups is greater than 
R. It is clear that node A ∈ α can not communicate with node C' ∈ β for that they can not authenticate each other beyond the legitimate transmission 
range R according to the predefined accessing policy. 
Now consider the scenario that node B ∈ α and node B' ∈ β are both compromised by an adversary, then a virtual sensor node B'' possessing the 
information (IDB, IDB', lB, lB', nB, nB', LKB, LKB') can be created. Whenever the node B'' wants to swindle the network information from legitimate 
nodes ∈ α, it broadcasts the authentication request (IDB, lB, nB) and uses LBK LKB to derive the valid shared keys among legitimate nodes ∈ α. On 
the other hand, to inject bogus messages into legitimate nodes ∈ β, B'' can instead broadcast the authentication request (IDB', lB', nB') and uses LBK 
LKB' to derive the valid shared keys among legitimate nodes ∈ β. The attacking diagram is shown as Fig. 1. 
By doing so, the virtual sensor node can successfully pass the authentication procedures without being detected. Thus, the above attack against 
Zhang et al.’s neighborhood authentication scheme counts.
