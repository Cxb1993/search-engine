建立能自動與公開資料庫連結的 robot，利用公開資料庫的查詢結果，驗證分析
結果，用來對所尋找到類別作功能註解。 
 
四、研究目的： 
我們會將我們的系統與其他公開的資料庫作連結，透過robot 來抓取必要的
資訊。例如基因功能註解，群組功能查詢等，以協助評估所得到的群組基因其代
表的功能意義。而這些評估的結果，也可以作為系統的回饋，藉以重新評估不同
的類別，所需要的參數值大小為何。 
而第二階段將要建置一個互動式線上處理系統。該網站將針對生物學研究人
員的需求與使用習慣，透過互動式的介面設計，讓使用者可以透過網路，進行線
上資料分析，並將分析後的結果透過網頁呈現，可以讓使用者即時的看到共同表
現的基因群以及表現樣式。但由於目前許多分析系統，對於系統介面的設計，並
沒有針對實驗者使用習慣，以及需求進行分析和製作。本計畫希望能提供友善的
使用者介面，對於分析後的結果，能以分階層、互動的方法來呈現；而互動技術
的使用，能夠透過圖形化的操作介面。可以讓使用者任意瀏覽不同階層的基因表
現樣式、基因群集，以及表現資訊。所以，該階段主要的工作項目，包括使用者
需求分析，互動使用者介面與資料庫建置等等。 
 
五、文獻探討： 
KGML： 
 KGML(KEGG Markup Language)顧名思義，它的構成方法是利用 XML 的格
式來表達資訊。現今 XML 已經成為了資料表達的主流。資料在經過 XML 標示
後所形成的 XML 文件，能夠容易的在網際網路上被傳遞、交換與使用。因為
XML 對文件資料的描述能力遠比 HTML 好，所以閱讀者從控制文件標籤名稱的
定義中即可很明確的知道標籤中的資料所代表的意義。對於文件而言，XML 除
了更具結構性之外，更能夠充份的描述資料本身的特性。所以當 KGML 使用了
XML 語法來描述 Pathway 的資料後，更能夠在網際網路上作資料交換與分享。
