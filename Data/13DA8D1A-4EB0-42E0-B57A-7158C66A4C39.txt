建構營建工程 BIM-based 知識管理系統建構之研究 
Developing the BIM-based Knowledge Management System for Construction 
計畫編號：99-2221-E-027-093- 
執行期限：99 年 8 月 1 日至 100 年 7 月 31 日 
主持人：林祐正  國立臺北科技大學土木工程系 
 
一、摘要 
 
知識管理(Knowledge Management)是
近年在國內才新興的觀念，然多盛行於研
究單位及每個產業。本研究利用網路技術
及 BIM 概念應用於 3D 知識地圖並建構適
用營建工程知識管理系統，有系統地彙整
蒐集及管理工程執行中所產生的知識，以
利提昇營建企業工程之效率與效益。利用
BIM 概念應用於 3D 知識地圖來整理相關
營建工程知識，主要的好處在於能夠有效
率地瞭解執行工程專案內部及外部相關之
知識資產，有效地彙整及管理工程專案的
知識。以目前來說國內的營建企業甚少利
用知識地圖的方式管理知識，而國外已經
有許多工商企業證實利用知識地圖進行知
識管理是一種有效的管理方法，不僅可以
用來衡量企業知識的庫藏，甚至可以用來
作為員工教育訓練的基準。營建工程未來
應用 3D 圖形將成為工程專案執行控管之
一種趨勢。因此本研究將透過相關文獻收
集分析及合作企業訪談，彙整出營建工程
運用 BIM 概念應用於 3D 知識地圖進行知
識管理活動之需求，模式及效益。同時整
合目前知識管理相關之學術理論基礎以及
工程專案實際落實知識地圖進行知識管理
之做法，建構出針對營建工程專案實行知
識 管 理 之 BIM-based 模 式 知 識 管 理
(BIM-based Knowledge Management)架構
及系統。本研究將提出營建工程專案知識
地圖建構模式與需求，主要可以分成三個
部分來進行：1.首先建構 BIM-based 之 3D
知識地圖及知識管理模型建構方式，討論
工程專案式知識整理之管理策略、工程專
案知識管理模式、3D 知識地圖知識擴散模
式、和 BIM 地圖型模式知識地圖分享策
略，並且討論 BIM 地圖型模式知識地圖運
用之有效性評估；2. BIM-based 知識管理應
用模式、架構及系統；3.利用個案營造廠的
實際工程運作實際應用 BIM-based 知識管
理模式進行知識管理，以提升營建工程專
案之知識管理之效用。本研究以工程實際
案例做為系統實例執行對象，首先對於國
內營建工程專案知識管理過程進行需求分
析，進而探討系統應用於實際案例進行工
程專案知識管理可能面臨之問題，並提出
適用於營建企業知識管理之知識地圖模
式，作為未來建立營建企業知識管理系統
之參考。本研究所建立之營建 BIM-base 知
識模式、3D 知識地圖方法及知識管理系
統，將可應用於其他工程公司導入知識管
理系統進行知識管理之參考。。 
 
關鍵詞：知識管理、建築資訊模型、資訊
管理、網路式系統、網際網路 
 
Abstract 
 
Knowledge management (KM) has 
become the most critical organizational 
strategy for enterprises. Knowledge 
management involves creating, securing, 
capturing, coordinating, combining, 
retrieving and distributing knowledge.  The 
knowledge can be reused and shared among 
the involved engineers and experts to 
improve the construction process and reduce 
the time and cost of solving problems.  This 
paper proposes a new and practical 
methodology to capture and represent 
construction project knowledge by using 
network knowledge maps approach.  Using 
3D-based Knowledge Maps (3DKM), users 
can get an overview of available and missing 
knowledge in core project areas and take 
appropriate management in tacit and explicit 
knowledge.  This research is a pilot study to 
develop a generic model using knowledge

知識地圖來整理相關營建工程知識，主要
的好處在於能夠有效率地瞭解執行工程專
案內部及外部相關之知識資產，有效地彙

管理方法，不僅可以用來衡量企業知識的

 可以用 3D 知識地圖模式方式進行工程
專案知識儲存、分享及再利用，以有效
提升管理施工過程所產生之隱性知識
及顯性知識。 
 利用 BIM 之 3D 之特色使 3D 知識地圖
做為規劃知識管理系統之核心雛形，使
合作企業能以較短的時間有效管理工
程專案之知識管理，將知識管理與工程

BIM-based 知識管理系統之知識展現方
式，使日後營建企業能更有效執行企業
工程專案知識管理。 
 因採用 3D 型知識地圖，降低工程師嘗
試導入知識管理之門檻。 
 因工程一般都會使用 CAD 施工圖作為
施工之依據，過程中若將所獲取之知識
及經驗以 3D CAD 圖形方式儲存，使可
提升知識管理之效能。 
建 築 資 訊 模 型 （ Building Information 
Modeling, BIM）的概念由 Graphisoft 公司
於 1987 年所提出，並由 Jerry Laiserin 利用
Autodesk、Bentley System 以及 Graphisoft
所提供之技術推廣於市場中。一般常見的
建築設計圖往往採用 2D 平面之方式傳遞
設計需求及概念，然而，非專業的建築設
計人員實在難以透過 2D 平面設計圖來完
整地了解其建築設計，有鑒於因依靠 2D 平
面設計圖說傳遞建築資訊導致溝通不良或
傳達資訊不明確之情形層出不窮，便有了
將建築設計圖透過視覺化的 3D「實體」模
型呈現以傳達各項設計理念及要求的概念
形成，透過視覺化的 3D「實體」模型實際
呈現建築物的基本構造、方向、平面、立
面以及相關細節可幫助非專業的建築設計
相關人員了解建築師所設計之建築物，並
增加溝通的便利性，降低溝通不良或資訊
傳遞錯誤的情形發生。然而，3D「實體」
模型於製作上耗時費力，而且完成後的模
型亦無法隨時隨地任意依要求或討論之結
果進行修改，於設計過程中，其設計相關
的建築資訊也無法包含在 3D「實體」模型
中，故建築設計人員便有了透過電腦資訊
及技術解決這些問題的想法產生。初期的
建築模型僅是將原本以 2D 平面表示的各
項建築物設計圖在電腦中建立成為 3D 的
建築物立體模型，模型中並不包含任何資
訊，亦無資訊轉換、交流及統整之功能，
其主要目的僅是為了提供較為便利的瀏覽
方式而已，這是 3D 建築模型的初始概念，
也於此建築資訊模型之概念有了開端。 
本研究系統 BIM 軟體採用 Autodesk 
Revit。開發工具主要是利用 Autodesk 之
API 及使用網際網路相關之資訊技術
(Information Technology, IT)來建構，主要
以 HTML (HyperText Markup Language)與
JSP (Java Server Pages)為主，配合 DHTML 
(Dynamic HTML)與 JavaScript 等技術，只
要使用者連上網際網路便可使用，無地點
與時間之限制，如此協力廠商不但可以減
少系統開發或購置軟體之成本，亦可降低
自行開發之風險與降低資料交換標準之問
題。本系統的設計使用 Java 的 Applet 和
Servlet 程式。Applet 的全意是指「一種很
小的應用程式」，可附著在網頁(Hyper Text 
Markup Language 簡稱 HTML)上，下載到
