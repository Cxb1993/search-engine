性。透過服務(services)之供應、存取與處理，空間資料基礎建設中之資源交換、
流通、整合與使用之效益將可有效提升 (ISO/TC204, document N271)。依此概念
發展，不但可結合服務導向架構（Service-Oriented Architecture）之技術，更可進
一步善用 OGC 制定之空間網路服務(Geospatial Web Service)，進而發展更為便利
之應用環境(Boucelma et al., 2002; Yuan et al., 2009)。 
雖然開放地理資訊系統技術已可有效改善許多過去之應用障礙，但就資料之
整合應用及互操作性而言，現有之技術仍僅侷限於資料格式（data format）或模
式(data model)之層級。透過商業軟體格式記錄之地理資料事實上是一種高度約制
之成果，其允許之記錄方式及內容都受軟體規格之影響，生產者雖可決定資料之
內容，但軟體規格不允許記錄之內容即不可能記錄，也往往無法加以處理。即令
完成所謂的「轉換」，也並不包括資料所代表之意義，遑論有關資料規劃設計之
內涵與知識，因此對內容之解讀與運作仍必須由專家來進行。Bishr（1998）曾
提出一個包括六個層級的互操作架構，其中第五個層級為資料模式(Data model)
之互操作，目前雖可透過開放式地理資訊系統技術(Doyle et al., 2001; Kresse and 
Fadaie, 2004; Inc., 2004; Boucelma et al., 2002)而達到圖形結構及屬性資料之傳
達，但最高層級之應用語意(Application semantics)互操作則仍然力有未逮。各領
域由現實世界現象至地理資訊系統(抽象至實體)實作之過程存在認知、模擬、概
念化之不同，造成同一現象在不同領域有不同之表現方式，甚或有同樣概念化成
果在不同領域代表不同意涵之情形。使用者若不能取得概念化及語意層次之正確
認知，即令取得資料，單憑對名稱的解讀也無法判斷，遑論進行異質資料之整合
與應用。由空間資料基礎建設之觀點，生產單位對資料具有最完整之認識，因此
如何具體呈現各領域設計之理由、過程及成果勢必納入為其發展之重要考量。這
樣的問題不可能單憑資料本身之記錄內容（空間及屬性）解決，必須由生產者切
入，發展一個由設計到應用的一條鞭完整描述架構，且整體機制最好還可基礎於
國際通用之技術標準，未來才可能正確及具體傳遞資料生產者之知識，以突破異

紀錄地理資料之相關知識，包括資料之語意表達、資料關聯性和資料應用規則
等。透過知識本體，可令電腦自動解讀語意和領域知識，從原本人與人之溝通，
晉升為機器與機器溝通之層級，不僅大幅降低人為操作，並可透過語意推論獲得
更多不同領域間之額外應用知識。知識本體可用於表達領域知識，將資料作明確

究分析地理資料之共同特性，以此形成上層及領域知識本體之架構。再由地理資

之發展程序。該領域知識本體將能提供領域知識與實質意義之記錄，透過確認領
域資料之類別意涵，可進一步提供為跨領域知識本體整合之基礎，有助於發展後
續跨領域整合應用。此知識本體架構因納入階層式分類參考系統之考量，可適用
於複雜之階層式語意描述架構，本研究將以土地利用為例，說明階層式語意知識
本體之組成元素與紀錄方法。除此之外，異質資料之品質差異也是目前地理資訊
系統應用上之一大障礙，本研究透過將品質資訊及其相關知識結合，發展為任務
知識本體之雛形，再結合地理資訊系統之運作，實際展示納入品質考量之運作模
式，可供未來新一代地理資訊系統發展之參考。 
二、文獻探討 
知識本體可協助資料語意之正確表達，促進地理資訊整合之精確度及提高自
動化處理程度。本計畫發展一個基於知識本體之地理空間資訊系統整合架構，首
要之務為發展可明確描述資料、清楚表達語意、以協助使用者透過知識本體瞭解
資料本質及意涵之理論與實務架構。以下分別針對知識本體架構、知識本體描述
語言與建置及地理空間資料知識本體與應用等三個課題進行分析。 
2.1 知識本體架構探討及其描述語言 
知識本體(ontology)源自哲學領域，為探討物體存在(being)之科學，探究真
實存在之物體及如何加以描述(Lowe, 2003)。近年之研究著重在如何透過資訊科
學技術發展知識本體之架構，由概念化之層次具體落實為可執行之資訊技術，
Gruber（1995）與 Studer 等人(1998)將知識本體定義為將某領域進行概念化後，
以該領域之專業術語及正規化描述方式而建立之成果，可記錄該領域之成員關
係、邏輯、限制與實例內容。近年普遍採用之知識本體架構包括四類元件：概念
(concepts)/ 類別 (classes) 、關係 (relations) 、公理 (axioms) 與實例 (instances) 
（Stevens，2000；Goble et al.，2000；Keet，2004），概念、關係與公理均可依
需求與性質而再加以細分。各領域於實作時之呈現方式雖不盡相同，但共同點為
以領域具有共識之術語記錄知識本體、記錄各類資料之語意，具有確切表現領域
資料內容及關係之能力與特性(Uschold and Gruninger, 1996)，如此才可建立跨領
域之溝通管道。 
依知識本體之架構或用途，可發展出不同種類之知識本體，前者係依概念化
 6
統進行互操作時，能先行解讀資料之語意、瞭解處理程序之語意。由知識本體發
展之觀點，不同領域可透過標準化之語言與綱要設計其知識本體將可大幅改進後
續結合應用之可行性。為使知識本體語言能以標準化格式流通，全球資訊網協會
(World Wild Web Consortium, W3C)訂定知識本體標準描述語言，最初以資源描述
架構(Resource Description Framework, RDF)、資源描述架構應用綱要(Resource 
Description Framework Schema, RDFS)語法表現知識本體，考量知識本體尚具備
邏輯推理能力及更多元關係之記錄後，W3C 由 RDF/RDFS 擴充語法及記錄規
則，制定標準化之知識本體描述語言─網路知識本體語言(Web Ontology Language, 
OWL)。OWL 目前已廣為知識本體相關研究者所採用(Antoniou and van Harmelen, 
2008)，包括三個次語言：OWL Full、OWL DL、OWL Lite，各自具有不同等級
之邏輯推理能力與 RDF/RDFS 支援度，可彈性依使用者之需求而選擇合適等級
之知識本體描述語言(McGuinness and Van Harmelen, 2004)。地理資訊系統本就為
資訊之一環，若可藉由語意架構而建立知識本體，不但可強化領域知識之描述，
更增加後續跨領域整合應用之可能性。知識本體之建置雖沒有標準化之程序，但
綜合歸納不同學者提出之成果後(Noy and McGuinness, 2001; Subhashini and 
Akilandeswari, 2011) ，可大致區分為下列 8 個步驟：1.確定知識本體使用範疇、
2.考慮由既存之知識本體延伸、3.羅列重要詞彙、4.定義類別及其階層性、5.定義
類別的屬性、6.定義屬性之限制條件、7.建立實例、8.檢驗與評估知識本體。其
中 Subhashini 及 Akilandeswari（2011）特別加入探討知識本體之編碼(encoding)、
整合(integration)和文件化(documentation)的步驟，以形成開放之知識本體描述成
果 。 Horrocks et al. （ 2004 ） 之 研 究 結 合 OWL 與 語 意 網 規 則 語 言
(Semantic Web Rule Language, SWRL)，以進行領域規則之描述，進而推論領域知
識。總結而言，資訊領域在知識本體之建立及相關技術上已有許多成功之案例，
但地理資料之空間特性與傳統文字與數字為主之資料有所不同， 因此無論其上
層共用之概念或特定領域資料之獨有概念都必須進一步檢討，以提供足夠之字彙
及建立具有空間考量之描述架構。

三、研究方法與成果 
本研究之目標為透過語意之觀點探討異質性地理資料之整合與應用，將以知
識本體之建立形成開放及正規化之描述架構，使流通地理資料之意涵可由資料提
供者正確描述，再由使用者正確解讀及應用。此架構將建構於現行流通資料之基
礎上，使實體資料與形成資料之知識在流通層次結合，建構一個智慧型的地理資
料流通及應用環境。依目前地理資訊系統及知識本體發展之狀況，本研究之基本
研究策略為： 
 
1. 依可依實際描述之對象規劃對應之知識本體。 
2. 知識本體應具有基本之架構，以供後續之一致性解讀之參考，但仍需保
持各領域設計之彈性。 
3. 由地理資料共同特性之分析延伸發展上層知識本體之字彙及語意，可供
領域知識本體建立之參考，以避免重複定義。 
4. 善用目前已具有共識之標準技術（例如 OWL），以提升資源重複應用之
可能性。 
5. 知識本體須透過開放架構描述，並可與開放格式之地理資料結合，透過
服務架構進行實質之運作。 
6. 知識本體之設計成果可與詮釋資料之描述內容有效結合，提供後續運作
之基礎。 
7. 強化地理資訊系統軟體有關任務知識本體之發展，納入知識之考量，提
升決策之正確性。 
 
整體而言，流通地理資料之語意及相關知識必須透過具有共識之架構，以開
放、正規及明確之方式描述，並與實質流通資料之識別、處理及應用高度結合，
使電腦系統可以自動化判斷異質性之資料，並突破語意層次互操作性之障礙。但
在同時，我們也希望研究成果可緊密結合現有之標準技術，並強化其不足之處，
 10
得之資料。事實上資料之名稱及定義是領域知識的具體規劃成果，名稱反應領域
所選擇，且具有共識之字彙，是領域內正確溝通之必要工具，每一字彙必須具有
明確之語意，正確定義後，可提供不同領域解讀資料集建立語意對應之參考。基
於各領域具有本身之字彙規劃，且不同領域可能使用相同之字彙，本研究規劃各
領域建立知識本體時，必須發展參考系統（Reference system），使取得任何領域
選用之字彙時，都可正確區隔彼此之差異，也可形成對應關係記錄之基礎。上層
之知識本體必須考量基本（Primitive）及必要(Necessary)之描述架構。針對每個
用以描述主題之字彙，其定義必須具體交代其意義、適用場合、測製方式等內容，
以具體彰顯其特性及相關知識。以目前之發展狀況而言，許多地理資料雖具有名
稱及定義，但僅設計為提供使用者參閱，並沒有辦法由電腦自動判斷後加以應
用。其定義須由人、事、時、地、物等各類因素切入，具體列舉分析之觀點，再
透過選擇字彙及標準化之架構描述，形成更為具體的主題知識。 
 
部分資料為具有較為廣泛主題之資料，其下可進一步區分為不同細緻分類程度之
階層式主題，例如我國之地形資料及國土利用調查資料均屬此類資料。此類資料
除包括各單一主題資料外，尚包括階層式之分類架構，且此架構自然形成繼承之
關係，其語意之間除常見之 is_a 關係外，也包括 part_of、compose_of 等語意關
係。本研究由位相關係之觀點切入，提出可透過 equal、disjoint、overlap、contain、
containedby 描述兩物件之語意關係，就繼承關係而言，A type is a B type 與 A type
之語意 within B type 之語意一般。擴展至一對多之狀況後，前述之 part_of、
compose_of 等也可進一步納入。由於位相關係之推動機制已相當成熟，上述之
語意關係描述可被納入知識本體之一環，提供單一知識本體或兩知識本體之關係
表示，進而建立不同主題資料之間的語意關連及推論機制。此架構納入通用知識
本體後，意味凡以此架構設計之主題資料，均將具有相同的描述架構，方便進一
步建立兩者之間的對應關係。也就是說，未來將可由上層知識本體架構之發展而
減低領域知識本體架構發展及應用之障礙。本研究將在下一節之討論中，透過土
地利用資料知識本體之發展而說明設計策略及具體成果。 
 
（三） 空間因素 
由通用知識本體之觀點，地理資料之空間因素主要包括空間資料型別、坐標系
統、空間關係等三項因素，均涉及必須建立描述之共識，以強化領域資料概念之
傳達。空間資料型別之選擇為圖徵設計之重要因素，但相同物件在不同領域可能
以不同之空間資料型別表示，且參考依據或篩選條件還可能各不相同，因此需納
入資料之知識表示中。坐標系統影響記錄之成果及後續之應用，尤其在結合參考
不同坐標系統資料時，必須確實瞭解其設計之知識，方可能建立正確的對應關
係，也可避免品質之惡化。空間關係為地理資料通用之概念，也被廣泛使用於人
類日常生活之描述，是極為重要的一種空間知識。較為著名之 9IM 或 DE9IM 模
式雖已廣為使用，並由 OGC 納入為空間查詢之規範中，但目前地理資訊系統所
 12
要之條件為建立領域知識本體建立方式之共識，包括使用之字彙、代表之語意、
配合之技術、處理之策略及推論之模式，都必須在共識下發展，才有可能由電腦
自動解讀與判斷相關之內容，並進行決策。異質地理資訊之共享與應用本質上就
是一個非結構化的資訊環境，各類資料由不同領域個別生產，雖然資料格式之應
用障礙在開放地理資訊技術之協助下逐漸被排除，但若沒有從知識本體之觀點切
入，流通的內容僅止於浮面的圖形與屬性記錄內容。本研究由地理資料之共同特
性觀點切入，已明確劃定設計之範疇，並將以此建立領域知識本體及任務知識本
體，請參見以下之討論。

3. 類別名稱、代碼與色碼 
類別(class)為組成知識本體之基本元素之一，故需將欲表達之概念以類
別表示，由於分類名稱已具有共識，且具有定義，故將其選擇為領域字彙，
例如稻作、旱作等。除類別名稱外，尚允許設定各類別之屬性、關係與限制
條件，此三者皆係表達領域知識並作為推論之依據。在土地利用知識本體中，
代碼與色碼皆為類別之屬性。例如稻作類別，代碼為 010101，色碼為
#66FF00(以 16 進位表示)。原始之類別說明為自由文字，使用者透過解讀文
字說明，形成對該領域資料之知識，進而瞭解該詞彙於該領域之意涵。為使
電腦可正確讀取與解讀該分類系統所描述之知識，本研究透過解析各類別之
定義說明，消除贅詞、統整句法、歸納描述詞彙，將自由文字進行結構化，
使各類別詞彙與定義具有明確之意涵表達。描述係以三元組(triple)之方式紀
錄於知識本體，自由文字之說明將經由結構化之程序，以「主語(subject)-謂
語(predicate)-賓語(object)」之架構表示，依其複雜程度，自由文字將被解析
為一至多組之三元組結構紀錄。以判斷為稻作之土地利用資料而言，其說明
文字為「稻作-係指從事稻米栽培之土地。包括水稻、陸稻」，可將之轉換為
「稻作(主語)-has 被種植(謂語)-稻米(賓語)」；又如「倉儲設施：係指供糧食、
肥料、資材存放之設施」，可表示為「倉儲設施(主語)-has 儲存(謂語)-(作物種
類 or 農業設施 or 肥料)(賓語)」。以稻作為例，農糧署每年兩度針對我國稻作
之耕種情形進行調查，並更新其耕地坵塊圖，兩者在語意上即具有關連性，
當兩者均具有知識本體後，可透過結構化之語意描述而建立自動之關連。

3.3 由品質資訊提升任務知識本體 
 資料品質是地理資訊的一項重要性質，但長久以來並未被包括於地理資料
之實際記錄內容中，連帶產生之效應為使用者僅倚賴地圖套疊之結果進行視覺之
判斷，完全忽略異質資料品差異可能帶來之問題。由資料品質之觀點，其產生至
少包括以下之考量： 
 
1. 地理資料品質種類 
2. 地理資料品質評估方式 
3. 地理資料品質描述方法 
4. 地理資料品質記錄及流通方式 
 
理想而言，異質地理資訊若要在互操作性之環境中運作，必然得考量不同資
料之品質差異，而品質差異之比較必須基礎於標準化品質資訊，否則縱然各單位
願意建立品質資訊，對於自動化的互操作應用判斷仍然無濟於事。標準化之品質
資訊必須基礎於專業知識的判斷，將地理資料之品質知識納入知識本體，並進而
與流通資料結合應用，才可能提升品質資訊之應用。ISO19100 系列標準中針對
品質資訊之建立規劃 5 類資料品質元素，分別為資料完整性、資料精準度、屬性
精準度、邏輯一致性及時間精準度，其下再進一步細分，共包括 15 類品質子元
素，可視為地理資訊品質資訊之共識，配合品質量測（ISO19138）及詮釋資料
標準（ISO19115）之規定，每一份資料所建立之品質資訊將可透過標準化之詮釋
資料描述，與資料一併提供。利用這樣的架構，我們得以建立標準化之品質資訊，
無論地理資料種類及其特性之差異為何，當品質描述可被標準化後，在使用者端
的互操作性即可有效改善。由知識本體之觀點，其記錄架構將為 
 
1. 某資料具有標準化品質描述（has standardized quality information）。 
2. 所有品質描述具有相同之架構及項目。 
3. 若採用相同品質量測，則可具體比較異質資料之品質差異。 
4. 若採用不同品質量測，則可透過知識評估是否可以進行轉換及轉換之配套。 
5. 若品質資訊係以符合規範之方式建立，則再納入領域技術規範之專業知識，
由領域建立其獨有之知識本體(domain ontology)，但品質資訊仍將採取標準
化之方式建立。

四、結論與建議 
隨地理資訊蒐集技術及網際網路技術之進步，為來之地理資訊應用將大幅基
礎於資料共享及空間資料基礎建設之機制，但異質性地理資料之障礙將構成成功
推動之一大阻礙。本研究由知識本體之觀點切入，強調語意資訊之重要性，並在
現行之開放資料流通架構之上，發展可正確協助語意應用之地理資訊系統知識本
體架構。透過地理資料共同特性之分析，本研究提出上層知識本體必須包括主
題、空間、時間及品質等面向，並針對其描述架構、使用字彙及運作機制提出基
本的規劃。針對領域知識本體之建立，本研究具體規劃領域分類參考系統之階層
式語意知識本體建置方法，除可描述領域所訂定詞彙之語意外，更可透過結構化
之描述而促進跨領域類別之對應分析。另再以 SWRL 語言建置類別之語意網規
則，成功進行現實土地之土地利用類別歸屬。針對目前極為欠缺之品質知識，本
研究透過標準化之品質描述，結合詮釋資料及品質應用知識，發展可加強地理資
訊系統軟體正確應用之任務知識，此架構可與目前各單位被要求建置標準化詮釋
資料結合，透過系統自動解讀相關內容，並設定必須符合之條件而建立正確之決
策。目前之成果大半仍集中於知識本體基本架構及描述元素之探討，目的在於凸
顯知識本體在異質地理資料整合應用之重要性及必要性，未來必須進一步強化知
識本體與實質地理資料之結合，使地理資料、詮釋資料與知識本體成為未來異質
地理資料共享機制之穩固基礎，可透過電腦系統自動處理，如此方可善用目前快
速成長之地理空間資源。

1 Introduction 
Internet-based geospatial technology has seen rapid growth, 
with millions of users, both novice and professional, using it 
on a daily basis. Although using geographic information 
system (GIS) operations seems to be straightforward and 
produces appealing visualizations in a map interface, a large 
portion of users either ignore the discrepancies of data quality 
between different datasets, or are totally unaware of the 
hidden risks caused by the diversity of data quality. Although 
data sharing has become increasingly easy and convenient, an 
interoperable GIS-based application should not only display 
available heterogeneous geospatial data, it should help users 
correctly interpret and use the acquired data. An approach that 
considers data quality in the process of integrating geospatial 
data is thus necessary.  
Quality information is essential in decision-making. Data 
quality can be defined as the degree of excellence exhibited 
by the data in relation to the portrayal of the actual 
phenomena [1]. ISO/PAS 26183:2006 defines data quality as 
a measure of the accuracy and appropriateness of product data, 
combined with the timeliness with which those data are 
provided to the people who need them. As GIS-based 
applications often involve diverse types of geospatial data 
produced by different organizations, the quality of the selected 
datasets varies over a large range. Users must thus be able to 
(1) determine the quality of selected datasets and (2) correctly 
make decisions based on the quality information. This would 
require a comprehensive examination of the characteristics, 
evaluation, encoding, distribution, interpretation, and 
visualization of geospatial data quality from both theoretical 
and implementation perspectives. In the geospatial domain, 
the fundamental characteristics of data quality  information, 
namely completeness, logical consistency, positional accuracy, 
temporal accuracy, attribute accuracy, semantic accuracy, and  
lineage, are covered in ISO 19113 [2] and a study by Devillers 
et al. [3]. Based on this framework, ISO19114 [5] and 
ISO19138 [6] extend the scope of standardization to include 
the evaluation procedure and quantitative measures of quality 
information. ISO19115 [7] offers a common framework for 
recording quality information via standardized metadata 
elements. These standards not only enable data suppliers to 
precisely convey their knowledge about data to users, but also 
offer the possibility for users to correctly understand the 
quality of acquired data and determine its fitness for use, even 
if data is dynamically acquired from previously unknown 
sources. 
Although consensus agreements on the categorization and 
encoding frameworks for quality information have been 
reached, there is a lack of an effective mechanism that can 
seamlessly integrate quality information with its 
corresponding geospatial data. The concept of quality-aware 
GIS [8][9][10][11] argues that GIS must be aware of the 
diversity and heterogeneity of data quality information and 
offer effective strategies for dealing with the selected datasets 
and metadata. The link of data and metadata was mentioned in 
[12]. Recently, the primitive modeling frameworks proposed 
in [13] offer a standardized structure for encoding the 
identification, spatial and temporal attributes, and quality of 
distributed geospatial datasets. Hong and Liao [14] introduced 
the concept of valid extent to spatially illustrate the data 
completeness status in a GIS-based map interface. Since the 
results are dynamically determined from the data 
completeness status of the selected datasets, the method’s 
success depends on the quality information provided by the 
data owners. The linkage between geospatial data and its 
metadata is the basis for developing quality-aware GIS 
operations in the present study. 
Different from the data quality information given in ISO 
standards, the quality information represented by valid extent 
denotes a new type of quality information that can only be 
deduced after a particular GIS operation, e.g., map overlay, is 
issued. As the number of georesources increases, it becomes 
less likely and even impossible for users to rapidly build a 
comprehensive understanding of the selected datasets.

Abstract 
As the sharing of geospatial data becomes increasingly easy and convenient, the discrepancy and heterogeneity of data 
quality between datasets dynamically acquired from various georesources must be taken into consideration. Such awareness of 
data quality must extend to the design of geographic information system (GIS) operations, which GIS users often naively use 
to analyze and derive new information. This study proposes a quality-aware approach for incorporating standardized quality 
information into the design of GIS operations to ensure that the necessary quality information is always visible in a map 
interface for correct interpretation. Since the proposed approach can largely reduce user effort when dealing with unfamiliar 
datasets, it can be seamlessly integrated with current sharing frameworks based on geospatial web services to facilitate user-
friendly and interoperable geospatial applications.

3 Quality-aware Approach 
The fundamental concepts for developing a quality-aware 
GIS are: (1) all distributed features are self-described and 
published in open data formats; (2) applications have built-in 
knowledge to intelligently parse necessary metadata according 
to the GIS operations, and (3) applications present the 
evaluation results of quality information to users in a 
meaningful way to aid decision-making. A simplified 
workflow of the proposed quality-aware GIS approach is 
illustrated in Figure 2, which subdivides the whole process 
into three major stages, namely the modeling, quality-aware 
GIS application, and decision-making stages.  
 
Figure2: The concept flow of Quality- aware approach 
in distributed environment. 
 
 
 
The major goal of the modeling stage is to ensure that the 
distributed geospatial data and quality information are self-
described. Individual datasets are created according to their 
spatio-temporal conditions using one of the primitive 
frameworks proposed in [12]. Information about the 
identification, spatial and temporal attributes, and 
dataset/feature-level of quality are respectively created. Data 
providers are responsible for the modeling process because 
only organizations participating in the production and 
distribution of geospatial data have in-depth knowledge about 
the specification, restrictions, and quality of the datasets. 
From the perspective of SDI, this requirement must be 
considered while establishing the collaborative relationship 
between participating organizations.  Otherwise users will not 
be able to take full advantages of the geospatial data available 
from SDI. 
A quality-aware GIS application must parse the required 
metadata created in the modeling stage and intelligently 
determine how to use the acquired data. Although the 
algorithms for each operation are designed independently, 
common principles may be shared by a number of operations. 
Quality-aware GISs are mainly designed and operated by 
application users with professional expertise because the 
functions required are application-dependent. Finally, every 
GIS operation must be examined to ensure that the design of 
its processing strategies and outcomes takes the necessary 
data quality information of geospatial data into consideration. 
This is the responsibility of GIS software developers or 
vendors. In the decision-making stage, the results are 
transformed into visual aids and presented to users. 
Using data completeness as an example, Figure 3 shows the 
modified workflow after the consideration of quality 
information has been added to the design of a GIS operation.  
 
Figure 3: Workflow of quality-aware spatial operation. 
 
 
The operation begins with the user selecting a number of 
datasets. The application automatically parses the information 
of the surveyed area, completeness status, and positional 
accuracy of the datasets from their metadata. Whenever a new 
dataset is selected, the parsing procedure is triggered to update 
the quality information for the analysis module. To provide 
meaningful visual aids about the data completeness status, 
three conditions must be tested. Firstly, the valid extent is 
determined by calculating the geometric intersection of the 
surveyed area of selected datasets. If the valid extent is empty, 
a warning message must be issued to users to indicate that 
some information is missing. Secondly, the analysis module 
needs to test whether the spatial extent of the queried region is 
entirely within the valid extent. If not, a message should be 
issued to users. Finally, the completeness status of all the 
selected datasets must be “YES” to make the valid extent 
meaningful. If it is not, a warning message is shown to users. 
Since the judgments are based on the features being within the 
valid extent, positional accuracy is also included for user 
reference. Note that this analysis depends on the selected 
datasets and metadata, so the valid extent must be recalculated 
after any modification to the selected datasets. In the decision-
making stage, the information of the valid extent is

5 Conclusion 
This study integrated the standardized encoding of quality 
information and the fundamental characteristics of GIS 
operations to improve the correctness and interoperability of 
results. By incorporating the proposed quality-aware approach 
into GIS operation design, the operations detect possible risks 
during data processing and then provide meaningful visual 
aids in a map interface. With increased sharing of data from 
various organizations, the proposed approach offers a 
comprehensive framework for considering quality information 
and increasing the interoperability of GIS operations. With its 
open nature, the proposed framework can be easily developed 
in a service-oriented architecture environment to improve 
internet-based applications.

2.2 The components and structure of land use ontology 
 
The land use ontology is designed to encompass the 
fundamental properties of land use data discussed in section 
2.1. The land use ontology can model the hierarchical 
semantics of land use data and provide the knowledge about 
the distinguished characteristics of the individual land use 
type. Fig. 2 shows the design of land use class and its 
corresponding metadata, metadata explains the fundamental 
information of individual version of ontology. Since 
ontology can be seen as an electronic resource, Dublin Core 
[11] is used in metadata design to record necessary 
information in a standardized way, e.g. title, subject, 
publisher, date, and rights, etc. Version information is 
recorded by the item of owl:versionInfo and 
owl:priorVersion. The item of “scale” is used to denote at 
what scale the ontology is used. The taxonomy of land use 
data is presented as hierarchical classes. Properties of 
LU:Land Class includes the geometry, coordinate system, 
land use color, class name, land use id, reference, definition, 
etc.. The Sub class represents the taxonomy at the second 
level or the third level. All sub classes inherits the properties 
of LU:Land Class. 
The structure of land use ontology consists of three major 
parts. The first one is land use taxonomy (Fig. 3 left); the 
second is the land data that needs to be inferred (Fig. 3 
middle); the last part is the structure for recording the 
definition of land use concept/classes (Fig. 3 right). 
LU:Land class has properties, i.e. geometry, coordination 
system, reference resource, and definition, etc.(Fig. 4). The 
input data could be imported as instance in LU:Land class. 
That is, a land could be reasoned in land use ontology and 
known what land use type a land belongs to due to it has 
structural definition.

4. CONCLUSIONS 
 
To facilitate the interoperable use of geospatial data across 
different domains, the process and consideration for 
developing domain-specific ontology is demonstrated using 
land use data of Taiwan as examples. The land use ontology 
is designed to model the hierarchical semantics of land use 
data and enable users to perform reasoning via formalized 
knowledge about land use data. The proposed framework 
consists of metadata, land use ontology components and 
definition structure. These three structures work seamlessly 
to provide domain knowledge about selected data. This 
framework offers advantages of re-use, flexibility for 
expansion and semantic query via ontology. We further 
demonstrate that the links between geospatial data and its 
corresponding ontology can be established with Open 
technology, such that users can easily acquire the necessary 
knowledge about the geospatial data being used. Although 
the current framework is only designed to meet the 
modeling demands of land use data, the ultimate goal in the 
future is to develop a common framework that can be used 
for developing the ontology for any selected geospatial 
domain, so expansion to current components and associated 
methodology will be considered in the future.

二、 與會心得： 
各國在資訊科學方面的進展都是此次應令人印象深刻體驗，由其是雲端技術
結合地理資訊的整合應用以及 GIS 在社群與語意上的發展，無論是技術的發展
或理論的研究都有令人讚嘆的地方。而在資訊科學的研究領域部分，由參與發表
的文章數來看，不難發現此領域研究的廣度與深度。同時與會的學者中，除資深
有名的學者外，大部份都是年輕的學者與學生，此也同時反映出經驗交流與傳承
的意味。 
透過研討會的熱烈討論，筆者認為在網際網路及開放式地理資訊系統技術陸續移
除資料流通之技術障礙後，各應用領域應不再抱持「所有資料都由自己生產」之
觀念，反而更可藉由國家空間資料基礎建設中之不同資料來源而取得其他業務單
位生產之空間資料，加以整合，並進而開發相關應用系統。 
當資料之應用因 Web Service 技術之引入而挑戰「即時」之資料整合與應用
時，資料品質描述與語意的問題無疑更應受到重視。地理資料為某範圍空間現象
在特定時間狀態之描述結果，資料取得之方式及篩選原則將影響記錄之內容，也
因此影響資料之各類品質評估結果。在對資料品質一無所知的情況下使用空間資
料，使用者之決策將可能因此潛藏危機，甚至可能造成後續更大且嚴重之錯誤而
不自知。若能獲得一份標準化且描述良好的資料品質資訊，對於資料之瞭解與後
續之正確應用將有莫大之助益，再透過語意的分析，降低異質性資料整合分析的
門檻，協助各資料生產領域建立一套正確之品質評估程序及方法，允許更為廣大
之使用者了解取得資料之品質，將成為後續國土資訊系統擴大應用的契機。 
而透過本次研討會的參與，可認識同領域的各國研究者，透過會議間之提問
與會後之交流，可由不同面上了解同一個問題，更可因此刺激思考，提升問題發
現與解。 
 
三、 發表論文全文或摘要 
AGILE 2012 – Avignon, April 24-27, 2012 
 
Without a solid theoretical framework that considers the 
outcome of GIS operations, decision-making using geospatial 
applications may become extremely difficult because even a 
simple GIS operation may have to deal with a complex and 
unpredictable mixture of data quality information. All GIS 
operations must thus have a built-in capability to 
automatically determine the applicable conditions of the 
selected datasets based on the quality information. Different 
types of quality information may need to be taken into 
consideration for different operations. Extended metadata 
elements and implementation strategies may need to be 
considered in addition to the current metadata standards. 
The interoperability of GIS operations is not limited to the 
issues of reading GIS data formats or illustrating selected 
datasets in a map interface. The design of GIS operations must 
take correct data use into consideration. This present study 
proposes a quality-aware approach for improving the 
interoperability of selected datasets. The rest of this paper is 
organized as follows. Section 2 introduces the encoding 
strategy of data and quality. Section 3 describes the proposed 
quality-aware approach for distributed application 
environments. Section 4 uses two examples to demonstrate 
the benefits of the proposed quality-aware approach. Finally, 
Section 5 summarizes the major findings and suggests future 
directions of development. 
 
2 Encoding Strategy of Data and Quality 
For a quality-aware GIS, to ensure that all the quality 
information can be properly interpreted by users, the 
distributed data must be both open and self-described. As 
geospatial data is dynamically selected according to 
application needs, the quality status after data integration must 
be dynamically determined. Theoretically, every quality 
description has its own data scope, which represents the 
domain of the data from which the quality information is 
evaluated. This scope information must be unambiguously 
specified for every individual quality evaluation result. 
Although the majority of current quality information refers to 
individual datasets, it may also refer to a dataset series, feature, 
or attribute. For example, the quantitative measures for data 
completeness are based on the omission error and commission 
error after the dataset has been compared with the universe of 
discourse, so the data scope by default refers to datasets. The 
enriched set of metadata elements given in [13] (e.g., 
surveyed area of the dataset) also refers to a single dataset. 
The positional accuracy may refer to either a dataset or a 
feature depending on the positioning technology or the testing 
specifications. Table 1 lists the data scopes and corresponding 
quality elements considered in this paper.

The strategies proposed here follow the standardized 
framework of self-described features proposed in [12]. In 
addition to the quality information, every feature is required to 
include a ValidTime attribute to denote its temporal status. 
This helps applications determine whether the GIS operations 
are dealing with features that coexist at the specified period of 
time. This type of information should be considered in many 
GIS operations, but are often ignored.

distributed datasets allows applications to transparently parse

4 Use Cases 
The following discussion uses two widely used GIS 
operations, select by region and distance measurement, to 
demonstrate how the quality information can be assimilated 
into the design of GIS operations. Decision-making for a 
school evacuation caused by flood is chosen as the test case. 
A system prototype for these two operations was developed in 
Visual Basic using ESRI ArcGIS 9.3. 
The commonly-used algorithm for the “selection by region” 
operation is very simple; users specify a region as the spatial 
constraint and the GIS responds with the information of the 
features within the specified region. It is typically regarded as 
a geometric operation, where only location is taken into 
consideration. Users can easily determine the schools that 
need to be evacuated by specifying the predicted flooding 
region as the constraint and using the operation “selection by 
region” to determine whether the location of schools meet the 
"within” condition. This, however, only works when the 
surveyed area of the school dataset contains the spatial extent 
of the flooding region. Under such circumstances, it can be 
ensured that all of the schools within the flooding region are 
found. Figure 4 shows the school evacuation scenario. The 
results would be rather different if such data quality 
information is not taken into consideration.  
 
Figure 4: Valid extent of school and flooding datasets. 
 
Because the surveyed area of the school dataset only covers 
a portion of the map interface, users have no information 
about the number and spatial distribution of schools outside 
the surveyed area. The green polygon in Figure 4 represents 
the overlapped region between the predicted flooding region 
and the surveyed area of the school dataset. With the addition 
of quality information, it is validated that all schools within 
this region have been found. The yellow polygon indicates the 
surveyed area of the school dataset outside the flooding region. 
Schools within this region do not need to be evacuated. The 
blue polygon indicates the subpart of the flooding region 
where no information about schools is available. Without the 
consideration of data completeness, a user may naively 
assume that the schools being found within the green region 
(e.g., the ValidExtent in Figure 4) are all the schools that need 
to be evacuated, which may lead to erroneous decisions. 
Therefore, the operation of “selected by region” is not as 
simple as it appears to be. The testing of the surveying area 
for each selected dataset needs to be continuously updated 
with changes of the selected datasets, and a warning message 
should be alert when the surveying area don't completely 
contain region. The message box in Figure 4 is automatically 
prompted to users to indicate that users should be cautious 
about the data completeness status of the searched results. 
Furthermore, visual aids (e.g., the green and blue polygons in 
Figure 4) must be promptly presented to remind users of the 
data quality status of the illustrated content in the map 
interface.  
The parsed quality information can be presented to users in 
an integrated interface for reference. Figure 5 and Figure 6 
show an interface that simultaneously presents information 
about the surveyed area, completeness, and position accuracy 
of each dataset. If the measure of the data completeness is 
Boolean, users can make reasonable decision based on the 
features within the valid extent (e.g., Figure 5). Users will also 
notice that only 23% of the flood region is within the valid 
extent from the presented information, so the queried result 
may not represent all the schools in the flood region. If the 
measure is based on the percentage (e.g., 98% in Figure 6), 
the selected features may not represent all the schools within 
the valid extent, so the number “98%” must be presented to 
users to avoid wrong decision making.  
None of these mechanisms work if no metadata about the 
selected datasets is available. The proposed approach thus 
allows the contributions from data providers, application 
developers, and software designers to be effectively integrated.
