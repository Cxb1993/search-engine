【多媒體服務元件模型】 
1. 分散式物件計算 
為了要設計一個多媒體元件模型，必須先探
討 分 散 式 物 件 計 算 (Distributed Object 
Computing，DOC)的概念。通常提到分散式物件
計算環境，指的是使用者所需要的物件(程式)可
以在執行時期才加以連結，並呼叫其所提供的服
務函式，通常藉由中介層(Middleware)來抽象化物
件存取的細節；此外中介層也提供了必要的服
務，以確保物件的一致性等需求。如圖 1 所示，
DOC 中介層介於作業系統與應用程式之間，並
隱藏了網路傳輸的細節。 
 
圖 1：DOC 中介層示意圖 
 
在分散式物件的環境下，用戶端與伺服端之間
的通訊，通常是利用遠端函式呼叫(Remote 
Procedure Call，RPC)的方式來達成，也就是在用
戶端建立遠端物件的本地參照，讓用戶端在使用時
不知道(也不必知道)實際物件的所在位置。我們通
常稱遠端物件為skeleton，而用戶端的的物件參照
為stub。圖2顯示了這兩者之間的關係。 
 
圖 2：stub 與 skeleton 的關係 
 
這種架構稱為以物件為基礎(Object-based)的
分散式計算，另外還有以事件為基礎(Event-based)
以及以訊息為基礎(Message-based)的架構，適合
用於多點傳輸的分散式環境。本計畫採用的是以
物件為基礎的架構。這種架構簡化了用戶端呼叫
遠端函式的行為，並可自動產生 skeleton 與
stub，並且能加以管理與認證。 
分散式中介層平台標準中，最有名的就是由
OMG (Object Management Group)所提出來的
CORBA 標準 (Common Object Request Broker 
Architecture)，而微軟亦提出了 COM/DCOM 
(Distributed Component Object Model)以及最新
的.NET 架構。此外 Sun Microsystem 所推行之
Java 版本 RPC，稱為 Java RMI(Remote Method 
Invocation) 以及 Sun Open Net Environment (Sun 
ONE)，都是目前主流的 DOC 環境。 
然而，若將 DOC 架構套用在多媒體系統
中，卻缺乏了一些重要的特性，例如考慮 
利用 COBRA 來傳送媒體串流，因為 COBRA 標
準中，所有的資料傳輸都是可信賴的(reliable)，
並且所有的函式呼叫都是同步的(synchronous)，
也就是呼叫者會被 block 住直到函式呼叫已完
成，此外，COBRA 也不讓使用者去存取傳輸時
的網路 QoS 參數，這些特性都不適合有時需要丟
棄封包，並必須滿足即時限制條件 (Real-time 
constraint)的多媒體傳輸。有鑑於此，COBRA 已
經提出了適合多媒體分散式系統的延伸版本，包
括了 A/V COBRA 以及 RT-COBRA。圖 3 為 A/V 
COBRA 的示意圖。 
 
圖 3、A/V COBRA 
 
本計畫參考 A/V COBRA 以及 RT-COBRA 
等標準，設計出一套合適的多媒體元件模型。

【多媒體行動商務系統】 
隨著近年來無線網路與 pda 及行動電話的進
步與發展，電子商務漸漸將他的領域從原本的個
人電腦延伸到行動平台上，我們稱之為行動商
務。然而，因為行動平台上的總總限制，行動商
務的建立並不如傳統的電子商務一樣簡單且便
利。而我們實作出了一個系統，重新劃分了行動
商務中每位參與者的任務，並且引進了 business 
rule 來讓業者能以最快速以及最方便的方式將他
們的服務建立在行動平台上。在另一方面，我們
也考慮到使用者以往在使用 mobile device 時所遇
到的種種不方便性，針對這些問題給予改善。 
在現行的電子商務中，使用者一般都是使用
配有鍵盤以及滑鼠的電腦來使用，除此之外，所
使用的顯示工具(一般來說是使用螢幕)也都是據
較大的顯示範圍，所以將這套電子商務的架構搬
到行動手持裝置時，對使用者來說是個不方便的
系統，所以我們透過二維條碼與線上即時通訊軟
體的來簡化使用者的操作，讓行動手持裝置上的
電子商務能更為方便的使用。圖 6 為本系統的一
個架構圖。

三、結論與討論 
本計畫以二年的時間建立上述所提的研究架
構。實作出一個多媒體中介軟體系統，並以此為基
礎，開發出網路互動電視與多媒體線上購物系統來
驗證此介面的可行性並評估其效能。且將新的研究
成果發表為論文推向國際。而本計畫之技術將對於
現今消費電子商品的開發有一定的影響，我們將會
把研究成果轉換為有助於提升實際應用的模組。並
與產業界密切配合，提供產品研發之先進技術。 
計畫參與者可藉由這個計畫接觸到最新的想
法與技術，並進而提出更先進的技術。就各個分項
技術而言，可獲得的相關訓練包括： 
多媒體系統設計：本計畫將整合各個不同領
域的多媒體技術，包括了聲音/圖片/影像的播放、
處理、傳送等等。 
中介軟體設計：Middleware 的技術一直以來
僅限於在資料庫以及網站等應用，然而，將來的
SOC 產業日趨複雜，各式各樣的 Middleware 的需
求將快速增加，因此，計畫參與者將獲得寶貴的開
發經驗。 
本計畫不僅為國家培育研究人才，也對國家
推展的 SOC 產業、數位內容等計畫以及數位台灣
之願景提供幫助，並且提供了新一代的數位生活藍
圖。
