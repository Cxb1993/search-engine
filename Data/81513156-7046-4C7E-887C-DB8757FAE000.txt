z 研究目的： 
本期研究計畫必須建立於前兩年的卓越
研究成果，我們將前兩年所開發出來之智慧
型高速高品質影像壓縮理論以系統方式實現
不僅僅是演算法上的改進，更配合系統硬體
的控制來完成一智慧型高清晰監控系統。除
此之外，本系統也進一步結合了微電腦周邊
硬體，並將行動通訊裝置配合嵌入式系統將
網路連線軟體做修正，成為一個多功能智慧
型監控系統(Versatile Intelligent Surveillance 
System)。 
在今年度的計畫發展中，我們結合了
RTP（Real-Time Transport Protocol）遠端視訊
傳輸技術，以將監控視訊即時且高畫質的呈
現於網際網路和無線行動裝置上。如此，本
研究計畫所開發之編碼技術擁有相當高速的
效能，故即使在多個監控畫面下，仍然能夠
保證監控畫質與流暢度，其主要之創新與貢
獻列表如下： 
一、將高速、高清晰度之影像壓縮理論以系
統方式實現 
二、結合 3G 行動網路與手持式裝置 
三、使用者可無時無刻、隨時隨地瀏覽監控
影像 
 本研究計劃將架構於開發一套以小波轉
換技術為基礎之具備高速度、高影像品質與
高壓縮倍率之視訊編碼技術，並結合 RTP 遠
端視訊傳輸、影像擷取卡與 CCD 攝影機，設
計製作一套高效率的多錄影通道智慧型監控
系統。同時，本研究亦將進一步地結合了行
動裝置，成為一個具備行動能力的行動監控
系統。 
 結合了以上眾多具有實用性與創新性的
視訊編碼技術，其擁有相當廣泛的應用層
面，除了一般監控用途外，運用在運鈔車與
救護車的即時遠端監控、智慧型交通道路即
時影像分析系統、視訊會議、視訊電話與網
際網路遠距教學等應用範疇均可發揮其所
長，並可結合了微電腦周邊硬體，及行動通
訊以一個建立全方位的數位化家庭安全監控
中心。

定義了資料的傳輸限定，所以能否達到即時
傳輸要看 RTP 下面一層的傳輸協定是使用什
麼通訊協定，例如：RTP/TCP/IP，RTP 的下
層通訊協定是 Transmission Control Protocol
（TCP）[1]，則表示每筆傳輸的資料必須經

Server Push 有很多實做方式（依其內部
的 Protocol 不同而不同），但依據 Netscape 
Navigator 1.1 的 Server Push Protocol，其實作
是採用 MIME 的 Message 型態來完成的，對
於一般的 HTTP Request，如 Client 讀取一個
影像檔， Server 首先回應 Content-Type: 
Image/jpg，告訴 Client 接下來的資料要以
JPEG 的格式來解讀，之後 Server 才會送出影
像檔的資料，當檔案完全送出後，連線便中
斷了，由此可以得知 MIME Message 是 Client
判斷 Server 所送資料解讀方式的重要依據。 
對 Server Push 來說，由於要有持續的連

multipart/x-mixed-replaced，另外為了要區別
