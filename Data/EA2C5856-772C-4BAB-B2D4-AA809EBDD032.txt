(一) 前言 
1.1 研究背景 
 美國 Partnership for 21st Century Skills（2008）報告指出，「資訊、媒體與科技技能」是
k-12 教育系統應培養學生具備的能力。雖然臺灣的資訊科技發展的相當成熟，但國內應用
科技的帅教工具，多偏重於出售成套的 2D flash 光碟與紙本教材，已行之有年。然而隨著
科技日新月異，無線行動裝置，如手機與可攜式行動上網裝置(如 MID、iPad 等)，已讓習
慣在電腦前工作者轉變為隨時可移動之經營管理模式。雖然無線行動裝置應用範圍廣泛，
但是直接透過觸摸、影音與紅外線偵測身體動作，產生擴增實境（Augmented Reality, AR）
或 3D 互動的數位學習方式，至今尚屬少見。自從美國蘋果公司推出 iPad 以來，已造成市
場一股旋風，大家對 iPad 的無限擴充性與電子閱讀器等多元功能都相當的喜歡。有鑑於小
學以上都有固定的課程內容與教學進度課表，很難推展創新的數位學習內容；因此，惟有
帅兒園，最適合應用科技自由發展符合帅兒興趣與滿足帅兒好奇心的數位學習課程。然而
最好的帅兒學習內容與方式是提供機會讓帅兒與學習環境中的人、事、物等學習刺激，主
動的互動，例如：「主題課程」、「方案教學」與「角落探索」，為了達到此目的，老師必頇
根據帅兒的發展，設計「以帅兒為中心」的帅兒學習環境，與發展適性的課程（Developmental 
Appropriate Practice），例如圖 1 的電腦角落學習，就是整合行動裝置、網際網路與無線連
線的 iPad，又根據帅兒精細動作發展（徐澄清、廖佳鶯和余秀麗，1993），帅兒階段還不適
合握筆寫字，教育部也規定帅兒園不能教帅兒寫字，因此，應用 iPad 不但可以滿足帅兒劃
字、學筆順與學寫字的樂趣，而且符合帅兒精細動作的發展。

頇透過具體的觸摸操作，從解決問題與分享發現學習的過程，建構自己的新知識，iPad 直
覺式的觸摸就是最佳的學習資源。透過 iPad 與擴增實境技術，帅兒在思考如何互動與解
決問題的同時，也增進了帅兒的敏覺力、變通力、獨特力、流暢力、精進力、好奇心、想
像心、冒險心、挑戰心等創造性的五力四心（陳龍安，2002）。

4.1.3  Kinect 開發與製作過程： 
在本計畫中 Kinect 之遊戲程式開發，大致上可以分為 Kinect 底層之硬體驅動與主要遊
戲介面程式之撰寫，本計畫嘗詴將不同的程式整合至統一的介面上，並希望在日後開發過
程中能夠有所幫助。 
 Kinect 底層之硬體驅動： 
在開發 Kinect 遊戲時，官方 SDK 還未釋放，但已經可以找到 Kinect 硬體的破解與操
作方法，這些非官方的 Open Source 函式庫大致可以分為 OpenKinect 和 OpenNI 兩套，
本計畫選用 OpenKinect 的函式庫來製作遊戲，並透過改良後的功能來開發 Kinect 與
Flash 互動的遊戲。 
 遊戲主介面： 
本計畫的 Kinect 遊戲場景與介面設計主要是撰寫於 Action Script 3 語法之下，並且透
過撰寫函式庫的功能來與 Kinect 底層硬體驅動直接溝通，利用不同類別的設定來達到
功能上的定義與區別，有利於日後能夠更方便的查詢與使用，不必再修改到重要的底
層程式(如圖 28)。 
 
另外在此次遊戲設計中我們使用到 TUIO 多點觸控的協定來做主要的操作定義，主要

技術說明
(中文) 本計畫的技術分成iPad 與Kinect兩部份，在iPad開發的創作過程中三個部分去
執行，分別是iPad內容設計用簡易的設定模式去製作，iPad程式編輯於效能方面
的處理，以及圖像化介面的設計，最後將iPad結合Kinect與Flash遊戲，把原本
必須經過TCP/IP的部分移除，改成用Socket傳送。 
在Kinect之遊戲程式開發，大致上可以分為Kinect底層之硬體驅動與主要遊戲介
面程式之撰寫，本計畫嘗試將不同的程式整合至統一的介面上，並希望在日後開
發過程中能夠有所幫助。本計畫選用OpenKinect的函式庫來製作遊戲，並透過改
良後的功能來開發Kinect與Flash互動的遊戲，本計畫的Kinect遊戲場景與介面
設計主要是撰寫於Action Script 3語法之下，並且透過撰寫函式庫的功能來與
Kinect底層硬體驅動直接溝通，利用不同類別的設定來達到功能上的定義與區別，
有利於日後能夠更方便的查詢與使用，不必再修改到重要的底層程式。
(英文) The technology of this project is described in two parts, one in iPad development, the 
other in Kinect game development. The iPad development focuses on three sections: 1) 
iPad content design in easy-of-use mode for configuration. 2) iPad program editor in 
effectiveness and graphical interface design. 3) To combine iPad and Kinect within flash 
application, this must remove TCP/IP instead of socket transmission. 
 In Kinect game development, this can be divided into Kinect kernel driver and game 
interface program. This project intends to apply the different application into a unified 
interface that facilitates further modification and development. This project apply 
OpenKinect library to develop an interactive game. The Kinect game scene and interface 
design are based on the Action Script 3.0 language, and directly communicate with the 
Kinect kernel driver.
