多功能遠端監控與故障診斷的行車資訊系統之研製 
吳建中 
南台科技大學資訊工程系 
wucc@mail.stut.edu.tw 
 
摘要 
本系統開發出一套可以結合透過3G數據
通訊與 ODB-II的協定架設一個可以遠端監控
汽車的即時資訊的系統，可以即時將行車電腦
中的相關數據、衛星定位座標、即時行車影像
等的訊息透過無線通訊傳遞給使用者監控；同
時也可以將資訊傳遞至遠端伺服器中儲存。 
 
系統中透過DMA-PXA270開發板並架在
Embedded Linux 2.6.9 作業系統下完成，此系
統透過 ELM327 晶片連結汽車診斷系統，提
供故障即時診斷的功能，當汽車行駛中發生故
障時也可以直接點選液晶螢幕中的選項查詢
故障車輛故障狀況與原因，並在車輛故障時讓
遠端車廠專家立即提供車輛故障診斷，提供立
即協助並將資訊提供給前去維修或拖吊業者
資訊，方便給予正確的協助。 
 
除此之外，本系統提供一個跨平台及非常
友善的人機介面，讓使用者可以很方便透過網
頁瀏覽的方式即時遠端查詢包括行車電腦、即
時行車影像、目前汽車所在位置與地圖以及車
輛故障資訊。同時透過收集個別汽車回傳的資
訊整合在伺服器中，可以監控多台汽車狀況的
車隊監控系統。 
 
關鍵詞：嵌入式系統, 汽車診斷系統, 遠端監
控, 遠端故障診斷, 車載資通訊。 
 
ABSTRACT 
The proposed system that to develop an 
intelligence transportation system, including 
tachograph, on-board diagnostic system, 
wireless communication system, and database 
system. In addition, the information in the ECU, 
the GPS position, real time driving image and 
sensing information will be transmitted to 
remote server by the wireless communication 
system.  
 
Besides, the on-board diagnostic system 
will be integrated with this project. The trouble 
codes can be showed in the LCD panel and be 
remote monitored by using web browser. It can 
work well regardless of the working platform, 
the use of operation system. The real-time and 
correct information will be supported to the 
roadside rescue system.  
 
The hardware of the designed system is 
based on the DMA-PXA270 development board 
and it is based on the Embedded Linux 2.6.9. 
Besides, the diagnostic information can be 
retrieved by the ELM 327 module.  
 
Keywords: Embedded system, ODB-II, 
remote Control, Remote Diagnostic, Telematics. 
 
前言 
自從西元 1885 年德國工程師賓士先生將
內燃機裝配在座椅下方，並連結兩個動力用的
輪胎，以及一個方向用的輪胎這一台三輪車就
在歐洲上路後。不斷的進步至今，已經過了數
十後，汽車的噴油系統也從西元 1967 年開始
第一台由電子控制噴油[1]。 
 
隨著如環保、重量、維修、價格等多種因
素的考量，汽車內部的車內網路已由傳統一堆
電線連接控制的方式改採用由電子網路系
統，包括：控制區域網路匯流排(Controller area 
network ， CAN) 、 本 地 互 接 網 路 (Local 
interconnect network，LIN) 、媒體導向系統傳
輸(Media-oriented systems transport，MOST)
等等[2]，透過電子網路中匯流排的連接可以
減少車上電線的重量，更有利於未來車輛的維
修。然而也正因如此，因為汽車功能的複雜度
增加，汽車故障已經不是單純由經驗來判斷故
障發生原因，有時會因為高山氣壓或氧氣濃度
系統中的遠端監控與故障診斷的行車資
訊系統則分成幾個子系統分別進行，其中包
括：汽車行車紀錄系統、車上診斷系統、行車
通訊系統與遠端監控系統四個子系統。汽車行
車紀錄系統紀錄跟行車資訊相關的資訊，例
如：車速、引擎轉速、冷卻系統溫度等等；車
上診斷系統判斷汽車故障的狀況；行車通訊系
統主要偵測車輛對外通訊狀況，並作為車輛與
外界傳輸相關資訊的媒介；遠端監控系統則負
責提供一個互動式、跨平台及友善的人機介
面，讓外界查詢車輛目前狀況。 
 
1.2 系統設計 
 
嵌入式開發板架構圖如圖 2，為了降低系
統的電源消耗，本系統採用 Intel®  PXA270 為
核心的嵌入式系統開發板，並使用 Embedded 
Linux 2.6 作為系統開發的核心，透過 Linux
電源管理與記憶體管理的機制可以大幅提高
系統運作速度與降低系統耗電的需求，並採用
通用異步收發傳輸器(Universal Asynchronous 
Receiver/Transmitter , UART)介面接收來自於
衛星定位系統的訊號進行汽車定位，透過採用
Quallcomm 3G 通訊晶片的 DMA-3G Modem
進行行車通訊的介面，並透過 ELM-327 USB
介面跟汽車電腦進行溝通[7]。 
 
 
圖 2. 系統方塊圖 
 
除 此 之 外 ， 在 系 統 中 加 入 一 組
USB-Webcam 進行行車影像的擷取，本系統
影像視訊部分建立於嵌入式開發板上，並於開
發板整合 WebCam 驅動，使用 Java 將影像壓
縮，透過影像串流技術與 3G 數據網路方式，
將資料儲存在系統中或是傳給伺服器主機，其
方塊圖如上圖 2 所示。 
 
1.3 軟體架構設計 
 
1.3.1 行車通訊系統 
對於行動通訊介面，由於 Wi-MAX 目前
尚未普及，本計畫採用 Quallcomm 3G 通訊晶
片的 DMA-3G Modem 作為通訊介面[9]，由於
此介面仍屬 modem 撥接性質，在建立通訊時
取得 IP 地址是浮動的；即是每次都可能不相
同。因此，本系統中增加自動紀錄目前 IP 位
址的機制，並透過 3G 數據網路與伺服器主
機，將更新的 IP 位址紀錄在伺服器中，其示
意圖如圖 3 所示。 
 
 
圖 3. 行車通訊系統連線流程 
 
1.3.2 遠端監控系統 
 
本系統將對外提供遠端連線監控都先連
到伺服器網頁後，根據點選模式及車輛，系統
將會將網頁轉到車輛上，此種設計主要為了降
低汽車通訊的流量，除了即時影像外，其他包
括行車資訊、車輛故障碼以及 GPS 座標都詴
透過文子資料傳給伺服器主機[10]，伺服器網
頁將會透過收到的 GPS 座標透過 Google Map
的雲端服務，及一些圖形介面提供給使用者較
友善了畫面，同時對於帳號認證相關的管理也
在伺服器端端進行，其連線流程如圖 4 所示。 
 
 
圖 4. 遠端網頁監控網頁連線流程 
 
圖 5. 遠端監控與故障診斷的行車資訊系統
的實機照片畫面 
 
系統中白色視訊擷取裝置(Webcam)採用
OV511，可以將其安置在儀表板或是想要監控
的地方，Quallcomm 3G 通訊晶片的 DMA-3G 
Modem 進行行車通訊的介面，ELM-327 USB
介面則透過 USB 跟 DMA-270 開發板連接，
另一頭則接到車子儀表板下方的 ODB-II接頭
中，本系統是透過 TOYOTA Wish 的 ODB-II
作實際量測，並可成功的量測到相關的訊號，
並以 3G 數據通訊方式儲存至遠端伺服器中。  
 
本系統提供一個跨平台及非常友善的人
機介面，即時透過網路將行車電腦中的相關數
據、衛星定位座標、即時行車影像等的訊息透
過無線通訊傳遞給使用者監控，同時也可以將
資訊傳遞至遠端伺服器中儲存。除了可以紀錄
行車資訊，同時當汽車行駛中發生故障時也可
以直接點選液晶螢幕中的選項查詢故障車輛
的故障狀況與原因如圖 6。 
 
 
圖 6. 嵌入式開發板上的觸控式螢幕 
 
 
除此之外，透過遠端可上網電腦，也可以
直接透過網頁瀏覽器連接到車上 3G modem
的 IP 位址，也能監控車輛的狀態與座標，未
避免網路流量太大影響傳輸品質與造成功率
消耗擴大，如圖 7 直接連接到的網頁僅顯示
GPS 座標，並無透過 Google Map 的圖形畫面
顯示汽車位置，其連線畫面如所示，左方顯示
的是目前行車即時視訊畫面，右側則是汽車相
關的資訊，汽車位置則透過文字方式顯示GPS
座標。 
 
 
圖 7. 遠端連線至車內網路監控畫面 
 
本系統提跨平台方便的人機介面，直接透
過網頁就可以使用，只要夠過可以瀏覽網頁的
行動裝置或電腦都可以輕易的讀取車輛即時
的資訊，如圖 8 是透過智慧型手機觀看行車
資訊系統[12] 。 
 
 
圖 8. 透過智慧手機連線的畫面 
 
 
結，因此，汽車的座標改成由 Google Map 的
方式呈現。因為流量只會增加在伺服器端，不
會造成行車資訊系統流量的增加。 
 
利用此成果參加「2010 年全國大學院校
嵌入式系統設計競賽」獲得「設計完成獎」的
肯定。 
同時，發表四篇會議論文，並將成果公開
在自由軟體的網站上[20]。 
 
為了增加伺服器的效能以及減少負擔，加
入了近年來頗具話題的雲端伺服器，將 270
改成 real 6410 並增加了車輛與駕駛的行為分
析，透過行為分析，可以了解駕駛以及車輛的
狀況，此成果參與「2010 第六屆數位訊號處
理創思設計競賽」的「硬體系統整合組」，並
獲得入圍的肯定，並將修正初版的缺點及增加
功能。 
 
    另一方面，將嵌入式板改成智慧型手機，
利用藍牙連結汽車診斷系統，增加系統的穩定
度，利用此成果參加「2010 第六屆數位訊號
處理創思設計競賽」的「行動裝置平台應用
組」，獲得入圍的肯定，繼續競賽爭取佳績。
