摘要 
建立一個觀光導覽管理定位服務之平台架構，此平台架構結合無線射頻辨識系統技術達成
定位服務並且整合產品資訊系統為例達到定位導向之 U(Ubiquitous)化空間服務。利用 RFID讀
取器與個人行動助理(PDA)整合，使用者可藉由行動裝置讀取 RFID 資訊並且透過無線網路將
定位資訊與服務閘道伺服器(Service Gateway Server)對應其定點位置以及導向使用者所需服
務，進而將產品資訊系統以定位導向之方式提供空間行動化服務。另外本文之目的主要是藉由
網路 SOA(Service Oriented Architecture)架構開發定位導向服務平台提供使用者可透過個人行
動裝置達到定位服務，而以Web Service的技術將後端資訊系統以彈性化的方式匯入而將原有
的資訊服務轉化為定位行動化資訊，藉由此定位導向服務平台之架構應用於溫泉導覽各資訊系
統，而使用者只要透過行動裝置以及無線網路則可獲得產品無所不在的服務。 
關鍵詞：資料管理系統，人機介面，產品定位

依據上述架構則可分為三部份： 
NB使用者介面以及 RFID模組：必須提

(b) 提供使用者顯示介面，必須顯示來

驗證訊息等等。 
(c) 使用者登入介面，此為提供使用者
輸入帳號與密碼的介面。 
(d) 資料回傳介面，部分的資訊，例如：
貨物清點、點名單等等，此類型
的資訊必須有回傳功能，所以必
須建立回傳的介面。 
經過此架構的運用，可以把多方面的服
務做整合，讓使用者可以藉由簡單的介面以
及便利的行動裝置而達到行動化服務的目
標。

後台管理系統製作 
本計畫最重要部分在實作一個定位導向
服務的平台架構，此架構包含了 Application 
module、Database module、Management and 
Configuration module三個模組架構：
Application module代表與行動裝置之溝通橋
樑，其中包含了行動裝置Web-based人性化操
作介面以及定位導向服務伺服器；Database 
module主要為儲存資訊以提供管理平台的所
有資訊；Management and Configuration 
module此模組為提供資訊的管理平台，其中
包括了產品名稱、產品定點、產品說明以及
觀光導覽管理附加資訊等等，也進而讓管理
者可以藉由此管理平台觀察定點服務的提供
的效率等等。 
本文的目的則是以上所敘述四大模組的
整合來實現此定位導向服務平台的架構，此
平台架構用意在於結合 Location-Based 
Service的架構以及彈性化的整合現有的產品
資訊系統，而讓所有的產品資訊行動化以達
到 U化服務的概念，如圖 3所示。

4. 服務紀錄控管模組 
此模組主要用於評估服務的定點問
題以及服務的使用率控管，當龐大的服

好，而良好的管理平台必須擁有控管服
務紀錄的能力，才能夠依據這些數據來
修正定點位置並且評估服務的使用率，
藉此來了解資訊的傳達率或是顯示方式
是否符合使用者，並且清楚表達資訊，

品工廠資訊或是針對系統的各項意見做
納入，進而改善並且延伸更多服務。 
Web Services module 
XML Web services(以下簡稱 web 
services)[6]主要是提供網路服務的架構，其功
能就是將服務註冊至網路上並且提供給網路
上的使用者使用，而 web services主要採取
HTTP為通訊協定並且與 web的應用環境緊
密結合，前端的應用程式可透過 SOAP的方
式向後端 web server索取服務， 
本文之平台架構則是藉由web services之
服務機制實現彈性化擴充服務資訊系統，其
步驟如下： 
1. 後端服務資訊平台可藉由
UDDI(Universal Description, Discovery, and 
Integration)的標準來完成所有的服務註冊，以
提供給定位服務應用伺服器搜尋並且使用，
主要功用為做集中式的服務目錄。 
2. 而註冊完成的服務可以藉由
WSDL(Web Services Description Language)標
準來完成服務的描述，它也是一個以 XML為
基礎的描述語言，用以描述一個Web services
所支援的方法、參數和傳回值的型別等資料。 
3. 中間則藉由 SOAP(Simple Object 
Access Protocol)機制來完成定位服務應用伺
服器以及後端服務資訊系統之間的溝通，
SOAP屬於一種包裝訊息的機制，主要功能是
讓雙方清楚互相傳送的資料格式。 
4. 其中，兩伺服器間的資料傳送則是以
XML的格式傳送資訊。最後藉由 HTTP協定
傳送以上資訊。 
而藉由以上步驟，則可彈性化的擴充後
端服務資訊系統，當將所有服務註冊完成並
且將服務相關資訊紀錄於定位服務應用平
台，如圖 4所示。

份，希望把重點放在群組研發過程中產品資

處理及不斷的設計進化，產生最好的共識結
果並且能在良好的權限規範下進行資料分
享。

六、未來研究方向 
希望未來經過校務資訊系統長時間的評
估以及改善後，可以將此位置服務導向整合
資訊平台擴展應用，例如：與校園服務結合，
藉由單一 RFID tag 代表某一服務，而節省傳
統排隊填表格之傳統服務，進而節省人力、
時問，以及與全業、公家機關、教育單位等
等結合，將多項服務實際整合於資訊平台並
且擴展服務範圍，使之成為一完善的位置服
務導向整合資訊平台。應用上可在: 
1. 商業：藉由行動裝置發送要求，取
得使用者活動範圍附近的 ATM、餐
廳或是相關的特價活動等等。 
2. 交通：由使用者行動裝置接收警告
訊息，提供駕駛者交通壅塞路段或
是相關於交通工具的訊息，例如：
加油站或是維修場。GPS衛星導
航：在汽車上使用的導航系統，會
隨著汽車開到不同的地方而改變顯
示之地圖，並且提供景點的資訊等
等。 
3. 物流：使用者透過行動裝置取得該
定點之物流清單，並且配合使用 RF
的相關技術執行清點貨品的動作，
進而將盤點資訊回傳給予資訊系統
而達到節省人力與時間的成效。 
4. 藝文娛樂：走進一個國際會議場地
或是展場裡，行動裝置顯示出場地
的地圖，走到演講廳裡，演講者的
資料及講義與展品資訊馬上就在行
動設備上顯現出來，這些都是典型
的定位服務。  
5. 醫療：一般用於醫院患者，患者透
過 RF的硬體裝置結合感測網路，一
方面達到院方可以追蹤患者另一方
面也可配合相關服務便利患者。

建立一個觀光導覽管理定位服務之平台架構，此平台架構結合無線射頻辨識系統技術
達成定位服務並且整合產品資訊系統為例達到定位導向之 U(Ubiquitous)化空間服
務。利用 RFID讀取器與個人行動助理(PDA)整合，使用者可藉由行動裝置讀取 RFID
資訊並且透過無線網路將定位資訊與服務閘道伺服器(Service Gateway Server)對應其
定點位置以及導向使用者所需服務，進而將產品資訊系統以定位導向之方式提供空間
行動化服務。另外本文之目的主要是藉由網路 SOA(Service Oriented Architecture)架構
開發定位導向服務平台提供使用者可透過個人行動裝置達到定位服務，而以 Web 
Service 的技術將後端資訊系統以彈性化的方式匯入而將原有的資訊服務轉化為定位
行動化資訊，藉由此定位導向服務平台之架構應用於溫泉導覽各資訊系統，而使用者
只要透過行動裝置以及無線網路則可獲得產品無所不在的服務。
