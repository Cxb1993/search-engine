中文摘要 
關鍵詞: FlexRay, 線傳駕駛, 電磁干擾, 錯誤注入平台, 失效模式與影響分析, 安全度/
強韌度驗證, IEC 61508, EN50159, 容錯機制, 安全等級 
全球汽車工業的趨勢正逐漸將傳統的機械式/液壓式控制系統取代為電子式控制系
統。其益處有可節省成本、能源以及提高系統的性能與安全性。然而電子控制 系統會
比傳統的機械或液壓式系統更容易受到電磁波、輻射粒子或串音(Crosstalk)的影響。此
種現象當製程進入深次微米時代後變得更加嚴重，導致軟 性錯誤(Soft error)的發生機率
比以往高出許多。當晶片運用到汽車電子上時，系統運作必須具備更高的強韌度。現今
的高安全性應用可遵循的國際安全規範以 IEC 61508 最為普及。而專為汽車電子式控
制系統制定的安全性規範－ISO 26262，即以IEC61508 為基礎。FlexRay 為先進的車用
網路通訊協定，可用以架構高安全性的線傳駕駛系統(Drive-by-wire)。建構FlexRay 線傳
駕駛系統時，有兩個重要議題值得探討：如何有效驗證系統安全性/強韌度以及當系統的
安全性不能達成預定目標時，如何有效地加以提升。本計畫將針對FlexRay 線傳駕駛系
統提出有效的強韌度驗證及強化設計流程。在此設計流程中，首先透過錯誤注入後得到
Failure Mode and Effect Analysis (FMEA)結果，並藉此評估FlexRay系統的安全性。假如
系統的強韌度達不到安全性規範的安全等級(Safety Integrity Level)要求時，可根據FMEA 
結果找出系統的弱點並加入容錯機制加以克服。此設計流程將會透過FlexRay 線傳駕駛
系統的個案研究，來展示其可行性及有效性。成果展示包括FlexRay 線傳駕駛系統的
FMEA 結果，FMEA 結果是否能幫助提升容錯設計的有效性，並藉此評估提出的不同
容錯設計各自對系統強韌度的提升程度，這些成果將會成為將來設計高安全FlexRay 線
傳駕駛系統時的重要參考資料。

簡介 
隨著嵌入式系統的發展成熟，智慧型系統的應用也越來越廣泛。而智慧型系統的應
用，常常是與人類生命安全有相當大的關聯(如智慧型汽車駕駛系統及智慧型機器人)。
所以，對於系統可靠度/安全度的要求是相當嚴苛的。目前有許多的智慧型系統被應用在
汽車的控制系統當中，而汽車工業的趨勢是取代以往傳統機械式和液壓式系統，將控制
系統轉換成電子式的控制系統。這樣的替代方式能有效的節省成本/節約能源和提高性
能，以及安全性。然而使用電子控制系統卻比機械系統或液壓系統較容易受到電磁波
(electromagnetic interference, EMI)、輻射粒子干擾(particle strike)或是串音(crosstalk)的干
擾。因此，汽車電子系統安全性和強韌度的問題，必須在開發過程中能有效的解決。系
統晶片使用在智慧型系統上，已是一種必然的趨勢。隨著製程不斷的向更深次微米 (very 
deep sub-micron) 發展，晶片的電晶體變得愈來愈小、容量愈來愈大、系統功能越來越
強與多樣化、執行速度也愈來愈快和越來越低的操作電壓，造成電路對neutron and alpha 
particles的敏感度增加，導致soft error 的發生機率大為提高，造成晶片發生錯誤及失敗
的機率也相對提高。除了系統晶片被使用在智慧型的車用控制系統外，當電子控制單元
與其它電子控制單元需要溝通時，就需要藉由通訊網路架構來達到溝通的目的。以電子
控制單元來說，受到深次微米的影響，比較容易受 single event upset (SEU) 的干擾，而
通訊網路架構方面容易受到 EMI 的干擾，這也是目前車用通訊網路系統所面臨的重要
問題與挑戰。 
如何驗證智慧型控制網路系統是否符合安全性與強韌度的要求，就成為一項非常重
要的研究課題。因此，可靠度將變成未來設計系統晶片時，必須考量的一項重要因素。
而對於智慧型控制網路系統而言，若是只有保護系統晶片是不夠的，因為控制系統除了
系統晶片外，通訊網路架構也是會影響系統運行可靠度/安全度的重要單元，所以可靠度
/安全度也是通訊網路架構在設計時，必須要考量的重要因素。所以有兩個主要的課題需
要去探討：一個是車用控制系統可靠度/安全度測試驗證及分析平台的開發，另一個是如
何提升車用控制系統的可靠度/安全度。由於控制系統本身的設計已經夠複雜了，一旦要
將安全度及容錯設計的議題也加入在控制系統的設計中，將會使控制系統的整體設計更
加的複雜及增加系統成本。因此要考慮加入容錯設計在控制系統中去提高系統的安全度
時，一定要知道系統所面對的錯誤類型有那些、單點錯誤或是有可能多點錯誤、錯誤發
生在系統中不同的單元所造成的失效反應及造成的危險是如何、控制系統本身的脆弱點
又在什麼地方、最需要加強保護的單元有那些，以及元件和系統的安全等級有沒有符合
規格要求？這些問題都需要有一個完善的安全度測試驗證及分析平台來幫忙。而一旦知
道系統的脆弱點後，就要針對所找到的脆弱點加入容錯設計，來提高控制系統的安全
度。在加入容錯設計後還需要回到安全度測試驗證及分析平台，再次進行安全度分析。
在安全度測試驗證及分析平台開發，以及加入容錯設計來提高系統的安全度時，都需要
一個很重要的元素，就是錯誤資料庫的完善與否。所謂的錯誤資料庫就是把系統欲驗證
分析的元件，針對這些元件上可能發生的干擾和錯誤，建立元件完整的錯誤注入資料，

2. FlexRay 系統容錯設計方法 
在針對FlexRay節點、FlexRay節點之間的通訊、FlexRay節點之間的同步進行錯誤注
入，並且得到安全性分析驗證的數據後，我們應以此數據為基礎來加入適當的容錯技術
以改善系統失效率，進而提升系統的安全度。但目前限於實驗設備的限制，我們無法更
改節點內部的硬體架構，因此無法針對FlexRay節點內部暫存器進行保護。另外一方面，
針對FlexRay節點、FlexRay節點之間的通訊與同步的保護，則可以透過硬體冗餘
(Hardware Redundancy)與資料冗餘(Information Redundancy)兩種方式來實現，如1.2節所
展示的實驗系統。本研究的具體做法則是以1.2節中的實驗平台為基礎，在原來的系統設
定中，節點2只接收同步訊框，不做同步訊框的傳送。為了實現容錯機制，我們將該系
統設定更改為三個同步節點，傳送六個訊框。六個訊框與五個訊框的系統，差別在於節
點2也會傳送同步訊框，給另外兩個節點，也就是3節點都會傳送同步訊框，給另外兩個
節點作同步使用，其他排程皆不變。但關鍵在於因為多增加了一個同步訊框，因此系統

和同步訊框在發生延遲錯誤時，資料訊框接收成功率和同步訊框造成同步失敗的機

static slot 的長度，對於資料訊框接收成功率是相當有效的。從表 11 static slot = 54 的

以下針對同步訊框來做比較，在固定 APO = 3 參數的資料中，我們多加入了 static 
slot = 56, 58 兩項實驗來更完整的比較固定 APO 與變動 APO 的同步失敗發生比率。由
表 12 可以看出在 static slot 長度 54 以後，固定 APO 的系統同步失敗機率大幅上升，
當達到最大延遲極限Δt = 6us 時，系統同步失敗機率最大為 30.5%。其原因為當固定
APO = 3 時，不管 Idle 或 static slot 長度如何變動，同步失敗發生時間點皆從 4.25us 開
始。因此 Idle 區段加大只會讓該區段範圍更大，反而提高系統的失敗率。而在相同的
static slot 長度下，加大 APO 和減少‘Idle’長度，能達到延後同步失敗發生的時間點，
即系統對於同步訊框延遲錯誤的容忍能力提升。針對傳輸訊號為資料訊框時，我們發
現固定 APO 在接收成功率上來得比增加 APO 高，但當延遲錯誤發生在同步訊框時，
結果則相反。所以設計者需要同時考量頻寬使用率、資料訊框接收成功率以及同步訊
框延遲錯誤對系統同步的影響，來找出較佳的 APO 及‘Idle’值。 
上述發生這些同步失敗的行為，是因為節點 3 接收到節點 1 傳送過來的延遲同步
訊框，經過時脈同步程序 (clock synchronization process)來校正節點時脈，包括 
‘offset’及 ‘rate’校正。此時校正過的節點，因為是根據延遲同步訊框的時間，反
而造成同步節點 3 和同步節點 1 之間的同步出問題。因此造成接下來傳輸訊框之間的
碰撞，如圖 9，同步節點 3 收不到節點 1 的同步訊框，發生 MISSING_ TERM 錯誤，
最後導致系統同步失敗。在系統參數設定上我們將 gMaxWithoutClockCorrectionPassive
設為 3，gMaxWithoutClockCorrectionFatal 設為 5。也就是連續三個奇數週期發生
MISSING_TERM，節點會進入 normal passive，此時無法傳送訊框出去，只能接收外
來的同步訊框，做自己內部時間調整。若時間調整回來，節點回到 normal active 繼續
傳送訊框。若是調整失敗，連續達到五個奇數週期發生 MISSING_TERM，節點則會
進入 halt 狀態。

3. 結論 
在此計畫中，我們執行了 FlexRay 系統的安全性驗證，且建立安全性的驗證分析
平台。利用此平台來執行錯誤注入實驗，收集結果並統計分析，幫助我們更了解
FlexRay 系統的失效行為。此次計畫的成果為提升 FlexRay 系統強韌度提供了非常有
用的參考數據。在整個強韌度/安全性驗證分析和容錯設計過程中，我們將 FlexRay 系
統的失效行為分為 FlexRay 節點、節點外部的通訊與節點之間的同步三個方向來探討。 
在節點內部錯誤導致的系統失效行為方面，我們開發了一個線傳轉向系統作為實
例展示。將錯誤注入到節點內部 CPU 暫存器中，藉此找出對系統失效影響最為顯著的
暫存器，並觀察系統有哪些失效模式、影響與估算系統的失效率和安全等級。 
而在節點通訊與同步方面，依據 EN50159 標準來開發錯誤類型產生器，提高了
實驗的效率與方便性。利用所建立之平台分析 APO 與 static slot 在不同的參數調整下，
對於延遲錯誤的容忍程度。實驗數據顯示，隨著 APO 或 static slot 長度增加，系統能
容忍的錯誤延遲時間也越長，No effect 機率明顯增加，表示容錯能力上升，但也降低
了通訊的頻寬使用率。但當接收訊框為同步訊框時，則又需要考慮到同步失敗的範圍
是否會增加，反而提高了系統進入 halt 的機率；而在改變 MISSING_TERM 容忍參數
