摘要 
個人化醫療是健康照護的趨勢，特別是癌病的治療；癌病的療程是依病患的
狀況量身訂製，而療程的規劃需彙整病患的各種相關資料。但由於資料的來源種
類眾多，且散佈於不同的系統與部門。所以在規劃的過程中，不但必須往返參照，
而且還要自行彙整與歸納，相當耗時費事。再者目前的處理方式無法發揮這些資
料應有的價值。本整合型計畫的主要目的在於以個人化圖譜的概念，分享癌病相
關醫學資料。以完整呈現病患的狀況，輔助診斷與治療。除了資訊的整合之外，
本整合型計畫還將進一步發展醫學影像處理與基因表現分析之技術與工具，如此
即可完整地提供歸納、分析與模擬功能，充分發揮腫瘤資料在治療、教育、研究
方面的潛力。本計畫是整合型計畫的子計畫一，主要是提供基礎架構；負責前端
的使用者管控與後端的資料整合。本計畫的目的在於提供各子計畫協同工作的環
境、資料的保全與資料的流通。本計畫擬以新的醫療資訊協定來彙整各種異質資
料，並呈現在網頁系統上。使用的技術核心為 IHE ITI（Integrating the Healthcare 
Enterprise Information Technology Infrastructure）與服務導向架構（Service Oriented 
Architecture；SOA），提供安全、迅速、可靠的資訊分享環境。所提出的架構不
但可以輔助癌病的治療、教育與研究，且可供健康照護機構沿用於發展醫療資訊
系統。 
關鍵字：病患導向、電子圖譜、IHE ITI、服務導向架構（SOA）

1. 前言 
個人化醫療（Individual medicine）是未來健康照護與醫療的趨勢。現今醫療
院所朝向醫療專業分工化發展，許多臨床檢查是採用委外進行，例如：生化檢驗
服務。如果要以現行醫院資訊系統架構來整合這樣分散式的臨床資料做為臨床、
教學以及研究使用，將會遇到許多困難。在早期大多數的文獻認為要達到病歷交
換的目的，首先必須要將病歷格式統一。只要病歷格式統一後，則電子病歷的跨
院交換將不是問題。因此過去許多研究陸續提出病歷格式轉換的機制與方法讓病
歷能在跨院之間達到病歷語意上的互通（syntax interoperability）。由於病歷的範
疇十分廣泛，許多學會或國際組織針對專科或不同部門的病歷格式分別訂出不同
的標準，造成目前全世界的病歷格式標準已達數十種。若是訂定出放諸四海皆準
的統一格式則花費的成本過高且實作上十分困難，加上各醫療資訊廠商定義病歷
格式的內容不盡相同，導致雖然病歷格式符合標準但是系統彼此間仍存在整合性
的問題。為了解決這個問題，IHE（Integrating the Healthcare Enterprise）從 2003
年開始陸續針對跨院交換的作業流程以及規格制定許多規範（ Integration 
Profile），其中最重要的規範為 XDS（Cross-Enterprise Document Sharing）。此
規範主要為建構電子病歷跨院交換平台的一個通用架構。從 2004 年起，每年舉
行全球性的 XDS 連測活動（Connectathon）以解決病歷交換所面臨到互通性的問
題。2010 年美國的 IHE XDS 連測活動一共有 104 家醫療資訊廠商，接近 5 百位
測試工程師進行 3,650 項互通性測試。目前全世界以 XDS 做為跨院交換的地區
一共有 59 個，且有超過 1/3 已經商品化。因此，目前全球醫療資訊廠商皆有共
識地以 IHE XDS 做為未來電子病歷跨院交換之技術架構。 
2. 研究目的 
本計畫將依據 IHE ITI（IT Infrastructure）技術架構以及網際網路資訊整合服
務（SOA）的技術建構各系統整合所需的資訊傳輸機制。利用 IHE XDS 分散儲
存與集中索引的概念，將分享的資料不管其內容為何，皆視為文件；並以最低更
改限度整合目前病患存於各方的病歷資料、檢驗資訊及個人日常生理參數。所有
的資訊將以病患為中心（Patient-Oriented），索引屬於此病患的健康資訊，達到
3 
提供一個臨床資訊中心來整合不同類型的醫療機構。希臘 HYGEIAnet 平台則發
展出一個開放式的健康資訊基礎建設（Health Information Infrastructure；HII），
提供工具讓許多獨立的應用與平台共用。在香港醫院管理局也建置了一個臨床資
訊系統（Clinical Information System；CMS）與電子病患記錄（Electronic Patient 
Record；ePR），其範圍則是包含整個香港地區臨床資料的縱向整合。 
4. 材料與方法 
本研究目的在於建置個人化醫療平台。軟體方面，使用 Microsoft ASP.Net 
（C#）動態網頁技術作為資料來源端（Source）與客戶端（Consumer）的開發平
台；以.Net Framework 以及 WCF（Windows Communication Foundation）技術設
計文件儲存庫（Repository）與文件註冊中心（Registry）。IHE XDS 的概念為「文
件分散式儲存、索引集中管理」，其主要角色與交易流程如圖 1 所示。

4.3 交易（Transitions） 
在 XDS 互相交易流程中，可分成兩大類：文件註冊與上傳（資料來源端）
與文件檢索與取得（客戶端）。「文件註冊」負責資料註冊與文件索引；「文件
檢索與取得」為檢索文件索引與取得文件，這些交易步驟結合的流程細節如下：  
¾ 文件註冊與上傳 
分成兩大交易程序：Provide and Register Document Set-b（ITI-41）以及
Register Document Set-b（ITI-42），相關說明如下：首先參照規範後，
我們制定流程範圍，設計出可提供多文件以及詮釋資料上傳至文件儲存
庫。在 ITI-42 中規範，當文件與詮釋資料上傳至儲存庫後，儲存庫將詮
釋資料上傳至註冊中心。詮釋資料提供相關文件的資訊與儲存位置
（Repository UID），由註冊中心解析接收的資料格式與內容是否正確
並解析後，以註冊中心自身系統制訂的方式儲存。完成索引註冊動作，
以供客戶端查詢及下載文件。 
¾ 文件檢索與取得 
分成兩大交易程序：Registry Stored Query （ITI-18）以及 Retrieve 
Document Set-b （ITI-43），相關說明如下：首先參照規範後，我們制
定流程範圍，將設計出可提供多文件以及詮釋資料上傳至文件儲存庫。
另外為了防止 SQL 隱碼攻擊，將非法的 SQL 指令傳至註冊中心，所以
將原本的 SQL 指令儲存在註冊中心。當客戶端選定所要查詢的類型以
及輸入所條件參數，傳至註冊中心解析，即可查詢。另一方面，可以將
查詢方式根據程式開發者的對於資料庫設計不同而有彈性的改變。客戶
端利用資料庫當裝特定欄位指定需要使用的 SQL 語法，若輸入的條件
正確，即可查詢到搜尋的內容。 
5. 結果與討論 
本平台整合使用者健康記錄並包含醫療院所的病歷資料與檢驗資訊，並透過
網頁程式實作新增元件，令本平台提供個人健康資訊多元化呈現以及應用。其架
構如圖 3 所示。經由傳輸層安全協定（Transport Layer Security；TLS）從醫療院
所內部不同的資訊系統產生的病歷報告、檢驗資料或由健康照護中心收集的各種
生理資訊；透過詮釋資料產生器（Metadata Generator）產生符合 IHE XDS 標準
的詮釋資料，並將詮釋資料送至註冊中心註冊。本平台將入口網站（Portal Site）
與註冊中心結合，將接收的詮釋資料轉換成關聯式資料庫儲存，提供客戶端在單

一、議程 
整個議程分為：（1）亞太中華地區醫療資訊現況的分享會議、（2）廠商應用於醫
療的解決方案說明會議與（3）醫療資訊相關產品展覽。 
¾ 第一天 
1. 第一天上午的內容主要是 HIMSS 博覽會的前置活動―中華醫院訊息網絡大
會。分別由中國當地衛生相關部門或機關人士上台報告分享當前中國在醫療
資訊領域的發展概況，同時也指出：除了運用資訊系統幫助解決複雜的醫學
臨床流程之外，亦持續推動電子病歷內容設計與電子病歷系統的建置，以朝
向互聯、互通的目標邁進。 
2. 另外還有 Microsoft 與 IBM 等廠商代表上台分享在中國地區對於醫療資訊領
域的貢獻與參與。 
3. 下午則是有幾場以學術及廠商輔導醫資為主的專題會議同時進行。分別參加

二、與會心得 
1. 對於中國大陸在醫療資訊的發展速度真的是頗為驚人！ 
2. 會議地點雖仍在北京市，但已屬北京外郊，交通除非是搭乘計程車，否則須
多次轉車才可到達，乃此次過程中稍稍美中不足之處；所幸住宿與會議場所
同屬一棟，免去舟車勞頓之苦。 
3. 或許真的是人口眾多、城鄉差距等因素，當地醫療尚需多方資訊手段的資源
投入來做改善與改造。 
4. 往返的旅程途中，發現中國北京經過 2008 奧運的洗禮，的確蛻變不少，整
體的城市基礎建設與公共運輸網絡真的讓人印象非常深刻。 
三、展覽活動 
1. 會場以國際性及當地知名廠商為主，所提供的產品服務首重醫療資訊相
關，也有一些結合資訊軟體的儀器或設備廠商，但主要都是在提供醫療服
務應用的解決方案。 
2. 有多家提供醫學影像及傳輸系統（PACS）建置的服務，且多結合檢驗資訊
系統（LIS）成套提供。 
四、建議 
1. 會議的註冊作業似乎仍有待改善，處理效率與應變能力皆不足。會場的安全
人員與服務人員似乎對於會議要求與規定均不是很了解，人員訓練仍有很大
的改善空間。反觀我們國內研討會其實也有類似的弊病。 
2. 會議地點雖同屬華語地區，但畢竟是國際性會議，外語溝通仍是必要，尤以
英語更是國際性共通的語言。咸少以英語溝通的機會，一旦想與外籍人士進
一步了解其詳細內容時，總是會不知如何流暢地表達所想要知道的領域或範
疇。因此平時英文的表達訓練相當重要！
