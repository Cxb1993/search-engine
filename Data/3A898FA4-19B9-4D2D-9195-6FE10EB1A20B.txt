A. 問卷系統後台 
1. 問卷新增-提供問卷作者建立新的問卷。 
2. 問卷列表-系統內所建立的問卷，提供所有的問卷預覽、問卷內容的修改以及刪除
問卷。 
3. 統計報表-系統內已完成的問卷，是為各個問卷結果之分析的統計資料。

B. 問卷題型說明 
問卷調查系統提供的題型樣板，可分為以下幾類型： 
1. 「單行文字輸入」、「多行文字輸入」這類型的開放式的題目，提供受訪者任意填入
各種想法或意見。 
2. 「單選型-4」、「單選型-5」、「單選型-7」、「單選型-9」、「複選型-4」、「複選型-5」、
「複選型-7」、「複選型-9」這類型的選擇題型除了可以輸入一般的選項，還提供可
自行輸入文字的"其它"欄位供受訪者輸入。 
3. 「同意度題-3」「同意度題-5」、「同意度題-7」、「同意度題-9」提供您設計問卷問
題的同意度或是滿意度之級距的題型。

技術特點說明： 
 
本研究係發展行動裝置之問卷系統，主要是運用在行動平台之上，以 HTML5.0 之技術
為主： 
1. 問卷系統伺服器 
本研究所採用的伺服器為 Linux 作業系統(Ubuntu)，以 Debian 為開發藍本，每
六個月會發佈一個新版本，而使用者可免費且即時地獲得此作業系統。Ubuntu
的最大重點在於讓個人電腦變得簡單易用，並包含最新的 GNOME 桌面環境。 
在伺服器上採用 Apache、PHP、MySQL、phpMyAdmin 的方式進行，所有的軟
體皆為免費使用，可大幅降低使用者建置問卷系統的成本。 
 
2. HTML 5 
HTML5 可以說是 HTML、CSS 和 JavaScript 的一套整合技術，可以減少瀏覽器
的外掛程式，如 Adobe Flash。HTML 5 多了許多標籤，如：用於繪畫的 canvas、
用於媒體的 video 和 audio，以及內容呈現的 article、footer、header、nav、section，
和表單控制的 calendar、date、time、email、url、search，這些新的標籤將有利
用網頁介面的開發，以減少原先大量使用 Flash 及 Javescript 的方式。 
 
3. 網站伺服網頁開發、Restlet 
“Restlet”是以 Roy Fielding 所闡述的 REST 概念，提供了一個輕量級而全面性的
框架，而達到適用於客戶端及伺服端的應用方式。Restlet 在 HTTP 之客戶端與
伺服器之間在架構上並沒有太大的差別，不需要用不同的開發套件，也就是說
應用程式可以是 Web 客戶端，也可以是 Web 伺服器。 
 
4. HTML 5.0 圖表功能 
HTML 5.0 之圖表功能係採用 RGraph，它是免費提供做圖表的函式庫，係以
Javascript 為基礎，支援包含有 Tooltips、Visual effects、Zooming、Context menus、
Interactive adjusting、Custom events、AJAX、JSON、Dynamic updating、Resizing
等等。
