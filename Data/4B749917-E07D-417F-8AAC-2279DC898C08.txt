摘要 
本計劃主要探討利用 802.11(Wi-Fi)短距離無線通訊技術於老人居家照護之應用，在
不需改變被照護者日常生活習慣下，加入無線通訊科而使被照護者的位置得以被追蹤，因
而可以得到完善的照護。照護者對被照護者進行追蹤與觀察，當發生緊急事故時可以減少
尋找受照者所在位置的時間以避免延誤治療。此計劃運用 Wi-Fi 的識別與定位的功能建置
一套居家照護定位系統，此系統可以觀察被照護者目前的位置和先前位置記錄以及停留時
間長短。我們利用 AP 建構無線環境，使之能涵蓋全部被照護者居家生活空間，以利觀護者
得以掌控各種突發狀況，並將位置資訊記錄於資料庫以為各種照護工作參考。 
 
關鍵詞：居家照護、GPS、AP、Wi-Fi、WiMAX、PDA 
 
Abstract 
This project focuses on the implementation of 802.11 (Wi-Fi) technology for home 
healthcare systems. Without changing the elderly people’s activities of daily life, we use Wi-Fi 
technology for the caregiver to track their real time position. Thus, they can have properly cares. 
When accident events occur, the time to find them can be reduced and the delay for proper 
treatment is limited to a minimum. In this project, we use the Wi-Fi-based tracking system to 
identify and record the elder people’s daily life activities. The system can track and record their 
current and previous locations as well as the sojourn times. With AP’s, we implement a wireless 
system so that all the activity area can be covered. Thus, all the situations is under the control of 
the caregiver and all the location information is recorded for future reference.  
 
Keywords：Home-Care、GPS、AP、Wi-Fi、WiMAX、PDA

3.2 Wi-Fi 的探討 
 
IEEE 802.11b（又稱Wi-Fi）規格是 IEEE 組織在 1999 年 9 月所制定出來的高速無
線區域網路標準，因當初802.11規格制定時，所定義的速率是在 1 至 2 Mbps左右，而 
802.11b 則可高達 11 Mbps，不但在資料的傳輸速率上大大的提昇，而且與原先的 802.1 1
規格是完全相容的。 
IEEE 802.11b 規格當初發展的目的是為了提昇企業內部網路通訊的機動性、節省安裝
有線區域網路的成本、以及更彈性的利用網路上有限的資源，所以802.11b 是一項被設計
用來進行長距離或高速數據資料傳輸的技術。 
IEEE 802.11b 非常適合在室內的環境下使用，只需在室內部架設一個無線橋接器
（Access Point），則在其有效發射範圍內，就可以跟裝有無線網路卡的電腦達到網路連
線的目的，同時 802.11 b 無線傳輸技術在室內的傳輸距離可達 100 公尺之遠，傳輸速率
最快也可達 11 Mbps，如搭配上強波器，則可資料的有效傳輸可以更遠（300 公尺）[3]。 
 
4.研究方法 
 
本研究主要運用無線AP來當作基地台與被照護者使用內含Wi-Fi的設備來進行定位的
主要的設備，首先要設置 AP 的位置，使被照護者在任何地方都有訊號含蓋，這可以使我們
隨時監測被照護者所在地方，而如何放置在下面章節會說明，整個研究是要使照護者可以
很清楚被照護者所在位置，這可以讓我們更方便去照護。 
 
4.1 流程架構 
此研究是由被照護者(client)手持或配載含有 Wi-Fi 的設備，此設備包含平板電腦、
筆記型電腦、PDA 或者是任何嵌入式系統配備，使得 AP 可以搜尋得這些裝置，並透過我們
所設計的定位系統計算出這些裝置所在的位置。而要如何放置 AP 使得訊號可以含蓋被照護
者所會走在任何地方，我們在下面 3.2 節會詳細說明，而 Server 是負責計算與儲存資料的
地方，讓我們寫撰寫的系統負責搜尋並儲存資料至資料庫中，醫護人員也可以透過 PDA 來
了解被照護者所在位置、相關資料，此系統也可以與醫療院所做結合，讓醫療院所了解被
照護者現在的情形與位置。 
 
4.2 網路架構 
在居家照護定位系統的網路架構裡，我們經由主要無線 AP 得到學術網路的 IP，再由
這台 AP 配送 IP 給各個裝置所需要的 IP 位置，而 IP 配送是由 192.168.1.20 到
192.168.1.244 之間的值，而其他 AP 是配送固定 IP 可方便我們進行管理 
b、定位功能 
目前居家照護定位系統尚未完成，所以目前可以得知裝置的 IP 位置、MAC Address、
狀態，未來此定位功能可以直接和定位引擎做互相的溝通，直接由定位引擎傳送資料到居
家照護定位系統上，透過居家照護定位系統之定位功能將定位引擎上的地圖傳送到居家照
護定位系統上，我們可以選擇我們要追蹤那一個裝置，如 IP 位置為 192.1681.143 這個裝
置，它就會顯示到下方表格的位置而其位置也會顯示所有的資料，我們以 IP 為
192.168.1.143 舉例來說明此定位功能的動作會如何。 
c、尋找裝置功能 
尋找裝置功能是讓使居家定位的管理者，可以詳細得知他所使用的設備目前的狀況，
只要管理者輸入設備的 IP 或者 MAC 就可以得此 IP 的裝置資料，透過尋找裝置可以讓我輕
易的管理設備，我們只要輸入 IP 就可以得知資料，如顯示為＂Find a device＂這表示有
找到此裝置的資料，否則會顯示“Find 0 device＂，這可以直接判定是否有找到裝置。 
d、顯示裝置功能 
在上一個尋找裝置功能只能透過管理者輸入 IP，而顯示裝置功能可以得知目前和定位
引擎有連線的裝置，會把所有跟定位引擎連線的裝置的資料全顯示出來，透過顯示裝置功
能會顯示所有與定位引擎連線的裝置資料。 
e、裝置追蹤功能 
裝置追蹤功能可以讓居家照護定位系統管理者得知目前追蹤的狀態，是否有正常在運
作當中，居家照護定位系統使用裝置追蹤功能，用來追蹤裝置目前執行的狀態，顯示目前
所有裝置和定引擎執行的情形以下將裝置狀態顯示出來。 
 
5.結果與討論（含結論與建議） 
 
在居家照護定位系統中我們可以和醫療院所、安養中心做結合，形成全方位的照顧受
照護者可以讓我們更人性化的照顧老，未來台灣邁向老人化社會所以我們需要更好的照顧
環境，使得老年人可以安心的養老也使得子女可以放心出外工作，不必擔心老人在家發生
意外沒人照護，所以我們需要建置一套全方位的居家照護系統。 
此研究可以透居家照護定位系統了解被照護者現在所在的地方，在其中我們使用多角
定位方式，這種方式可以使定位的精準度大幅的提升，在誤差值方面也大幅的降低，透過
這套居家照護定位系統使得照護者、被照護者、醫療院所做全方位的結合，彼此經由系統
都可以得到詳細的資料。 
我們所提的居家、醫療、安養三方環境架構需要一個長距離的無線傳輸，使得彼此的
資訊可以方便的傳送，當在居家、安養中心、醫療院所三個地方室內都使用 Wi-Fi 環境，
當彼此需要溝通時就可以透過 WiMAX 的無線技術傳送資訊，所以未來也需要在 Wi-Fi 和
WiMAX 二者間的結合做深入的探討，二者的結合會使三方環境溝通更方便。
