service (u-service) executing environment. A u-
service is a distributed application implemented via 
data or control interactions among a set of self-
contained system services (s-services). These s-
services will run on the computing devices deployed 
in our living space. However, for users, it is too 
complex to configure these computing devices for 
executing their u-service requests. Thus, a service 
automation system, which could coordinate and 
configure these devices according to users’ quality 
of service (QoS) requirements, device resources, and 
network bandwidths, is required for supporting this 
u-service executing environment. The service 
automation system aims to constructing a service-
oriented overlay network (SON) for executing each u-
service request. Unfortunately, the SON construction 
process is subject to the resource constraints and 
variances of devices deployed in a heterogeneous 
network. Moreover, the device properties also 
influence the resultant service quality. In this 
project, we formulate this optimization problem as 
the resource-constrained SON construction problem 
(RSCP) which is further transformed to the multi-
dimension multi-choice knapsack problem (MMKP), a 
well-known NP-complete problem. Moreover, we proposed 
an algorithm to solve RSCP with considering QoS 
requirements and resource constraints. The proposed 
algorithm adopts a utility-based approach to quantify 
the service quality for satisfying QoS requirements. 
Thus, the proposed algorithm achieves better resource 
utilization and user satisfaction. The simulation 
results confirm that the mentioned features and the 
advantages of the proposed approach. Moreover, in 
this project, we have developed the prototype of 
service automation system for the application of home 
appliance control. The system is implemented based on 
the android embedded system which integrates WiFi and 
Zigbee networks. Currently, the home appliances can 
be controlled manually or automatically with pre-
defined configurations. In the future, the proposed 
algorithm can be implemented on this system for 
realizing the u-service executing environment.

                                     日期：100 年 8 月 1 日 
一、參加會議經過 
此次參與第 3 屆運算智慧、通訊系統與網路國際會議(CICSyN)有兩個主要的目的。第一個目的是
發表被接受的論文，『An Adaptive Task Allocation Approach for MapReduce in a Heterogeneous Cloud』。透過口
頭說明所發表的方法，可以與現場的學者作進一步的討論，如此不但讓自己的研究成果能夠接受檢驗
並且可以得到一些寶貴的意見。第二個目的是希望能夠吸收各國學者在雲端運算上的研究成果以拓展
自己的視野。因此，透過與學者交流其研究成果，也許可以得到許多自己研究上的新方向。 
我在 CICSyN發表的論文是被安排在 7/27下午進行報告。在 7/24號抵達峇里島時已經是當地下午
3:00點，但是因為天色還很亮，因此在卸下李行後我們就馬上到，住宿的飯店附近，了解當地的文化。 
隔天(7/25號)早上在飯店用完早餐後，便在峇里島市區體驗當地的文化生活，要體驗當地的文化，
最直接的方式，就是食用當地的飲食，並且與當地人溝通聊天，晚上回到飯店，便查詢隔天要前往地
會議地點，並且了解搭乘何種交通工具，最為實用。 
7/26 號早上用完早餐後，變搭乘計程車前往會議地點。由於這是我第一次參加國際會議報告，所
以我在 26號第一天的會議，先熟悉每位作者的報告方式，並且了解報告者與主持人，觀眾的互動方式，
已為我第二天能夠順利地應對。 
到了 7/27號，我報告的日期，心情是特別的緊張，因為本身英文不是非常的流暢，並且這是第一

一、 參加會議經過 
IEEE資訊理論國際研討會(以下簡稱 IEEE ISIT)所涵蓋的範圍相當廣泛而且深入，是資訊相關領
域相當重要的會議之一，這次大會選在離台灣很遠的俄羅斯聖彼得堡舉行，也是本人第一次到美
國與韓國以外的國家參加學術會議。IEEE ISIT是一個相當大型的會議，其議程包含了 5場專題演
講、4個短期課程、與數百篇研究議題涵蓋了網路編碼、無線網路、量子資訊、壓縮感知、圖形理
論、通道容量、通訊系統、樣式辨識、MIMO 通訊、資料壓縮、安全編碼、多重存取、分散式儲
存、網路遊戲、密碼學、感知網路、網路控制、量化理論、網路排程等等資訊相關領域的技術論
文發表。由於大會同時進行的場次約有 8 個，所以並無法參與所有場次，只能挑選幾個與計畫相
關的主題，例如感知網路、分散式儲存等場次來參加。 
當然，遠道而來也不能忘記一探俄羅斯，因此我與同行的老師每天從下塌飯店都是利用聖彼得
堡的大眾交通工具，例如公車和地鐵，搭配步行到大會會場並在途中會選擇一些著名景點參觀。
另外，大會也很貼心的準備了幾個行程讓來自世界各地的與會學者參與。本人對俄羅斯的地鐵建
築相當感興趣，俄羅斯的地鐵中從平面搭乘手扶梯平均都要 2~3 分鐘往下才能抵達地鐵站，其深
度相當可觀。從當時的時空背景可知其道理，但以當時的工業技術來說，不得讓人佩服俄羅斯當
時的決心。最後，每次出國都會去參觀該國最著名的大學，這次到俄羅斯選擇了莫斯科州立大學
(Moscow State University)參觀，校內最雄偉的建置也是莫斯科七大建築之一，從山上遠眺就可以看
到其他六個莫斯科最雄偉的建築，讓人嘆為觀止。可惜的是，因為時間的關係，無法一覽幅員遼
闊的莫斯科大學，希望下次能有機會專程前往拜訪。
