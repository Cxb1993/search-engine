的旅館則是他雖然沒有距離海邊十分的近，價格也沒有特別的便宜，但他卻是所有飯店中具有較近距

再進一步介紹天際線查詢前我們必須先介紹優控 (dominant)的概念。我們說 a dominate b，代表 a 在其
所有的屬性(此處為價格以及距離)都 “不輸給” b，且致少存在一個屬性 “優於” b。觀查圖表 1，a 不
管在價格，以及距離上都 “不輸給” b，且其價格及距離都 “優於” b，故我們說 a dominates b。同理，
讀者可以發現 a 並不能 dominate i，因為 a 的價格 “輸給了” i。套用前述 dominate 的概念，我們可以發
現，圖表 1 中除了 a，i 及 k 外，對於其餘的旅館而言，都會存在至少一間旅館，其可以 dominate 它。
因此除了 a，i 及 k 都，其餘的旅館都不是使用者「相對較佳」的選擇。在本例中，skyline query 的回
傳結果便為 a，i 及 k。請注意，在前述文章中， “優於”的意義是由使用者所決定的，例如在旅館的例
子中，使用者希望旅館能價格「低」，但離海邊「近」。因此價格低的旅館或者離海邊近的旅館都是較
「優」的旅館。

找出「不被其它dominate」的資料。這些skylines的結果對使用者而言，都是他們較為青睞的資料。由

2. 為了避免查詢處理時，傳送大量的非 skyline 資料給查詢節點，SkyDigest提出一個修剪策略 
(pruning strategy) 來有效的過濾資料並加速 skyline 運算。這個修剪策略是將現有的資料處理
成為一個  digest，接著將這些資訊  mapping 到幾個  bits上。當每一個  server 再執行 
dominance test 時只需將資料對應到這個 digest 的資料上，就可以判斷該資料是否為 local 
skyline 。如此一來可以節省許多比較時所需要花費的計算成本。 
3. 每一個 server 利用 digest 執行後所找出來的答案僅是 local skyline 並非 global skyline。因
此我們還必須進一步的將這些資訊傳遞給其他的 server 做進一步的比較。我們的概念則是讓
每一個 server將自己內部的 local skyline 處理成一個 digest，接著轉成 bits 的格式傳送給其
他 server做進一步的比較。當所有的 server 都過濾完後所剩餘的資料回傳給使用者則一定為 
global skyline。這樣的查詢處理方式可以大量的減少資料的傳輸花費。同時其他的 server 在
收到這些回傳的資料時，可以以最少的計算成本來比較這些資料。

二、與會心得 
本次會議的正式開幕的儀式是從中午開始，儀式中主席報告了之後三天的
議程，以及一些日後會議的走向。專題報告後，我參加了 2 個 sessions，
並聽取來自個國的專家討論他們的心得。這兩個 sessions 分別是有關於
data mining 以及空間資料庫的領域。會議中報告的論文對我來說並不會十
分難懂，因此吸收了許多相關的知識。 
會議第二天，我參加了三個 sessions。分別是有關 Mining Moving Object，
Query processing 的議題。我們的報告是在會議的第一天。報告的過程還
算平和，不過由於個人有點緊張，所以報告的有點快速，雖然報告的時間
是 25 分鐘，不過我大概在 20 分鐘就結束了整個報告，這和之前的預演有
點出入，算是這次會議的一點遺憾。報告後有兩個人問了問題，而在出國
之前在和指導教授討論報時的沙盤推演都有預料到這些問題，所以回答起
來都不算困難。 
 
整體說來，此次會議的內容既充實，而且高水準。與會的專家都準備充
分，而且報告的內容也都切中主題。希望日後還有機會再參與同樣高水
準的國際會議，並且學習到更多的知識和經驗。
