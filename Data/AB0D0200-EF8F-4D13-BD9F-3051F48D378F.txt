二、系統介紹
目前廣播教學軟體[3]可分為 Server端老師端廣播，至指定 Client 端學生端的 IP 或 MAC
位址學生無法控制鍵盤和滑鼠，以達到廣播教學目的，另一種為由 Client 端學生端自行登
入到 Server 端老師端，但是將控制功能全部封鎖，讓學生只能觀看而不能做任何的控制，
[4]第 1 種方法有可能會制造廣播風暴，但是老師端的控制權力較重，讓學生較無自由可言，
對於原本己經躍躍欲試的學生或是原本己經學會的學生限制性較大，第 2 種方法由於是讓
學生自行登入，教師端親自行操作或播放操作檔案，讓學生自行登入觀看，由於大部份課
堂教學活動是所有的學生，但是學生之間是學習程度是有差異性的，學習電腦操作上會表
現的更明顯，此外學生必須要自制不去做非課堂上的事情，否則學習效果將會降低。

依據過去的經驗，廣播教學軟體大多使用在 Server 端先壓縮畫面，再透過網路傳送壓
縮檔，Client 端收到壓縮檔後，再解開畫面。且廣播軟體對畫面延遲的容忍度較靜態畫面低，
卻較動態畫面高，故壓縮格式的選用，對廣播的品質影響很大。

四、結論
Server 端希望能開發建置一個 Linux 平台中文介面廣播的免費自由軟體，降低建置網
路電腦教室的成本，加速 Linux 教學系統的推廣。Linux 平台開發困難度較高，但是沒有
授權費用問題。利用本計畫提出的方法將可以提供老師一套簡單易用的 Linux 廣播教學軟
體。本計畫原本構想是將網路的 VNC 原始碼加以修改成為可讓 Server 端把畫面廣播到各
使用者的畫面，但由於困難無法突破，故只做出設定 VNC 及 X windos 以達成廣播教學的
效果。我们將繼續努力希望未來可以做成自己的 Linux 廣播軟體。
