2一, 中文摘要
下世代的行動通訊系統將以整合行動異質網路（Mobile Heterogeneous Network）的方式呈現無所不
在的行動網際網路，對於現有網際網路上一些相當成功和成熟的多媒體服務也將可能延伸應用在行動寬頻網
路環境中。然而，由於行動和無線網路本身的特性與限制的影響，對於目前在網際網路中運行的一些多媒體
檔案傳輸以及串流服務技術和應用等，若直接套用在無線網路中使用時將會面臨諸多的挑戰與效能問題。有
鑒於此，本研究計劃將以所轄之四個研究子計畫，針對行動異質網路的多媒體串流服務進行適性傳輸之研究。
以兩年為期，此計劃將在以 Android 及 OSGi 為基礎的嵌入式開放服務串流平台開發(研究子計畫一,註:可惜
被砍掉,計畫未獲通過)、新型態串流服務的創建與應用(研究子計畫二)、多媒體串流服務之網路監控與管理
方法(研究子計畫三)以及行動異質性網路針對多媒體串流服務的適應性傳輸技術(研究子計畫四)上進行探
索，並以車載網路以及行動同儕網路為實例，建構技術發展與驗證的基礎。期藉由此計畫之執行，而能將下
個世代行動通訊的隨處運算環境推向更為進步而臻至成熟的境地。
關鍵字：多媒體串流服務、行動異質網路、QoS 服務品質
4服務的管理等各方面問題，均亟需提出新的解法或不同的替代方式以克服之。
我們提出「行動異質網路串流服務之傳輸平台與應用」之總計畫研究，並且具體說明兩個計畫性發展之實
施例（實施例一：車載網路多媒體資訊應用，實施例二：行動P2P串流內容傳播）。整體計畫的發展理念是在本
身即是“開放”的行動網際網路環境當中，建構開放式的行動異質網路串流服務之傳輸平台，支援各式各樣新
穎的串流技術與服務的快速導入，設計串流服務之服務管理機制，進行串流服務品質控制與回饋調節，發展高
效能與適應性的無線傳輸技術等等之研究，更將進一步垂直整合研發之技術與系統，提出參考架構並且展示雛
形。
綜合上述之，本計畫的架構依各研究主題之關聯性區分數個部分，由四個子計畫主持人及共同主持人依其
專長承接執行。各子計畫於此計畫所建立之平台上進行研發與系統分析、整合和測試，除總計畫外，其下包括
四個子項計畫：
總計畫：行動異質網路串流服務之傳輸平台與應用
子計畫一：行動異質網路下嵌入式開放串流服務平台之研發(註:可惜被砍掉,計畫未獲通過)
子計畫二：行動異質網路下新型態串流服務和應用之研發
子計畫三：行動異質網路下串流服務之服務管理機制
子計畫四：行動異質網路下串流服務品質與適應性傳輸技術之研究
三, 研究方法與成果
本計畫以行動異質網路串流服務之傳輸平台與應用為研發主軸，在下世代異質整合的行動通訊網路環境建
構支援新型態行動串流服務開發的環境，發展一組整合Google Android和OSGi（Open Source Gateway Initiative,
OSGi)平台之嵌入式開放串流服務平台，將分別完成車載與居家網路環境中不同類型的嵌入式平台，在這些平台
之上發展各式新串流服務和應用以及新服務快速導入技術，並提出適用在行動網際網路之多媒體串流服務訊息
交換協定，並且研發行動串流服務管理和品質監控機制，根據設計之網路串流傳輸服務品質（QoS）和使用者介
面上串流服務呈現（QoE）等參數化之量測數據結果，結合發展之行動異質網路資源管理及高效能和可調節的資
料傳輸技術，提出對應的串流服務品質回饋補償機制，透過平台與應用軟體之間可程式化的系統介面之設計，
支援動態調節串流服務的資源利用以及高品質的串流服務的保證，並據以最佳化整體系統效能。
本計畫也另將致力於行動串流服務的展示平台，促使各子計畫能整合彼此的研發成果並且具體落實。除了
計畫已規劃設計之兩個主要的實施例之外（以車載無線串流資訊服務和行動P2P串流服務），也會對行動異質網
路環境中未來可能出現的各式新型態串流服務與使用情境，例如數位家庭應用中的居家安全串流監控、遠端裝
置控制以及影音檔案傳輸，研究應用結合的可能，擴大本計畫成果的應用範疇和成效。
於本研究計畫中，總計畫將帶領各團隊研究並規劃行動異質網路串流服務之傳輸平台與應用之架構（如圖
一所示），協調各子計畫於測試平台上進行功能模組之初步整合與測試。並且也透過週期性會議，提供各計畫
參與人員門了解各別的進度與所遇到的技術瓶頸，並集思廣益提供解決的方法，並且設置網路論壇與檔案伺服
器將相關的參考文獻、技術報告以及系統手冊等，提供各子計畫參考，並分享各自的研究成果。
6據以提出一套無縫式多媒體串流傳輸與播放機制，這機制將可以幫助已移動的家庭網路裝置快速重新建立連線、
恢復影音串流傳輸並且維持多媒體播放不中斷。此機制之設計，無須修改實施裝置的系統底層與網路功能，可
直接以軟體應用程式的方式來達成，可視為一低成本的解決方法。同時，本研究提出以 OSGi 服務平台發展之實
作雛型，並佐以實驗量測結果來說明該機制之功效。
本計畫成果，完成以上兩套機制，以行動異質網路為網路應用環境，介紹許多以使用者為主的新型態多媒體
內容傳輸服務（訊息溝通、定位服務、檔案交換以及串流傳輸等），我們認為在開放式的IP通訊環境中，以人為
本、內容為主、技術應用為導向的應用研究，將逐漸成為行動多媒體服務的主流發展。
子計畫『行動異質網路下串流服務之服務管理機制』提出一個以串流服務等級協議（Service Level Agreement；
SLA）為基礎可自動補償之服務管理系統。SLA是自然語言所撰寫的文件，並經由人工的方式來產生與監控，可
應用於網際服務的品質控管上；補償是藉由SLA協議擬定違反參數與參數門檻，服務管理系統一旦發現即將違反
時，可事前或事後對於違反參數進行改善的措施，例如：當串流服務封包遺失率（(Packet Lose Rate)提高時，
改進的策略可以是提升頻寬或更改繞送路徑等多方面來著手。本子計畫之SLA協議參數將包括服務品質（Quality
of Service；QoS）與經驗品質（Quality of Experience；QoE），所謂的QoE是一種用於量測用戶對於使用服務
後的滿意度指標，例如：對於頻道切換的速度是否滿意。當串流服務開始時，本子計畫之管理系統會監控該串
流服務是否滿足SLA所簽署的QoS/QoE參數，一旦違反則立刻根據現有網路資源擬定補償決策。除此之外，本系
統可以對串流服務做預先補償的動作，使得串流服務的品質能夠符合協議規定的範圍，進而能夠讓用戶持續地
使用高品質的串流服務，同時服務提供者能在符合SLA協議情況下提升較多用戶數，以獲取最大的利益。
Streaming Management Server 是本子計畫的主要管理系統，功能是管理串流服務的 SLA 協議、擬訂補償策
略、部署 QoS/QoE 監控代理人（QoS/QoE Detection Agent）於行動異質網路中，代理人的功用是監控串流服務
的品質，確保符合 SLA 所規定之參數。所涉及的 QoS/QoE 指標參數（Metrics）可分類為網路層及應用層指摽參
數，例如：封包遺失率是屬於網路層級的量測項目。頻道切換時間與編碼解碼造成的延遲屬於應用層級。QoS
管理系統量測指標（Metrics）包括有：封包遺失率（Loss Rate)、資料位元失誤率 (Bit Error Rate)、封包
延遲時間 (Delay)、抖動率 (Jitter)、反應時間 （Response Time）、處理時間 (Access Time)、可靠率
(Reliablitiy Rate)、可用性 (Availability)、吞吐量 (Throughput) 等。QoE 常見的量測指標 (Metrics) 包
括有：頻道切換時間 (Channel Zapping Time)、STB 啟動時間 (STB Startup Time)、影音內容的清晰度和流暢
程度 (Quality of Video)、影音同步率 (Audio-to-Video Synchronization Delay)、VOD 節目控制指令的反應
時間 (Control Command Response Time)、編碼類型 (Encoding/Decoding Type)、編碼速度 (Encoding/Decoding
Delay Time)、認證失敗 (Authentication Errors)等。
本子計畫兩個年度之研究議題(1) 將研究監控串流服務品質 (QoS) 之網路架構，監控項目包括：網路延遲
(Network Delay)、抖動率 (Jitter)、封包遺失率 (Packet Loss Rate) 和吞吐量 (Throughput) 等，提供一
個即時且可靠的網路監控機制，並在這個基礎上透過議題 (4) 建置可行且有效的自動補償機制。議題 (2) 將
研究監控串流經驗品質 (QoE) 之網路架構，監控項目包括：整體影音品質 (Video Quality)、頻道切換時間
(Channel Zapping Time)、控制指令下達反應時間 (VOD Control Time Responsiveness) 等，提供一個即時可
靠的串流經驗品質監控機制，並在此監測基礎上透過議題 (4) 建置可行且有效的自動補償機制。議題 (3) 將
為用戶與串流服務提供者間建立一服務等級協議 (SLA) 的管道，並利用 Web Service 2.0 的技術妥善管理 SLA，
另將雙方協議的 SLA 視為是否違反串流服務 QoS 與 QoE 的依據。議題 (4) 將結合議題 (1、2、3) 研擬一套適
用於寬頻網路下之自動補償串流服務的管理機制，讓用戶與串流服務提供者達到雙方互惠的管理策略，同時以
此平台做為第二年度承續研發時之基礎。議題 (5) 整合總、子計畫的相關研究。

模擬機制與流程
[機制一]是目前普遍使用的機制，會以訊號強度的品質為考量，當使用者移動時收到目前連線的基地台的
訊號強度的品質漸漸弱到低於所設的門檻值時，代表訊號即將要斷線，此時接收到另一基地台的訊號強度品質
足夠好時，那麼使用者就會換手過去;若附近沒有同質性的基地台存在，就會換手到訊號強度品質夠好的異質性
基地台，避免連線的訊號會中斷的情況發生。
[機制一]的方式僅以訊號強度的品質做為決策換手的依據，且未考量到不同使用者的需求，可能會造成換
手後覺得服務品質變差，連線變慢等，且也有可能換手到的網路資源不足、如頻寬不足造成換手失敗的情況發
生，這是使用者最不願見到情況發生。所以本計劃所提的[機制二]的部份，會考量到不同使用者的traffic type，
並計算網路的資源，讓使用者滿足需求去決定該換手到那個適當的網路以維持服務質品質，另一方面也可提昇
整個網路的使用率且增加 throughput，創造網路服務提供者營收，達到使用者和提供者雙贏。

技術說明
（中文）
本發明利用 Gateway 結合 UPnP 及 P2P（以 BitTorrent 為應
用之實施例，簡稱 BT）之設計，讓我們的家庭(或企業)網路間
的網路裝置（例：具 UPnP 能力的家電、多媒體播放器、PDA
手機）能以 UPnP 協定相互發現並溝通，而裝置能透過 Gateway
利用 BT 向外部的 P2P 網路下載或分享影音內容。
UPnP 為一套網路協定，本著隨插即用的概念，在裝置連
上家庭(企業)網路後，便能和各種裝置相互發現及溝通，不需
特別進行繁瑣的設定，UPnP 支援零配置，自動取得 IP 及自動
廣播裝置本身存在，以便開始進行溝通，然而 UPnP 的影音分
享僅能在同一區域網路(LAN)下進行，我們提出的技術(以下，
稱為 UP2P)將 UPnP 和 BT 做結合，使得 UPnP 的家庭網路被視
為 P2P 上的一個節點，統一由 Gateway 向外進行要求，讓家庭
網路裡的 UPnP 裝置也能得到外部 P2P 網路上的影音分享。
另外，本技術利用 OSGi 平台將 UPnP 裝置及 Gateway 實
作成 OSGi 的 Bundle，再將 Bundle 掛載至 OSGi 容器內，這便
是利用 OSGi 平台的模組化概念，OSGi 可以支援 Bundle 無需
重新開機的更新、安裝及版安裝，集合 UPnP、P2P 及 OSGi 便
是本技術的最大特色。
