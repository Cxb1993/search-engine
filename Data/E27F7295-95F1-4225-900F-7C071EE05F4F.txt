研究摘要： 
為了建立企業形象及競爭力，企業利用網路的便利性來進行企業內部
組織運作的流程，並且讓內部人員能即時存取公司資源，以爭取作業時
效，因此內部人員存取資料的安全性就相當的重要。若發生內部人員使用
錯誤、使用不當或將竄改檔案資料、否認先前行為，造成企業的損失、引
起糾紛、甚至危害信譽與生存，因此，隨著電子商務的發展，身份辨識及
不可否認等安全機制也愈來愈受到企業的重視。 
本計畫為企業建置一個架構於公開金鑰基礎建設（ Public-Key 
Infrastructure, PKI）之下使用數位簽章技術的稽核警示系統，針對企業內
部各應用系統中必須加以監控的作業項目進行稽核記錄及管理，達到作業
項目之身分確認、規範其存取控制權限及不可否認性的功效，維護通訊內
容或儲存資料之機密性與真確性，並嘗試在可能發生安控問題的階段即時
提出警示訊息，降低人為不當使用、竄改、破壞、冒名行使或事後否認之
風險，將安全及失效事件所造成的損害降至最低，並監督這類事故並從中
學習，確保使用者了解資訊安全的威脅與問題，達到電子商務交易過程中
身份確認（Authentication）、權限控制( Authorization )、資料完整性
（Integrity）、資料隱密性（Private）、不可否認性（Non-repudiation）的
功能，讓商業交易或資訊交換的過程都能安心使用，進而創造客戶更大的
商機與效益。

圖二、系統功能模組 
 
‧ 應用系統端 API 介面 
應用系統端 API 介面為稽核警示系統提供給客戶端之企業應用系統
的程式介面，作為企業應用系統傳送稽核封包與警示控管封包的橋樑。
另一方面，警示控管 API 提供讓客戶端企業應用系統產生 XML 格式的
警示控管封包，並以使用者的自然人憑證 IC 卡對此警示控管封包進行
數位簽章，然後將簽章後的控管封包傳送到稽核警示系統伺服器以進行
後續處理的功能。 
 
‧ 稽核封包解析模組 
此模組將接收來自客戶端企業應用系統之稽核紀錄封包，並依照稽
核警示機制訂定之 XML 格式將稽核封包內的各項資料包含數位簽章等
一一解析讀出，再將讀出的資料傳送給稽核簽章檢驗模組。
