研究摘要： 
 
計畫係以開發數位觀光導覽護照為主，提供智慧型手機之使用者可藉由此數位護照，
進行景點之導覽、各景物或歷史解說、旅遊交通資訊查詢、優惠活動推廣等各活動使用。
其示範案例選擇「2010 臺北國際花卉博覽會」為數位觀光護照的示範景點，並以 Android
手機為開發平台，未來可進一步推廣至各景點單位。 
與業者之合作配合，完成 Android 手機上的數位觀光導覽護照開發，包含有各景點之
介紹資訊，並使用QR Code來定義GPS所定位不到的座標，讓使用者可以在任何有QR Code
的地方，快速瞭解自己的所在，以及相關的介紹。其導覽護照也包含有語音導覽的功能，
以方便更多的使用族群。合作業者期利用此基礎發展自主的應用軟體，並擴展應用於 iPhone
及 iPad 等其他平台，以降低開發成本，提高獲利空間。 
 
人才培育成果說明： 
 
本研究使用 Android 平台、Google API 及 QRCode 等應用，提供學生可學習相關的技
術，並完成數位觀光導覽護照的開發，掌握基本的開發能力。期間配合業者的技術及開發
經驗，使得學生有機會早一點接觸軟體開發業界的種種狀況，不只是學習各種技術能力，
更可以了解在業界工作時所需具備的其他能力。 
 
技術研發成果說明： 
 
本研究完成「數位觀光導覽護照」的開發，並以臺北國際花卉博覽會為目標
景點進行測試，其軟體的功能包含： 
1. 個人圖資 
以 2010 臺北國際花卉博覽會平面圖為基礎，將其套疊到 Google Earth 上，以取
得各展館間之相對座標關係。在座標資料儲存上，除了保留 WGS84 以供定位
參考外，亦轉換成 TWD97 座標，以做為後續計算設施間距離之參考。 
2. 語音資訊 
在語音資訊建立上，包括了展館介紹資訊及設施間相對位置資訊之語音，說明
如下： 
(1) 展館與設施資訊語音的設計 
以提供展館相對位置與設施之介紹為主，包括服務功能、操作方式及注意事
項等。 
(2) 方向語音的設計 
方向提示語音的設計，主要是用於提示使用者，其他設施與目前所在設施方
位之關係，規劃每隔 45 度角設定一個方向，可提供八個方向之定位，包括前、後、
左、右、右前方、右後方、左後方、左前方。搭配所設定方向錄製提示語音。除
八方向定位外，未來可視需求增加不同角度之定位提示。 
(3) 距離語音的設計 
距離提示語音之設計，主要用於提示使用者，其他設施與展館和目前所在位
容。 
(4) 地圖顯示 
當完成 QRCode 定位後，亦可結合地圖顯示所處位置在場站空間中的絕對位
置，在手機畫面中紅色汽球會標示所處之位置，亦可在手機中觸控移動地圖查看
周邊的設施位置。 
(5) 展館資訊 
數位導覽護照使用過程中，使用者若想知道所在位置之展館資訊時，可利用
此功能隨時查看，另外亦可透過 QRCode 辨識功能，盡快取得所需之資訊。 
在建立景點資料庫與週邊資料庫等設施資料庫前，須先就既有設施或指標進
行分析瞭解，並建立相關屬性資料，及屬性間之關聯圖，後續也才能依此架構建
置資料庫之內容。個人導引裝置的開發，則是以智慧型手機內建之設備（如電子
羅盤、相機、GPS 定位、無線上網等）、個人圖資、QRCode 辨識等功能，結合已
建立的設施資料庫，便可提供使用者更為便利的操作環境，以發揮「數位導覽護
照」之實務功效。 
 
技術特點說明： 
本研究係發展數位觀光導覽護照，主要是運用在 Android 行動平台之上，包含幾個技
術： 
 
1. Google API 
Google 在 Android 上提供 Google API，是 Android SDK 開發環境的延伸模組，
可以輕鬆地存取 Google 的服務與資料，其中一項重要功能就是 Google map 的
使用，可以自由存取 Google 所提供的地圖資訊。因此計畫中將選擇 Google API
來開發 Google map 之相關應用，並結合資料庫伺服器，將各類資料標誌就
Google map 之上。使用 Google map 之前需要先於 Google Code 網站中取得
Google MAPS API Key，加入專案之中，再利用 MapVew 的類別來開發 Google 
Map 的應用視窗。 
2. 座標轉換處理 
因為 Google Map 採用的 WGS 84 座標系統，而地理圖資常需要使用二度分帶橫
向麥卡脫投影座標系統，因此需要進行座標轉換的處理，才可能將 Google Map
之地理資訊與其他模組進行資料交換。 
3. QRCode 
計畫中分別進行QRcode產生器及辨識器兩個軟體功能開發，使用者或店家可配
合產生QRcode，後續可利用手機中的辨識器辨識QRcode，再進一步取得各項資
訊 。 QRcode 產 生 器 可 於 http://www.quickmark.com.tw/ 或
http://www.morovia.com/找到線上免費的版本，但配合資訊整合及取用的需求，
將自行撰寫產生器，並加入自有的辨識碼。利用Android SDK配合手機之相機功
能，撰寫QRcode解碼器，整合入數位觀光導覽系統之中。
