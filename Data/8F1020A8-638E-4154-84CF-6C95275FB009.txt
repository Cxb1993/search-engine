In the last decade, many web site metrics have been 
proposed to measure navigability with different ways. 
With user surveys, a web＇s navigability is evaluated 
by a set of criteria from a sample of the web site＇s 
users. There are many problems in this method, such 
as time consuming, geographical space, and 
representative of the sample. Another is the way with 
usage analysis in which the usage data such as log 
data are investigated. Theoretically, the user 
behavior can be traced by examining the usage data. 
In fact, it is difficult to have complete trace of 
user behavior. The structures of web sites are often 
used to measure the web navigability. It is not 
enough to characterize the web navigability by 
examining the hyperlink structure of a web site 
statically. 
This project was planned to be a 2-year research. But 
the second year plan was not approved. In this

the proposed model, the static structure of a web

and the dynamic user surfing behaviors are considered 
as the events or the emitted symbols in this model. 
Using the log of a web site, navigability of each

real web data. The result shows the proposed model 
has good accuracy. We not only proposed a model for

中文摘要 
在資訊社會中，資訊科技的快速發展與網際網路的廣泛使用，網站已取代傳統的資訊提
供平台，成為目前重要的平台。一個網站的成功與否決定於這個網站的可使用性的高低，如
果網站使用者無法有效率的在網站中找到他想要的資訊，他將會離開這個網站而降低網站的
可使用性，因此一個網站導覽度的高低決定了這個網站的可使用性也決定了網站的成敗。所

同特質，而且分佈在不同的地域中，如何找到他們得到解答是非常困難的。藉由資訊科技的
利用，要知道這個答案，我們可以從網站中的架構、內容、及使用記錄中獲得相關資訊。因

因此一個網站的可用性(usability)為ㄧ個網站成功的關鍵因素之一。如果一個使用者不能很有效地
在網站中找到他所需之資訊，他一定會很快地離開這個網站。如何很快地指引導覽使用者找到他所需
之資訊成為影響這個網站可用性之重要因素。所以一個網站的導覽度(web navigability)是影響網站
成功的重要特性之ㄧ。 
如何建立一個網站能符合它的使用者期待，是這個網站成功與否的重要議題。但是一個網站的使
用者，可能有許多不同類型且異質性高，而且分佈之地域性亦廣，要如何得知他們的真正之需求，再
實際面上是一個難題。在 Lappas[2] 的研究中提出，藉由現今資訊科技的發展，可藉由一個網站的結
構、內容、及使用者之瀏覽資料中擷取相關資訊，得到想要之答案。因此，建立一套適切的網站評估
方法(web metric)去分析了解、管理經營、及改善控制一個網站，對於這個網站的品質與成功與是迫
切需要的。在最近的十年中，已有許多研究者提出不同論點之網站導覽度評估方法，有些是以靜態網
站架構複雜度為出發點，計算網站中網頁(page)的個數、超連結(hyperlink)的個數、或網頁間之最
短路徑等；但這些方法，純以網站架構複雜度作為評估網站導覽度之依據，除了無法充分表現出網站
結構中網頁與超連結間之關係，更忽略了瀏覽者的期待之影響因素，常常誤導網站設計者設計ㄧ個多
而雜亂之網站架構，這也是目前許多網站的現況，讓使用者迷失在網站中，無法有效率的找到想要之
資訊。為了要瞭解網站中瀏覽者的期待，也有許多研究提出許多以網站使用者為出發點之方法，這些
方法大部分是以使用者調查為主要手段，但這些方法中也存在一些問題，例如全面之使用者調查之所
需時間恐怕很長，且所有使用者分佈空間很廣，因此全面性調查是不可行的，那如何取樣，能找出具
代表性之使用者又變成另ㄧ難題。還有ㄧ種方法利用資訊科技協助分析使用者瀏覽行為，他們利用網
站中使用者瀏覽歷程資料，使用不同之工具及方法，提出不同之評估模式；這一類方法，應該要比使
用者調查方法具較高之可行性及可信性，但只考慮使用者行為而忽略網站架構之複雜度，似乎又有顧
此薄彼，無法兼顧網站之影響因素之缺點。 
有鑑於此，建立一套能兼顧網站架構及使用者瀏覽行為的評估模式為本研究之主要目的，另外，
以前之評估方法，僅就網站做評估，並未對評估不理想之網站做修正建議，在本研究中，除了建立評
估方法外，並將網站依評估結果分類，再依分類結果，給予修正之建議，使該網站更符合使用者之期
待，提高網站之導覽性。在本研究中我們提出了一個以資訊理論為基礎的動態評估網站導覽度的方

件之機率相等時，即每個連結的機率為 1/r 時。事實上，當一個網頁的所有超連結被使用者平均的瀏
覽，表示這個網頁設計的非常符合使用者之期許，沒有不必要之超連結存在，是一個完美的網頁設計。 
有了網頁存在的最大熵值與實際的網頁熵值，我們提出一種評估方法利用它們之間的比例如公式
(10)所示，來分辨網頁瀏覽度之好壞依據，當熵值比例越高時代表使用者越平均的瀏覽網頁，顯示網
頁提供的資訊符合使用者需求，熵值比例越低時代表使用者瀏覽網頁越不平均，甚至是使用者有沒有
瀏覽到某些網頁的狀況，網頁提供的資訊較不符合使用者需求。根據以上之推論，我們可以利用熵植
比例做為是否修正網站架構之依據。

4.3 堆疊基礎的馬可夫評估分類資訊系統 
為了方便使用本研究之評估分類模式，我們利用 Java 撰寫，整合開發環境 NetBeans，並運用 Java
開放碼源的網頁爬蟲 crawler4j 來爬取網站的靜態結構，並將網站日誌檔(Log)利用日誌檔分析程式
WebLog Expert 處理成一筆筆的使用者瀏覽路徑，最後系統將結構與使用者瀏覽路徑結合，依照我們
的分類模式，可將網站內的網頁評估分類。本模式系統可分為四個步驟，分別為為爬蟲資料、網頁結
構、使用者瀏覽路徑和馬可夫模式分析。 
在爬蟲資料的部分，crawler4j 提供了 Proxy 代理伺服器的功能，由於網頁爬蟲抓取網頁架構時
會對網站伺服器送出大量的請求，在這樣的情況下如果從外部網路用網頁爬蟲擷取網站結構，網站伺
服器有可能會判定爬蟲程式是在攻擊網站伺服器，因而擋下這些請求。所以網站管理者在外部網路可
利用同網站伺服器內部網路的 Proxy 伺服器來避免這個問題。當網站爬蟲結束後，系統會將爬蟲資料
轉換成以相鄰矩陣表示的結構資料，在第二步驟的網頁結構中，有 4 種不同的功能來呈現結構資料。
網址清單表為所有網站編號與網址的對照表，網頁結構陣列則是以相鄰矩陣的形式展現網頁的結構，
結構分析結果則提供網頁結構數據方面的分析資料，個別網頁結構提供個別網頁擁有連結的清單表並
提供瀏覽器功能來檢視所選的網頁。系統操作介面、爬蟲資料處理過程、及爬蟲資料結果如圖 6、7、
8 所示。

就第一年的成果而言，本研究計畫中所提出之「堆疊基礎的馬可夫網站瀏覽度評估與分類
模式」，在學術研究上對於網站中的網頁，提供了一種新的評估的方法，這種評估的方法
涵蓋了大部分網站瀏覽者的瀏覽行為；並利用瀏覽者的瀏覽記錄分析，可將網站中的網頁
分類，提供網站管理者作為管理及改善網站結構的參考。另外在實際應用層面上，網站管
理者可使用本模式系統，分析其網站結構與使用者瀏覽習慣的符合度，並適時修正網站結
構，避免網站結構不斷成長而不符使用者期許，以提高使用者的滿意度。 
綜合分析本研究計畫，已完成預定之工作，並完成下列之成果： 
1. 提出一個「堆疊基礎的馬可夫網站瀏覽度評估與分類模式」，利用堆疊裝置取代傳統之
記憶裝置，以符合瀏覽者的瀏覽行為。 
2. 蒐集及分析網站相關資料，完成瀏覽路徑產生模組。 
3. 模擬各類網站瀏覽記錄，整理及分析網頁分類方式。 
4. 根據前面提出之評估與分類模式及研究成果，建置成為一套網頁評估與分類資訊系統。
5. 並利用多個網站及前述之系統，評估各網頁之瀏覽度，並進一步將網頁分類，作為驗
證本系統正確性之依據。結果呈現本研究成果具有不錯之準確度。 
6. 因本計畫之執行，訓練了三個學生在此領域中，學習到資訊理論、評估理論、網頁評
估及分類等相關知識，並熟悉相關工具。 
7. 已發表一篇國際學術研討會論文，另外正將該篇論文修正為國際期刊論文中。
