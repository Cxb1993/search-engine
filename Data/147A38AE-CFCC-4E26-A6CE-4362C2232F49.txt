Construct an interactive e-portfolio system for cancer 
patient healthcare management 
 
摘要 
    本研究內容以台北某醫學中心癌病中心為例，希望將數位學習與知識管
理的概念用於癌症病人之護理指導指引。本研究探討如何以系統化的方式，
針對病人診治狀況及階段(例如不同種類癌症與治療方式，放射治療前準備內
容、治療過程注意事項、治療後衛教等)，給與恰當的衛教內容。本研究將各
類癌症衛教內容儲存至學習內容管理系統中，便於後續導入教學平台(Moodle)
病人特定的課程當中。本研究並以一入口網站，連結 Moodle教學平台、學習
內容管理系統、以及放射治療管理系統，便於使用者整合放療衛教所有相關
資訊與系統，以此提供自動化且系統化的病人衛教；透過單一入口網站進入
系統後，醫護人員可使用 Moodle教學平台完善的課程與內容控管機制，透過
介面選取自動開課，產生病人個人專屬課程，讓病患或家屬得以使用其專屬
課程資源，以確保其隱私。從放射治療管理系統，可得知目前病人治療狀況
與階段，依此資訊從內容管理系統當中選擇恰當的護理衛教資訊，再將選擇
之內容封裝標準化，並轉入教學平台病人專屬的課程當中，以此建立病人個
人化之護理指導課程內容。另透過本研究擴充開發之系統，可讓醫護人員配
合衛教內容設計問卷表單，方便病人於平台當中填寫，以確認其對衛教內容
 護人員能透過統一之網路平台與病人或家屬互動，建立連貫一致的健康照護
指示，並藉由學習平台互動與追蹤功能確保病人了解並遵循處方指示。設計
之系統朝向以病人為中心的健康服務平台發展，以此平台補現行癌症診治不
足之處。 
    此數位學習平台功能與服務包含了，針對不同治療方式與階段的病人發
佈醫學護理指導數位內容，方便病患了解該類癌症的病因與治療注意事項，
提供醫護人員與病人共同使用之健康服務平台。未來亦可以此整合不同科別
之數位醫囑，建立單一之聯繫互動窗口，避免在一系列的治療中，病人與多
個部門聯繫之負擔，以及溝通不良之狀況發生，達成無縫之癌症醫療照護。
另外，在癌症病人術後或整個治療流程結束後的追蹤，像是給定病人營養、
運動、居家保健上的建議，可透過健康照護平台進行這些建議之互動與追蹤，
並設計評估表單紀錄病人施行與生理反應狀況。以此導引癌症病人術後預防
保健與提升生活品質。透過平台上之表單紀錄，並可將癌症病人居家之照護，
帶入實證醫學的範疇。本研究所建構之系統及方法未來不只可擴充應用於癌
症其他治療之線上護理指導，亦有可能應用於其它臨床治療領域，未來很有
擴充推行潛力。

圖一 系統架構圖 
 
    上述架構圖中使用到的系統及規範如下概略說明： 
1. 入口網站：負責架構圖中包含的子系統之權限管控，採用單一登入(SSO)，
當使用者從入口網站登入後，可訪問所有系統，不需再重複登入。 
2. Moodle數位學習平台：提供病患護理指導課程的學習平台，可記錄癌症治
療歷程並增進醫病間互動。 
3. 放射治療資訊系統(ARIATM)：病患放射治療資訊來源，包含病患基本資料、
治療階段資訊及相關記錄，內容會透過 Web RT系統呈現。 
4. 學習內容管理系統(LCMS)：提供護理指導課程教材製作及存放，課程管理
者透過課程內容列表，選取內容後將課程 URL包裝為 SCORM規格的壓縮檔，
再匯入 Moodle病患專屬課程中。 
  
 
圖二 學習內容管理系統(LCMS)功能示意圖 
 
    學習內容管理系統功能示意圖如圖二表示，課程管理者使用編輯工具製
作數位教材後，加入 metadata做教材分類及記錄作者資訊，再依照數位學習
內容標準規格(SCORM)封裝，上傳到學習內容管理系統(LCMS)，學習內容管理
系統會依照學習物件後設資料(Learning Object Metadata, LOM)標準，用以
描述學習物件資料與管理學習資源，以達到學習資源互換的目的。我們在學
習內容管理系統中，運用 LOM提供的學習物件後設資料標準，描述護理指導
課程內容，以方便管理者透過課程管理介面搜尋需要的教材內容。癌症病患
治療流程中，各個階段都有不同的護理指導重點，我們預計將課程內容分為
四個時期作管理，分別是轉介至癌病中心、治療前、治療中及治療後，依照
TW LOM規範的欄位名稱輸入相關訊息，以作為系統查詢教材類型的依據。

 4  取得衛教內容施行對象及時程 
    護理指導數位課程內容從學習內容管理系統中取得，因為護理指導給予
的內容是依照病患癌症治療病程，隨時由課程管理者新增，因此要配合病患
治療的時程提供相關護理指導資訊，病患治療流程資訊我們由癌病中心放射
治療資訊系統(ARIATM)中取得。癌病中心放射治療資訊系統(ARIATM)為洽泰公
司代理，系統功能紀錄放射治療過程中所有的專業資訊，除了影像資料外，
包含了病患基本資料、檢查與診斷結果、檢查排程、放射治療紀錄等等。由
於建置病患專屬衛教課程的過程中，需要使用到病患相關治療資訊，我們透
過入口網站下的子系統 Web RT，將 ARIATM中資訊以 SQL server資料庫轉存，
需要應用到 ARIATM內的資訊，則透過 Web RT系統提供的功能頁面取得，例如
新增病患帳號到入口網站及 Moodle時需選擇的病患清單，便是透過 Web RT
提供。癌病中心病患多半轉介自其它科別，病患到癌病中心報到後，經過醫
師看診及進行治療流程，行政人員將病患治療資訊輸入 ARIATM，課程管理者由
入口網站登入後，從 Web RT提供的病患清單，選擇新增帳號對象，透過入口
網站提供的 Web services將相關資訊傳到入口網站作權限管控，再透過
Moodle web services將病患帳號新增到 Moodle，並建立病患專屬課程。 
    除了由 Web RT提供病患清單，新增帳號到入口網站及 Moodle，亦可查詢
到癌症病患從報到開始提供的相關資訊，包括病患姓名、帳號、年齡、性別、
主治醫師、報到日期、連絡方式、負責醫護人員；接著進入診斷階段，系統
記錄病患診斷及治療方式，及是否製作固定模具。造影相關資訊包括安排造
 隊成員及病患可使用專屬課程內容。病患可透過平台內提供之互動功能，了
解疾病相關資訊及健康諮詢。醫護人員也可透過系統中紀錄資訊，了解病患
使用平台狀況，亦可做為日後統計分析及加強衛教之依據。 
 
6  系統應用流程及整合機制介紹 
    以上章節介紹了各個子系統功用及癌症護理指導課程 以病患癌症治療

觸的醫院部門，綠色區塊為醫院資訊系統，橘色區塊則為本研究規劃應用的
各個子系統；橘色路徑代表病患看診流程，病患資訊在系統間的傳遞以紫色
路徑表示。圖四預設的情境為，病患在出現頭痛症狀後，先到門診求治，經
過神經內科醫師做相關檢查後懷疑是腦瘤，安排住院做更進一步檢查，確定
病患診斷腦癌，並決定採取放射線治療，因此將病患轉介到癌病治療中心。
病患帶著黃色治療紀錄單到癌病中心登記處報到，醫師看診後安排病患做電
腦斷層攝影及安排一連串的治療過程，以上是步驟 1到 7。

    護理指導課程內容存放及封裝都在學習內容管理系統(LCMS)中進行，課

用護理指導課程。病患透過護理指導課程可預先了解往後治療過程及相關注
意事項，對課程內容有任何疑問也能詢問醫護人員，加強相關衛教。 
 
研究成果 
1   產生的標準化課程內容 
    所有護理指導課程的教材存放及製作，皆在學習內容管理系統中進行，
管理者介面以樹狀結構呈現所有教材內容，如圖五所示，以頭頸部癌症為例，
依照方法所提數位教材製作方式，蒐集頭頸部癌症相關衛教單張、錄製團體
衛教內容及放射治療流程，再將製作好的教材內容按照癌症治療時程分為轉
介到癌病中心、治療前、治療中及治療後，將各時期中需要讓病患及家屬了
解的護理指導內容做歸類。所有教材內容存放於學習內容管理系統中，依循
方法中介紹的 SCORM數位學習標準內容封裝方式，使用 Reload做包裝。課程
管理者從入口網站登入後，進入學習內容管理系統新增課程內容介面，左側
 護理指導課程的課程管理者，帳號由系統管理者新增至入口網站，並給予課
程管理權限。從入口網站登入後，左側樹狀選單列出醫護人員會使用到的系
統選項，依照研究方法系統應用流程及整合機制描述，醫護人員從入口網站
登入，之後進入各個子系統便不需再重覆執行登入動作，系統間的資訊傳遞
則使用 web services做溝通。 
    病患轉介至癌病中心，先到登記處報到，癌病中心資訊系統取得病患基
本資料後，醫護人員新增病患帳號及課程之介面如下頁圖所示，課程管理者
從入口網站登入，進入課程管理畫面，點選病患清單並選擇要開課的對象。
系統會要求輸入電子信箱及病患相關資訊，透過病患提供的電子郵件發送帳
號開通信，及作為系統識別病患身分其中一項資訊。若病患本身沒有電子郵
件帳號，也可透過 mail server由入口網站提供病患專屬的電子郵件。

選擇的病患帳號及電子郵件帳號被新增到入口網站，再透過 web services新
 源的衛教資訊，管理者亦可透過管理者介面輸入相關訊息，匯入病患課程中
做公告或通知。除了病患使用上較方便，護理人員平日工作繁忙，亦可縮短
現場口頭衛教時間，製作之衛教短片除了在系統上觀看，也可放在候診區電
視內重複播放，對於不擅使用電腦的民眾也較方便。先提供相關治療資訊，
可讓病患及家屬預先對於會接受的治療流程及相關注意事項有一概觀認知，
減少對未知的治療過程不必要的焦慮，亦能增加治療時配合度。透過數位學
習方式，無論在人、時、地各方面都較傳統口頭衛教方式便利，在病患治療
結束後續追蹤上，亦可提供聯繫甚至病況紀錄功能，利用學習平台提供之互
動機制，課程內容為病患切身相關護理指導及治療資訊，達到以病患為中心
之衛教系統。 
3  學習評量追蹤與分析 
    癌症病患透過 Moodle數位學習平台，可觀看院方提供相關護理指導資
訊，除了醫護人員口頭上詢問病患是否了解護理指導內容，也可透過方法中
描述的表單設計方法設計問卷或簡單測驗。我們提供設計好的題庫 excel檔
案，供醫護人員輸入題目及答案，將問卷答案轉存為 XML檔。使用 Reload將
問卷資料夾封裝成符合 SCORM標準的教材，上傳到 Moodle病患課程中。在學
習平台上呈現的題庫型式如圖九，左側樹狀選單顯示這個 SCORM課程裡包含
的教材內容，小測驗內容顯示於右側頁框裡，提供病患看完衛教內容後作答，
評值病患對數位課程內容了解程度，並能將結果記錄於學習平台內，進一步
做數據統計及分析。

討論與結論 
    基於目前醫療院所普遍未提供較具系統性的護理指導方式，本研究利用
Moodle數位學習平台及整合放腫科資訊系統、學習內容管理系統資訊，提出
個人化癌症護理指導課程之概念，以達到病患於癌症治療前便可了解相關治
療資訊及照護注意事項的目標。要達成此目的，必須院方協助病患帳號及衛
教課程建置，並告知病患或家屬系統好處及如何使用。為了確保系統使用可
行性，於系統正式運行前先透過問卷及訪談方式了解癌病中心需求及系統需
改善之建議。 
    本研究使用 Moodle學習平台呈現各系統間整合後資訊，提供癌症病患個
人專屬的護理指導課程，使用本系統的好處在於，可利用平台功能，增進病
患對癌症治療認知。使用入口網站單一登入，醫護人員需使用相關資訊系統
 到系統化癌症護理指導課程，使癌症病患更能掌握自己的治療狀況。但實際
透過癌病中心問卷及訪談醫護人員後，得到結論仍以考量使用系統族群多為
中、老年人情況，加上系統推行初期可能需要的人力協助考量，建議系統若
能以公開播放衛教內容方式或許較為便利可行。日後研究繼續推行，可能需
要先有較具體數據，說明使用系統化衛教方式未來可省下的人力成本，並可
使癌症病患及家屬得到更完善的衛教資訊。初期使用可能需要花費一些人力
成本來熟悉系統及告知病患，但病患若能預先了解疾病照護須知及檢查注意
事項，必要時還可以重覆觀看課程內容，門診時間再針對個人疑問由醫護人
員加強衛教，應可減少醫護人員門診時間對每位病患重覆執行冗長的衛教。

技術說明
(中文) 本研究探討如何以系統化的方式，針對病人診治狀況及階段(例如不同種類癌症
與治療方式，放射治療前準備內容、治療過程注意事項、治療後衛教等)，給與
恰當的衛教內容。本研究將各類癌症衛教內容儲存至學習內容管理系統中，便於
後續導入教學平台(Moodle)病人特定的課程當中。本研究並以一入口網站，連結
Moodle教學平台、學習內容管理系統、以及放射治療管理系統，便於使用者整合
放療衛教所有相關資訊與系統，以此提供自動化且系統化的病人衛教；透過單一
入口網站進入系統後，醫護人員可使用Moodle教學平台完善的課程與內容控管機
制，透過介面選取自動開課，產生病人個人專屬課程，讓病患或家屬得以使用其
專屬課程資源，以確保其隱私。從放射治療管理系統，可得知目前病人治療狀況
與階段，依此資訊從內容管理系統當中選擇恰當的護理衛教資訊，再將選擇之內
容封裝標準化，並轉入教學平台病人專屬的課程當中，以此建立病人個人化之護
理指導課程內容。另透過本研究擴充開發之系統，可讓醫護人員配合衛教內容設
計問卷表單，方便病人於平台當中填寫，以確認其對衛教內容之了解程度。以此
系統化的方式，提供病人衛教資訊，以減輕醫護人員衛教服務的負擔、增加衛教
過程之互動功能、方便病人了解放射治療過程注意事項、並利於衛教成效之統計
分析。
(英文) In this study, we take a medical center of radiation therapy as a case study of applying E-
learning and knowledge management concepts and technologies in cancer patient 
education. We have constructed a framework that integrates Single Sign On (SSO) Portal, 
Moodle e-learning platform, Learning Content Management System (LCMS), and 
radiation therapy system in this study. In the framework, all the contents and 
questionnaires for patient education were prepared, stored, and organized in LCMS. 
Basing on the integration, we can use the data about patient conditions and therapy stages 
(pre, within, or post therapy) that stored in radiation therapy system to decide what 
education contents should be deploy to patient; We can create education course on the e-
learning platform automatically for a specific patient first. And than, according patient’s 
conditions, we can assign the proper education contents that stored in LCMS to the 
patient’s specific course in Moodle learning platform.
