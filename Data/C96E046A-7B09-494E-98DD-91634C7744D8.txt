中文摘要 
近年來在無線通訊與電子技術的進展下，使得無線感測器這種
體積小、低成本、低耗電的多功能偵測硬體逐漸普及化，且具有偵
測資料、資料處理及無線傳輸的能力。感測器的出現造成了許多新
科技的發展，例如，應用於數位生活相關的居家安全或是智慧建築，
生態環境方面的動植物棲息地監控、軍事用途的電子戰場以及商業
上的應用等等。這些應用當中，與地理位置相關服務是最重要的服
務之一，而移動物體的位置管理是此服務的重要技術。本子計畫針
對無線感測網路下移動物體的位置管理進行詳細而且深入的研究。
在此計劃中，我們針對物體的定位與軌跡辨識提出解決方法。本子
計畫除了設計解決方案並且進行理論性的效能分析外，我們也會設
計相關的系統雛型，以進行實際的效能評估與驗證。

物體追蹤、位置管理

 醫療看護的緊急通報 
行動有障礙的病患在家中或在外可能會有意外發生，但又不方便打電話就
醫。如果在輪椅上或柺杖上放置有無線感測器，當輪椅車倒了或柺杖有不
規則的擺放那就會啟動無線感測器的自動回報，透過感測器的回報病患就
能得到快速的緊急就醫。在大型醫院裡，為了方便監控病患的病情，病患
可以戴上可感測體溫、心跳等功能的無線感測器[JEZDNS05]。當病情有變
化的時候，無線感測器可以自動回報病人所在的病房與其他資訊，以增加
醫療效率和減低人為的疏失。 
 肢體動作擷取 
無線感測器可以偵測肢體或物體的運動。在現今很多的電腦動畫與 3D 人
物，物體的繪製都可運用無線感測器來偵測。無線感測器比其他有線的裝
置較有力來用作肢體動作的擷取，因為無線裝置可以避免電線帶來的阻礙。 
 安全監控 
感測器可以佈置在建築物內來做是否有人入侵的偵查。小巧的感測器方便
讓人佈置在建築物的各個角落，無限的感測器更能方便和有效率的佈置在
戶外以及電線無法獲不方便達到的地方[DC03]。

三、相關研究 
現今普遍的定位技術是利用有地理資訊的感測器當來幫參考點，接著再以
這些參考點的無線感測器共同合作來估計物體座標。估計移動物體位置的演算
法有很多，主要是以參考點與移動物體之間的關係來取得移動物體的地理資
訊。現有利用距離資訊為基礎的定位技術主要可分為兩類：基於測距式方案與
無需測距式方案。基於測距式方案是假設無線感測器具有測量自身與一個跳躍
數(1-hop)鄰居節點間距離或角度的能力。基於測距式方案主要包括有 RSSI 
[AV06]、TOA [LS02]、TDOA [HX04]、AOA [NN03]。另一方面，無需測距式
方案是根據在無線感測網路中傳遞的訊息來推論節點之間的距離，例如根據訊
息的跳躍數計算(hop-count messages)。無需測距式方案主要有 DV-Hop [NN03]。 
 接收訊號強度指示(received signal strength indication, RSSI)  
此方法為讀取接收封包的無線電訊號強度(RSSI)，並利用該資訊來估計位
置。由於無線電訊號強度會隨著傳遞距離的增長而衰減。因此，感測器可
讀取接收封包的 RSSI來估算自身與傳遞訊號節點之間的距離。RSSI定位
技術在測距上是不需要任何額外的硬體，因此，它在感測器部署與能源消
耗方面是具有低成本的優點。然而，在無線的環境中，無線電訊號是很容
易受到干擾，正因如此，RSSI亦會變的不穩定。必然地，感測器在讀取來
自受干擾的無線電訊號的 RSSI值時，將會產生較大的誤差。 
 訊號到達時間(time of arrival, TOA)

四、研究方法 
4.1 不穩定無線感測器訊號環境下物體定位與追蹤之研究與實作 
本計畫的目的在於設計一個有定位以及追蹤移動物體的系統。在分散式的
環境下，在無線感測網路裡執行移動物體的定位、移動物體位置的預測、移動
路徑的預測等功能搭配靜態物體的定位機制來增加移動物體定位的準確度和追
蹤移動物體的移動路徑。所以我們在本計劃裡分別提出了靜態物體定位的演算
法以及動態物體軌跡追蹤演算法 
4.1.1 靜態物體定位的演算法 
本計劃提出了兩個在靜態物體定位的演算法：積極矩形切截 (Aggressive 
Rectangle Truncation, ART)演算法以及積極距離修正 (Aggressive Distance 
Correction, ADC)演算法。ART的精神是每個估計距離都有一定程度的可信度，
所以每個信標所估計的距離都會被採用;ADC 的精神則是試圖要挑出信標誤差
較高的估計距離並且校正它。 
ART定位的基本作法可分為兩步驟。如圖 2(a)所示，第一步是使用已知座
標位置的信標節點對物體進行測距，產生物體的不確定性區域範圍。因為 ART
使用的測距方式是 RSSI 測距，並且是不穩定的無線電訊號，所以測距結果是
一個距離範圍。而 ART便是利用此距離範圍的上界，做為建立物體的不確定性
區域。如圖 2 (b)所示，第二步是利用多個物體不確定性區域的交集來縮小物體
的可能位置，進而定位出物體的座標位置。

4.2 分散式無線感測系統物體追蹤策略設計與實作 
延續第一年計畫成果，在第二年計畫的主要目的就是設計一套追蹤機制，
讓追蹤系統能夠以最省電的方式決定目前負責追蹤移動物體的感測器為哪些，
使得追蹤系統能夠以最節省能源的方式追蹤移動物體。 
為了讓追蹤系統能夠針對當時物體的移動方式採用不同的追蹤策略，我們
勢必要不停的監控物體的移動方式，以確保當下的追蹤策略是正確的。但是，
不停的監控物體和決定追蹤策略這樣的動作事實上是相當耗能源的。因此與其
系統不停的檢驗物體當時的移動方式以及不停的根據物體當時的狀況決定適當
的追蹤策略，我們反而更希望感測器有能力察覺物體的移動方式已經改變，並
且在物體的移動方式改變時主動告知追蹤系統，讓追蹤系統能夠適時的調整追
蹤策略。使得追蹤系統能夠利用一直以省電且遺失物體率低的追蹤策略追蹤移
動物體。

4.2.1.3 活躍追蹤群組(Active Set) 
為了讓無線感測網路內部的一顆顆獨立的感測器能夠在同一個追蹤策略下
互助合作完成追蹤物體任務，我們在本計畫中將目前負責追蹤物體的感測
器組成一個群組，並命名為現用活躍追蹤群組(Active Set，簡稱為 AS)。活
躍追蹤群組的設計目的在於讓我們以有效率的方式管理目前追蹤移動物體
的感測器，使得無線感測網路不會浪費過多的能源在監督移動物體方面。
如圖 5顯示，當物體由左邊移到右邊時，我們去動態調整活躍追蹤群組的
成員，使得活躍追蹤群組的成員由 A, B, C轉為 E, F, G。而原本的 A, B, C

夠不停的追蹤移動物體。必須注意到一點，為了避免多個感測器所執行的
結果不一致(inconsistent)，導致系統所執行的策略有衝突問題(conflict)，以
上的所有運作皆由活躍追蹤群組的代理者完成。

成員，以避免遺失物體狀況發生。

言，活躍追蹤群組成員個數過多時，即可刪除多餘的活躍追蹤群組成
員，讓監督物體的能源消耗量減少。

決定目前追蹤系統所使用的追蹤策略，因此活躍追蹤群組代理人要非
常了解物體目前的移動方式。而為了避免利用傳訊息的方式讓活躍追
蹤群組了解物體目前的移動方式，所以我們設定活躍追蹤群組代理人
必須要偵測到物體。所以當物體已經離開活躍追蹤群組代理人的感測
範圍時，活躍追蹤群組代理人就必須尋找下一個更適合的感測器作為
下一任的活躍追蹤群組代理人，並將維護活躍追蹤群組的重責大任交
付給下一任的活躍追蹤群組代理人。這樣的動作，在本計畫中我們稱
之為活躍追蹤群組代理人交換 (hand-off)。 
 更改設定檔：當活躍追蹤群組代理人發覺到物體移動方式改變時，就
會立即決定適合目前物體移動方式的追蹤策略。當決定好了之後，即
會把新的策略修改到設定檔內，讓下一任活躍追蹤群組代理人可以根
據這個設定檔繼續追蹤物體。

Abstract
More and more users or applications showed high in-
terest in obtaining moving object locations from a wireless
sensor network. However, a moving object usually updates
its location very frequently, and thus, a great number of
messages for updating the locations in the wireless sensor
networks are incurred. In this paper, we propose an in-
dex structure, called the Gossip-enabled Spatial Index Tree
(GSI-tree), to support efficient communication for location
updates. Our idea is to utilize the gossip property of the sen-
sor communication to let the neighboring paths of the GSI-
tree share the transmitted message. In this way, the nodes
managing a past location can deregister the object location
independently by hearing the registration messages from the
neighboring nodes.

Abstract
Recently, with the rapid development of microprocessor
and wireless communication technologies, wireless sensors
have become more popular in academic and industrial com-
munities. The sensor network is used to be the interface
between information systems and users in the real world,
and sensors that support object tracking functions would
broaden the types of sensor applications. In this paper, we
propose an three-layered architecture, Hierarchical Alert
Model Architecture (HAMA), to provide a complete solu-
tion to the problem. In the first layer, the object tracking
server (OTS), is used to maintain the object locations with
less precision due to the reduction of the communication
for location updates. In the second layer, the tracking alert
model is designed to assist the object tracking server (OTS),
to manage all moving objects with higher precision in the
sensor network. In the third layer, the sensing alert model
is designed to assist the sensor nodes for tracking irregular
movement objects. We then conduct experiments for veri-
fying the performance of HAMA. The experimental results
show that HAMA has superior performance in tracking and
management of moving objects.

tracking, distributed systems.

3 Hierarchical Alert Model Architecture

Layer 1: Object Tracking Server
The Object Tracking Server (OTS) is a centric storage of
the sensor network. As a centric storage, the OTS maintains
two kinds of information, including the map and the status
of objects. The map information describes the framework of
sensor network, including the size of the sensing field and
cells, the deployment of cells, and etc. The status of objects
includes the relationship between the objects and the cells
which is represented as a tuple (object id, cell id), where
object id is the identifier of a target object, and cell id is
the identifier of cell where the object stays. Notice that the
OTS would take the center of the cell as the the current loca-
tion of the object due to the following two reasons. The first
reason is that the actual location at a specified time is not
valid in most time. This is because the object continuously
moves and the OTS only updates the location when the ob-
ject moves across a grid in order to reduce communication
cost. Hence, using an approximate location to represent
where the object resides in a specified period is acceptable,
and the concept is also the same as [10]. The second reason
is that estimating the object location by using the center of
the cell can obtain the minimal error between the cell center
and the actual location when considering all cases.

The TAM has two advantages from object tracking as-
pects as follows. The first advantage is the great reduction
of update cost. The TAM mechanism updates a new loca-
tion to the OTS when a moving object moves far away a
predefined region (cell region). In this way, the update cost
of the OTS is dramatically reduced. The second advantage
is the better control for object locations. Once a object is
lost, most works find the object back in the global sensor
networks. However, the object could be still in the vicinity
of the last observing location according to common experi-
ence. In TAM, a local region search can be easily achieved
due to the well-layered organization.

4 Object Management Strategy and Object
Location Query
In this section, we first describe the cell handoff strategy
for object management and recovery of the lost object for
tracking object in HAMA. Then, the useful query interface
with aggregated information provided in the OTS is also
presented.
4.1 Cell Handoﬀ Strategy
Managing a moving object in the sensor network in-
cludes three tasks: (1) determine a set of sensors to monitor
the object at different time, (2) determine an appropriate
cell to take over the object tracking task when the object
moves to the next cell, and (3) store the object locations.
Monitoring the object is assigned to SAM in our proposed
architecture. The other two tasks are assigned to TAM, and
are termed as the cell handoff. Determining appropriate cell
leaders and scheduling sensors to continuously monitor is
a quite important factor to reduce dramatically communica-
tion cost. Next, we describe the details of the cell handoff
in the following.
The idea of our proposed cell-handoff strategy is to
embed some event conditions in the pre-organized sensor
nodes, and the cell leaders are waken up in the actions of
events to handle the tracking task while the object move-
ment satisfies the events. We define two types of events for
each cell leader based on the above tracking alert model.
• Central-Territory Event (abbreviated as CT-Event).
CT-Event occurs when a sensor detects the monitored
object enters C-territory from the previous resident
cell. CT-Event means the object would need a period
to move to another cell.
• Marginal-Territory Event (abbreviated as MT-Event).
MT-Event occurs when a sensor detects the monitored
object enters M-territory from the previous resident
cell. MT-Event means the object could possibly move
to another cell soon.
When the object leaves the cell and enters another cell, one
of the above two events must be triggered by a sensor node
which detects the arrival of the object. On one hand, if the
CT-event is triggered, the sensor notifies its cell leader to do
the cell handoff, as shown in Figure 4(a). After receiving
the notification of cell handoff, the new cell leader informs
the OTS to update the object record so that the future loca-
tion query can follow the record to the new cell leader. The
new cell leader also notifies the old cell leader to remove
the object record to save the storage space. On the other
hand, if the MT-event is triggered as shown in Figure 4(b),
the sensor node that detects the object would select an ap-
propriate cell leader from its own cell leader list based on
the short-term prediction and notify the selected cell leader
to do cell handoff. Many prediction schemes on the object
trajectory is studied in related literature [5], and can be ap-
plied in this work to determine the better cell to continue
the tracking task.

4.2 Object Recovery Scheme
Since the trajectory of the moving objects cannot be pre-
dicted with 100% accuracy, none of proposed methods can
guarantee that a object is never lost in the wireless sensor
network. According to our observation, the lost object can
be found in the nearby area if the sensor system is aware
of the lost object soon. Hence, the proposed Hierarchical
Alert Model limits the search range to the vicinities of the
lost object to reduce communication cost. Once the object
is lost, the cell leader would find the lost object within the
cell before waking up all sensors for object recovery in the
network.
Figure 5 is an instance of finding a lost object. When a
cell leader receives the object location message from a sen-
sor in its corresponding cell, the cell leader sets a Sensing-
Object-Timer to wait the next object location message. In
case the cell leader does not receive the object location re-
ported by any sensor node before the timer expires, the
leader is aware that the object is lost. Then the cell leader
ing radius of each sensor node is 20 meters, and the alert
circle radius is 8 meters. When an M-event occurs, each
active node senses once per second to track the object. In
Alert Tracking Model, each cell is set to 200× 200m2, and
the M-cell width is set to 40 meters. For the wireless sen-
sor network, the greedy perimeter stateless routing (GPSR)
protocol [7] is adopted in delivering query and reporting ob-
ject information. In our simulation, the object velocity is
randomly set between [0, 40] km/hr.
The goal of the proposed Hierarchical Alert Model Ar-
chitecture is to minimize energy consumption of sensor
nodes for moving object tracking and management. We
compare HAMA to other two similar related methods,
EASE [10] and AbOT [3], in the experiments. The perfor-
mance metrics in our experiments include the number of to-
tal sensor nodes, the total number of transmitted messages,
and the total number of cell handoff.
5.1 The Impact of M-territory Size
The first experiment is to observe communication cost
under various M-territory sizes. In the experiment, we use
two types of different moving paths (including regular and
irregular pahts) to observe the impact of M-territory size,
and the result is shown in Figure 7. As our expectation, ob-
jects with regular moving patterns spend less the commu-
nication to manage locations than that with irregular move-
ment patterns, as shown in Figure 7(a). The reason is that
the regular moving pattern is easily predicted, and so it is
much easy to assign the appropriate sensors to track the
object. Another observation from the experimental results
is that the width of the M-territory for best performance
is around 40m in our simulation environment. Determin-
ing the width of the M-territory is basically an optimization
work on different moving patterns. On one hand, the small
width of the M-territory would incur large communication
cost for irregular movement objects, such as 20m in Fig-
ure 7(a). This is because the irregular movement objects
could roam around the boundary of two cells. Hence, the
number of cell handoff increases (refer to Figure 7(b)), that
is, communication cost increases. On the other hand, the
large width of the M-territory also increases the number of
cell handoff, because more cells are needed to be deployed
in the sensing field and the objects are easily to move across
cells.

The comparison of three strategies upon the number of
active nodes is shown in Figure 8(a). EASE+PES uses the
least number of active sensors because only few sensors
around the object are waked up to track the moving object.
For HAMA, due the predefinition of cells, the proposed
strategy could wake up more sensors to avoid missing the
object when an object moves across the cell boundary. From
Figure 8(a), one may think EASE+PES can save more en-
ergy than HAMA. However, when we further trace the num-
ber of messages in Figure 8(b), HAMA uses less number of
message than EASE+PES. That is, HAMA spends less en-
ergy than EASE+PES, although HAMA uses more sensor
nodes. The reason is that HAMA activates sensor nodes that
detected the target to track the target, and hence, HAMA
spends more nodes than PES. Such conservative manner re-
duces the number of target loss for HAMA. Hence, HAMA
can save more energy consumption than EASE+PES by
reducing the number of nodes spent on the object recov-
ery. By our experience from experiments, the difference of
energy consumption between HAMA and EASE+PES in-
creases as increasing number of irregular movement target.
