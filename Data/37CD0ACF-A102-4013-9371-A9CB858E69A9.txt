研究方法 
DICOM RT 物件分析 
 
解析 DICOM RT 檔案的標籤內容索取得之 XML 檔（如下圖），其內容有病人之基本資料與影像之對應
關係等，資料的結構有許多的階層，包含對應到哪些 CT 影像序列與每張影像所包含之輪廓資訊等。
其中在描述檔中 Tag30060010 的標籤內容，對應 DICOM TAG 的名稱為 Referenced Frame of Reference 
Sequence，表示這份 DICOM RT 文件所參考哪些造影影像之序列；Tag30060039 的名稱為 ROI Contour 
Sequence，此標籤紀錄所有參考影像所描繪之輪廓，而且包含許多的子標籤，並依屬性不同的輪廓區
域，以不同的顏色作為區分，記錄每種顏色的子標籤又包含輪廓對應的參考影像，圖中有兩個
Tag30060050 標籤是紀錄輪廓軌跡的座標字串，而字串紀錄座標的方式依序為 X, Y, Z 軸的位置，座標
數值之間以斜線作為區隔。為了將上述的資訊轉換到一系列的 CT 影像，本研究以每張 CT 影像為單位，
收集該影像對應到所有顏色的輪廓線條資訊，製作成一系列的描述檔，並利用 Silverlight 呈現相關輪廓
資訊在對應之影像。

入口網站安全管控 
本研究以 web solution 和 RIA 概念建構放療紀錄與放療影像之檢視系統，並透過入口網站整合相關資
訊與權限控管，使用者經由單一登入的方式，檢視相關放療記錄之系統架構。 放射治療過程中所產生
之相關紀錄資料（DICOM 影像、治療輪廓、治療計劃等），經由 web 化存入 RT 儲存庫，且將放療文
件的資訊傳遞 XDS metadata 至入口網站註冊，以便後續文件交換瀏覽。使用者登入單一登入之入口網
站，利用 SAML 安全機制管理 Web RT 等系統提供之功能頁面，進一步調閱檢視治療報告、影像輪廓
等放療資訊。以下詳述本研究所開發之放射治療資訊檢視系統，以及與入口網站整合相關安全機制之
