3.RFID 內碼讀取和記錄次系統: 
本次系統為 RFID 內碼讀取和記錄系統，如圖一的 RFID 感應距離自動檢
測系統結構概述圖所示，主要是針對閱讀器讀取內碼以及判斷內碼是否重碼，
接收 RFID 距離量測控制系統所讀取 RFID 卡片感應距離值，並判斷此 RFID
卡片感應距離是否大於設定值、依照 RFID 卡片內碼是否重碼與感應距離是否
大於設定值的判斷結果，將 RFID 卡片內碼和距離值記錄至適當的資料庫內。
RFID 內碼讀取和記錄次系統內的操作，主要分成三塊區域進行說明整個的結
構與組織: 
i.RFID 卡片內碼讀取: 
RFID 卡片內碼讀取的架構如圖五所示，首先應該設計一個內碼傳輸電路
(實體可參見圖四的標註)，當閱讀器感應到 RFID 卡片時，可以用本電路將閱
讀器所讀取到 RFID 卡片內碼透過 RS232 傳輸線，將內碼傳輸到電腦上，電
腦部分用 Delphi7 介面操作控制程式，將讀取到內碼以 SQL Sever 資料庫加以
記錄，因此上述介面操控程式應可接受內碼傳輸電路極限開關的觸發，讀取
RFID 卡片內碼，並將內碼值顯現在 Delphi7 的文字方塊(Edit)介面物件上以作
為檢核，同時經由設定之 Delphi7 介面操作控制程式與 SQL Sever 資料庫間的
傳輸通道，將內碼寫入 SQL Sever 資料庫。SQL Sever 資料庫應該包括:序號、
RFID 卡片內碼、感應距離等欄位。Delphi7 的文字方塊(Edit)介面物件在程式
執行時的情形可參見圖六所示。 
ii.RFID 卡片感應距離讀取: 
如圖九所示，當 Delphi7 介面操作控制程式接收到讀取器感應到 RFID 卡
片觸發訊號時，除了將 RFID 內碼記錄在 SQL Sever 資料庫外，並判斷內碼是
否有重碼，然後驅動 MMI ware 通訊伺服器，將重碼判斷結果經由 RS232 傳
輸線傳遞至 RFID 感應距離量測控制次系統的 PLC 控制器，此 PLC 控制器所
得到之 RFID 感應距離值，亦會經由 RS232 傳輸線顯現在 Delphi7 的文字方
塊(Edit)介面物件上以作為檢核，同時經由設定之 Delphi7 介面操作控制程式
與 SQL Sever 資料庫間的傳輸通道，將感應距離的值寫入 SQL Sever 資料庫。

的 SQL Sever 資料庫找到，此點可以在系統組合完成後，加以測試驗證。 
上述三個次系統間會有資料的傳遞，當載具攜帶 RFID 卡片移動向閱讀器
讀頭靠近，閱讀器讀頭感應到 RFID 卡片時，RFID 內碼讀取和記錄次系統將
會讀取 RFID 內碼，並判斷內碼是否重複，將判斷結果的訊號傳遞至 RFID 感
應距離量測控制次系統，此時 RFID 感應距離量測控制次系統也量測 RFID 卡
片的感應距離值，將感應距離值傳遞到 RFID 內碼讀取和記錄次系統，並判斷
此距離是否大於設定值，將判斷結果的訊號傳遞至自動化進退機構次系統，使
其將 RFID 卡片移置到適當的卡片夾。 
4.結論與建議 
i.為避免機構干擾到 RFID 卡片和閱讀器間訊號的傳輸，自動化進退機構次
系統的設計，在閱讀器和卡片間不能有金屬存在，因此在本計畫中，將機
構的外殼皆以壓克力板來製作，來確保 RFID 卡片訊號的傳輸不會受到干
擾，至於步進馬達、螺桿等，設計在閱讀器和卡片感測區域之下，應不致
於對 RFID 卡片和閱讀器間訊號的傳輸造成影響。 
ii.Delphi7 介面操作控制程式為本系統整合的核心，PLC 控制器以及 SQL 
Server 資料庫，資料的處理和傳遞，皆是藉由 Delphi7 介面操作控制程式，
此外，經由 Delphi7 文字方塊的介面物件，也可以對所處理的資料或訊號
加以監控，大致而言，相當好用。
