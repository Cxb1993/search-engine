data size. secondly, astronomy researchers have 
various interests in different sub sets of the 
observation data. Because good observation locations 
are limited and observation equipments‘ cost is 
high, accessing and analyzing data requires timely 
response and large amount of data transferring over 
global network. 
Grid computing, especially data grid system, provides 
the solution to the data center for the astronomical 
researchers. 
We therefore propose to construct an efficient data 
grid system that has the following features: 
1. The distribution of data is based on certain 
classification criteria for different research topic.
2. The system provides interface and services for 
different astronomical researchers, and allows 
distributed accessing of the dataset 
3. The data grid system supports distributed data 
warehouse structure that has been used in previous 
projects 
4. The data grid system supports back-up and recovery 
modules to maintain system availability. 
The construction of the data grid system will also 
adapt related database and data mining techniques to 
provide good system performance that most 
astronomical research need.

中文摘要 
 
本實驗室經由與中央大學天文所的合作，我們查覺到天文領域的研究者正面
臨以下的問題。第一，天文觀測的數據量非常龐大，像是著名的泛星巡天計劃
(Pan-Starrs Project)和中美掩星計畫(TAOS Project)，藉由新型觀測台的運作，可
以在短時間內產生數以TB計的資料量。為了因應這些多樣化的分析方式以及資
料的維護，不同的研究員都需要有比理論資料量更大的儲存空間。第二，天文
研究員對於觀測資料中的部分子集合也有著許多不同的研究興趣。因為良好的
星空觀測點受限於氣候因素有所限制，且天文觀測器材往往非常昂貴，因此國
際間的研究人員開始傾向於相互協同合作並在成員間分享資料。面對多樣性的
存取以及資料的分析，也需要以高度分散式的環境來做處理。而以分析為導向
的分散式系統架構特別需要能夠提供回應的即時性以及穩定且大量的資料傳輸
需求的網路環境。 
網格計算，特別是資料網格系統，是提供天文研究員建構資料中心的一個方
法。我們因此建議架構一個包含以下特點而有效率的資料網格系統： 
1. 資料的分散法是基於不同研究議題下，相同分類法分類後所得之結果。 
2. 系統對於不同的研究員，提供了不同的介面及服務，並可存取分散式架構中
的資料。 
3. 該資料網格系統支援在先前計畫中的分散式資料倉儲架構。 
4. 該資料網格系統提供備份以及復原模組，讓系統維護更為方便。 
該資料網格的架構也可使用關聯式資料庫以及相關的資料探勘技術在其中，來
提升系統效率，而這也是多數天文研究員所迫切需要的。 
一. 前言 
由於科學的改善使得科學家們得以建立各種高度精確的科學觀測技術
以及儀器，但是這些新式設備卻需要搭配更大的存儲和更快的數據處理技
術。從天文學家的觀點來看，當下最重要的挑戰是如何從巨量的觀測數據資
料中有效地提取有用的天文資料。雖然現今的分散式系統架構，例如網格系
統或雲端系統，可以提供科學家們較單一電腦強大的硬體運算資源，但是對
於超大型資料管理以及分析研究的相關軟體研發進度卻遠遠落後，而這尷尬
的困境也阻礙了計算資源有效統整及利用的可能。所以我們認為如能提供一
個統合，高效的資訊管理和分析系統將會非常有利於天文領域的研究進行。 
為了能夠在大量的天文資料中更有效的進行資訊分析，我們引入了基於
機器學習以及分群技術的新式自動方法。此系統將由三個子系統所組成，分
別三大類不同的問題。其中一個子系統將會用來提供一個統整的介面並且控
管所有資料傳輸及檢索。其餘兩個系統將會提供以分散式環境為基礎的分類
以及分群工具，讓使用者能夠直接選擇自己有興趣的資訊區塊並且專注於進
一步的資訊分析，而無須處理大量原始資料。本計劃的原型系統能夠讓研究
人員許在多數據處理的應用，例如分類研究目標，驗證已知的理論模型，以
及大量的數據管理上獲得更佳的效率。 
二. 研究目的 
本計畫主要目的為將商業資料倉儲系統以及大型分散式網格技術引入
天文領域，並發展出快速且有效率的資料管理以及分析系統來處理大量天文
資料。本系統可以被視作一個支援大型資料集合及分析的資料網格系統，此
系統將由一個資料管理系統以及兩個資料分析系統組成，分別處理資料管理
搜尋及傳輸問題，行星分群問題，以及相似性分類問題。 
的資料，並照著使用者的需求執行計算任務，最後將答案回傳至上層。本研
究之三個子系統運作方式茲分述如下： 
甲、 天文資訊管理系統(ASTROnomical Information Management System, 
ASTROIMS) 
天文資訊管理系統是由本實驗室先期與中央大學天文所合作完成之中
美掩星合作計畫資訊系統(TAOS Information System)[1]為基礎發展而
成，其目的為進行資料管理以及協助擷取分析系統所需資料，由於中美
掩星合作計畫資訊系統主要著重在資料倉儲系統之資料管理以及維
護，並無提供進階資料分析功能，且本身無法應付較複雜之分散式環
境，所以本次計畫將此系統重新設計，成為以網格環境為基礎之資訊管
理系統，並且與分析工具整合，成為分析工具的資料回饋及整合管理元
件。天文資訊管理系統提供不同的資料提取功能，除了底層需要大量計
算的分析工具之外，上層介面同時也提供一些簡易的分析及繪圖工具，
使用者可以針對較小型資料直接在上層介面進行基本分析。天文資訊管
理系統採用多維的資料倉儲概念,這個多維的資料倉儲合併了不同階層
的數據，讓使用者可以選擇較為宏觀或深入的分析模式。 
天文資訊管理系統由三個核心子模組構成：查詢處理器，資料倉儲，以
及計算節點控制器。查詢處理器負責分析用戶的查詢，如果有必要的話
也可將原本的查詢分裂成幾個較小的查詢，並將這些小查詢重新安排
（為了更好地利用站存機制），之後藉由第三層的數據傳輸服務將這些
查詢分配到不同的機器進行計算，在接收到第三層分析工具傳回的結果
後，系統將它們收集並結合成一個完整的結果，然後將它們存儲在資料
倉儲中。而在計算節點中有四個主要組成部分來保持群集和執行本地查
詢，分別是(1)集群控制(2)計算資源(3)資料池(4)互聯網數據提取。集群
控制負責將集群保持在最新的狀態。計算資源能在分散式環境和幾個內
置的分析功能中提供計算能力，當接收一個任務時，系統將會檢查本地
資料池來找到必要的資訊，並且從互聯網數據中下載缺少的資料，當系
減少分析系統的訓練所需時間。最後將每個從各節點蒐集到的分類器結果，
統計出每個分類器對於該 Evaluation Set 的正確率，挑選出正確率最佳的分
類器作為該次分類所用的分類器，如此對於不同的 Evaluation Set，最後挑選
的分類器可能會不同，因此有動態性。 
相似性分類系統提供了多種方式將大型資料劃分成無數塊的小型資
料，並同時用多個 SVM 平行分類每個小型資料並產生分類模型。當一個使
用者提交一個任務給相似性分類系統，使用者可以選擇一個已經存在的分類
模型來分類此數據或是創造一個新的分類模型來分類。本系統也提供多種已
知方法來培養和選擇合適的分類模型，例如 Local Accuracy 或是 Cross 
Validation。此外，使用者可以選擇只使用一個分類模型，或是同時選擇較
多分類模型合併成一個較複雜的分類模型。
