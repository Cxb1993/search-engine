摘要 
在既成的無線通訊網路環境下，透過基地台訊號強度進行定位估算，被視為是不需擴
充硬體設備且最易於實現完成的方法，後續的行動通訊的位基服務與相關運用亦可據此而
快速發展，但是依據前期 WiFi 定位發展平台的建構與分析研究結果顯示，在大區域的環境
下，參考行動通訊所架構的基地台配置進行定位，各式模型均有相當的改進空間，而針對
不同精度需求的定位服務，增加配置定位用的無線基地台，應是提升定為準度有效的方法；
本研究是針對市場上逐漸增加應用的 Zigbee 無線感測網路，與前期 Wi-Fi 定位發展技術結
合，建構複合式無線網路定位模型開發所需資料庫，同時採用以訊號強度為基礎的複合式
定位模式進行研析。 
本計畫建置複合式無線定位資訊蒐集平台，並依據實驗規劃建置評估實驗場與驗證需
求模組，並針對小區域提升定位準度的複合式定位模型的實驗分析與模式驗證進行進階的
探討，在實驗平台定位量測上，除在實驗場地繪製定位標識外，並分別開發紅外線、影像
與光學模組建立定位量測模組，提供自動量測路徑規劃的選擇與比對，相關模組的硬體說
明與軟體模組均以公開之協作平台方式，供國內學界參考運用，期能透過分享促進國內
Wi-Fi 或複合式無線網路定位，能朝大區域定位模型實用化的發展。 
 
 
關鍵字：Wi-Fi、Zigbee、無線感測網路、定位 
Abstract 
 
In the existing wireless communication environment, the position can be estimated by Radio 
Signal Strength Indicator (RSSI) that was considered the most practical method without extra 
hardware in a mobile device. Once a reliable model was verified, the locating based service will 
be progress fast as expected. The results from previous project showed that the existing wireless 
communication network infrastructure may not provide enough information for locating purpose 
by RSSI model. In order to enhance the positioning accuracy, to install extra wireless APs for 
locating purpose may be a practical way. This study will focus on the improvement by adding 
Zigbee modules as mixed wireless locating references since the wireless sensor network may be 
applied in building automation, industry automatic sensing and/or home applications in the near 
future. 
The study will expand the Wi-Fi position platform by adding Zigbee, position measurement 
modules and use them to build a mixed references positioning database while the information is 
not only used in construct positioning model but release them in public web site. 
 
Keywords: Wi-Fi, Zigbee, Positioning, Wireless Sensor Network 
 
研究背景與目的 
    隨著無線網路基礎建設逐漸普及等因素的激勵，LBS 應用服務之蓬勃發展將是不可擋
之必然趨勢[1]，在室內的位基服務的應用中，國內資策會網路多媒體研究所與台中自然
科學博物館進行合作，在約 6000 坪的展場配置了 155 個 ZigBee 節點(包括 Router 與
的主要道路上做室外定位測試，約每 100 至 200 公尺取樣測試，共取樣數 52 點，並使
用 Garmin eTrex 手持式 GPS 定位裝置量測參考座標用來計算定位誤差，雖然手持式 GPS
再受遮蔽處，平均誤差仍大，但相較於手機動輒三、四百尺誤差來得小很多。中正大學研
究群[18]2006 年發展一套使用 ZigBee 網路無線技術來實現室內定位系統，可應用於圖書
館，科博館等等地方，在 360x652cm2的空間中，分別配置 4個與 6個參考點進行驗證，當
節點越多時我們可以知道其定位精確度會提高。成功大學則分別有[19-21]用 Zigbee 與
WiFi 進行資料蒐集與模型比較，以 Zigbee 的驗證場地之空間為６ｘ６米與 8x24(x2)m2的
方形空間與 L型走道，配置 4個參考點進行驗證。另建立高斯混合模型之 IEEE 802.11 無
線區域網路定位演算法，以該校資訊大樓配置 14個基地台，進行驗證；同時亦有運用慣
性導航之導航資訊與Ｚigbee整合，整合慣性導航定位系統在4x6m2平面上配置4個Zigbee
參考點收集資料與 IMU 慣性系統之動靜態資訊比對。交通大學研究群則分別提出[22-25]
台 (VBS) 輔助演算法＂與 Geometry-constrained Location Estimation (GLE)的定位模
型，另外亦提出色彩理論之改良型動態定位技術，規畫實驗在 20 ×20 m2空間配置 3個 Mote
為參考點，四個mote當作測試點，改良型色彩理論模式的定位誤差由4.76m進報到3.17m。  
2007 年台灣大學的研究顯示[26]室內及室外環境與不同演算法比較 GSM 與 WiFi 的定位能
力，運用 GSM 的定位系統的平均定位誤差在室內/室外的環境下是 11/113 公尺，而 Wi-Fi 
的定位系統在室內/室外環境下的平均定位誤差為 5/29 公尺，所以預期以這樣的精準度建
構可適用於室內及室外的定位系統是可行的；2008 年中興大學研究群則加入方位預測提
供更精確的 WiFi 定位方法[27]。2008 元智大學林熙堯在線感應網路下之感應節點定位機
制演算法的研究中，演算法最大的特色是即使未知節點周遭只有一個或兩個錨節點，未知
感應節點仍可以推估出本身位置。 
    醒吾技術學院研究群，針對 U-care 課題展開之系列研究，在位基服務向上分別針對
藍芽[28]與 WiFi[29]定位進行相關研究，在 WiFi 定位研究子題，已完成自走式 WiFi 定
位資訊蒐集發展平台，提供較大區域的資料蒐集，除以自建 WiFi 參考源的方式，進行定
位模式的測試外，並針對校園不同區塊的既成無線網路架構，進行定位服務的可行性研究，
初步結果顯示，單層樓小區塊的定位預測，可以獲得較佳之準度，大區域與不同樓層的定
位分析，仍有相當改善的空間，若配合不同區塊環境，輔以定位用參考源，應可針對應用
需求，提供對應的定位資訊。 
    此外，在位基服務與圖資的開發工作上，亦有以 OpenLS 應用於停車資訊服務的系列
研究，題更完整的區域圖資與行動服務平台，對定位模組的實務整合與後續應用的發展極
有助益。 
    相關研究顯示，大區域的定位技術，依量測的基本原理，定位精度會受模型區域尺寸
的影響[30]，依據本研究群初步之成果顯示，在既成 WiFi 大區域網路架構下的定位技術，
仍有相當改善的空間，若配合更精確的定位需求，應可運用具有發展潛力的無線感測網路
的輔助，提升大區域的定位應用與整體的定位準度；所以本研究擬延續本年度研究成果，
並擴充無線感測網路的技術與定位量測模組，探討複合式定位系統在大區域定位服務可行
性的研究。 
複合式定位發展平台 
    本研究之規畫之發展系統示意圖如圖一所示，行動平台是由 WiFi 控制的全方位自走
車(Rovio)為行動平台，配置具備無線通訊單元、IR模組與光學位置感測模組，並配合實
驗需求的 WiFi 與 Zigbee 基地台，而在主控電腦端，除具備 WiFi 通訊埠外，並分別為配
前法類似，需進行前置的定位校正，且受限於攝影機的解析度，若需提高定位量測精度，
則攝影機數目需增加，若採用前法增加信標源則是較經濟的方式。 
(6) 位置量測模組3:本模組採用宇瞻M821 (Apacer) 2.4GHz 無線雷射滑鼠，雷射解析度，
可以透過按鍵在 800 以及 1600 CPI 兩種模式間切換，Rovio 自走平台的全方位輪提供任
意方向的平移，所以可以控制在不旋轉平台的情況下，移動平台置測試點，則一支定位滑
鼠即可提供自起始點後的高解析度平面移動資訊，為此系統會受測試地板情況影響。  
(7)主控電腦：主控電腦採用具備 USB/BlueTooth/WiFi 介面的一般 PC，分別配置 Zigbee
模組，可接收 Zigbee 模組傳入訊號，或透過它與其他 Zigbee 模組通訊，同時具備藍芽介
面，提供 wiimote 控制器連接，透過 Wiimote 控制器，手動調整自走平台姿態，或目視法
控制自走平台，在設定標記處，依序改變位置與資料擷取。 
(8)軟體模組：為提供更彈性系統架構，探討複合式無線定位中基地台最佳化配置等課題，
故本平台開發具備彈性增減基地台之功能，此系統主要分別在兩套作業平台上開發，主控
電腦端採 Windows7 作業平台，而 Zigbee 模組的軟體是採用 TinyOS 作業系統，自走平台
則運用製造商提供的應用開發介面，透過瀏覽器控制項進行發展；Zigbee 模組採用 ID來
規劃執行程序，三類模組共用相同程式，由行動端模組(ID=1)定時以 ID=10001 廣播訊號，
基地台收訊後回傳行動元件，俟行動元件接收訊號後，將訊號強度與 ID傳至主控電腦端
之模組，主控電腦端模組再將資訊傳回主控電腦，做為定位資料蒐集與定位計算運用，為
模擬以基地台為主控端的定位方法，亦建立multi-hopping功能模組，提供更彈性的發展。
此外，主控電腦端另行以 C#開發 Zigbee 解碼類別，供主控電腦控制掃描與讀取 Zigbee
基地台傳訊至自走車的訊號強度；運用Rovio API 建置透過WiFi控制自走車運動C#類別，
並使用 WiimoteLib 建置藍芽控制器控制程序，將個模組整合之測試程式，由藍芽控制器
依序進行：(1) 導引自走車到達測試格點位置，(2)下達掃描基地台到自走車 Zigbee 模組
的訊號強度與自走平台 WiFi 到達固定端主控電腦的訊號強度，(3)依序儲存基地台訊號強
度、Rovio 位置量測模組資訊；重複(1)-(3)步驟至測試完成。 
 
結果與討論 
    運用訊號強度方式建置的無線定位模型，有助於位基服務應用的發展，本研究開發的
發展系統，提供彈性的基地台配置、複合式無線訊號強度的收集、測試格點的規畫與資料
蒐集的自動化，便於針對開發模型的需要，配置因應的測試場地與資料蒐集。 
    自走車具備北星定位系統，此定位系統宣稱在屋頂高度為 2.5m 的情況下，2.5x2.5m2
下可獲的 1-4cm 的重複定位精度，但是經初步測試，在 2x2m2 場地測試，在較大範圍運用
時，系統開發廠商建議採用多點信標源，但是由於北星座標系統的數值，與物理座標並非
線性關係，自行開發校正的前置處理，除了增加直接運用座標資訊的複雜度，更需精準的
預置信標源的位置，自走平台亦運用較經濟的益眾遙控自走車，以數位輸出透過 433 遙控
器亦可作為定位感測器之測試載台，運行速度較快，但是相關定位模組亦須配合，但是由
於非全方位輪，旋轉運動會增加光學定位模組的難度，唯速度較高，開放式載台提供較彈
性的擴充功能。 
    雷射滑鼠位置量測模組提供解析度最高的位置資訊，但是對接觸面要求的限制，不易
在非平滑場所量測，同時地形的起伏、自走平台的旋轉等因素，亦增加此模組實際應用的
限制，但是若考量經濟因素與基礎測試，實驗環境的配合，此模組提供精度極高的位置資
訊。 
    複合式無線定位系統，可分別擷取 WiFi 與 Zigbee 的訊號強度，配合 Zigbee 基地台的
配置，另行建置具備點滴重量監控功能的無線模組[31]，Zigbee 模組的類比數位輸入端子，

結論 
    運用無線通訊訊號強度進行行動裝置定位，可提供更便利的位基服務，本研究建置的
複合式無線訊號接收自走平台，提供相關無線通訊訊號強度分佈特性之實驗分析與評估，
自走式平台提供更快速與完整的資料收集，在 WiFi 訊號強度模組部分，因改採用 Windows 
7 的主控平台，提供更快速的處理速度，運用自走車自動搜集相關資訊，提供探討基地台
的最佳配置，或定位輔助用基地台的需求與配置等相關議題，預期對無線定位的準度提升
有顯著的影響。

計畫成果自評 
    本研究將定位模組整合進一自動資訊蒐集與分析發展平台，對整合 WiFi 與其他無線訊
號源的定位環境，預期提供更佳的定位精度，同時定位服務環境的建置，能依據自行規畫
之路徑，自動搜集複合式無線網路基地台的訊號強度，建立目標區的定位參考點無線訊號
強度分布之資料格點，提供更精確定位模式的發展與驗證。 
    本計畫已依規劃完成之工作項目如下： 
1. Zigbee 定位模組開發 
2. 紅外線與光學定位模組開發(原規劃 MEMs 定位模組，採用運用 Wii Motion Plus 進行發
展，受限於雜訊比的處理技術，另行開發定位模組)  
3. 複合式無線感測定位模組整合平台 
4. 複合式無線定位資料庫建置 
5. 網路服務平台建置(https://sites.google.com/site/jouhexwls/)

計畫編號：NSC 98－2221－E－226－002－ 學門領域：WEB 技術 
技術/創作名稱 WiFi 定位發展平台 
發明人/創作人 周重石 
技術說明 
中文： 
本技術整合自走式全方位輪型機械人、Wi-Fi 接收器、Zigbee 接收
器，開發具備三項定位模組之位基服務發展系統，在既成無線網路
與輔助無線定位參考源環境下，自動依規畫路徑蒐集定位格點資
料，提供無線訊號強度分佈特徵與定位模型建置相關開發工作。 
 
英文： 
An location base service developing platform including Wi-Fi 
and Zigbee and several positioning odules provides the 
functions to automatically collect signal strength 
distribution in an existed Wi-Fi infrastructure environment 
with assiting zigbee references for determining and/or 
validate wireless Locating models. 
可利用之產業 
及 
可開發之產品 
行動定位、位基服務產業開發定位模型。 
技術特點 
運用既有無線通訊環境，運用可能被推廣的無線訊號參考源，彌補
通訊用基地台定位參考之不足，提供更佳的定位結果。 
推廣及運用的價值 
整合既有無線通訊環境，擷取預期推廣的無線訊號參考源，提供更
廣的輔助參考源與預期更佳的定位結果。
