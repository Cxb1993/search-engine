英文摘要 
In this study, we use two PC to simulate Hadoop MapReduce cloud computing architecture. To emulate 
MapReduce technology, this project disperses data analysis tasks to the cloud hosts, hence reduces data 
computation time. In order to verify the execution speed of the architecture, this study compared the proposed 
architecture with a single host computing architecture. The experimental results show that the execution speed 
of the proposed cloud computing architecture is superior to a single host computing architecture. 
 
Keywords: HTML tags, information hiding, smart phones, cloud computing. 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 1 
 
前言 
隨著電腦與網際網路 (Internet) 的發展，網格計算(Grid Computing)、分佈式計算(Distributed 
Computing)、並行計算(Parallel Computing)、效用計算(Utility Computing)、網絡存儲(Network Storage 
Technologies)、虛擬化(Virtualization)、負載均衡(Load Balance)等計算機技術和網路技術紛紛因為需求
而提出。 
雲端運算 (Cloud Computing) 就是將前述所有的技術整合在一起的概念，一方面是讓使用者以更
加便利的方式使用及取得服務，甚至用最簡單的方式開發新的服務。隨著各種雲端服務產生，對於運
算能力及儲存空間的需求，也會驚人地成長，因此雲端運算的另一方面就是整合組織內部運算資源，
以最有效率、最易於管理的方式，提供雲端服務穩定的運算及儲存能量。 
由於網路技術大幅進步帶動雲端技術產業蓬勃發展，早期的資料分析都是透過單機作業方式進
行，對於需要大量分析運算資料而言，往往需要耗費較多時間進行運算：例如：統計巨量資料裡相同
的項目。加速資料分析上的速度與縮短分析所需時間，是目前所面臨的問題。雲端運算技術的提出，
恰好解決這個問題，雲端運算在硬體架構上透過虛擬化技術讓雲端提供穩定與巨量資料處理能力，在
資料運算技術，透過 MapReduce 運算架構提供分散式平行運算，很適用在大量資料的處理工作。 
研究目的 
近年來雲端運算在世界各地皆普遍受到重視。在維基百科中的定義，雲端運算是一種基於網際網
路的運算新方式，透過網際網路上異構、自治的服務為個人和企業使用者提供按需即取的運算[1]。其
意思是，透過將網際網路上的伺服器集中起來並提供服務以減輕客戶端的負擔。雲端運算代表的不是
一種新的技術，是一種觀念。因為人們持有的資源有限，能處理的速度將不高，若能藉由利用網路上
成千上萬的電腦或伺服器同時及分散地幫人們處理事情，則運算速度及效能將會提高很多。換句話說，
使用者端設備之功能將會越精簡，大部分的應用及服務將會由雲端上伺服器提供。 
雲端運算可以大致分為以下三個範疇：Software as a Service (SaaS)、Platform as a Service (Paas)以
及 Infrastructure as a Service (Iaas)，其意為軟體即為服務、平台即為服務、基礎建設即為服務。目前網
路上已有很多相關的應用，例如 Yahoo 和 Google 的電子信箱服務，以及 Google 的搜尋引擎技術即是
屬於 SaaS 之範疇。而較著名之例子如 Google AppEngine 及 Microsoft Azure 即為 PaaS 的範疇。隨著網
際網路的發展，世界上各式各樣的網站每年皆快速地成長。 
近年來 Web2.0 的創新觀念，在網路上存取及分享資料已經普遍於我們的日常生活中。在網際網路
上的網頁檔案任何人都可以瀏覽，由於網頁使用普遍，及網頁檔案放在網頁伺服器裡，除非遭到侵入，
否則一般而言已具有基本之安全性的特性，於是有將網頁檔案做為掩護檔案的研究。因此，利用網頁
檔案來做為資訊隱藏的掩護檔案，將會是實用的資訊隱藏技術。本計畫目的，希望藉由雲端運算的觀
念與應用，能彌補上述傳統資訊隱藏技術的缺點，並且發展出適用於雲端運算環境下之安全及巨大隱
藏量的網頁資訊隱藏技術。 
文獻探討 
雲端運算是基於網際網路的新型態服務概念，不論終端是那一種的設備，都可以透過網際網路提
供即需即取的資源，如圖(1)。由於資源是在網際網路上，而在系統架構圖中，網際網路通常以雲形表
示。因此被形象化類比為雲端運算，透過網際網路提供。用戶不需要了解雲端基礎設施的細節，也無
需直接進行控制,只關注自己真正需要什麼樣的資源以及如何透過網路來得到相應的服務。 
 
 3 
 
竊取或竄改的聯想或臆測。 
目前網際網路上的網頁內容的呈現皆須由瀏覽器的剖析解譯，由於網頁文件對於標籤或屬性沒有
制式的排列順序要求，所以瀏覽器對於相關標籤或屬性排列順序皆存在著剖析解譯的彈性空間，所以
有學者藉由利用瀏覽器剖析彈性的空間，提出植基於標籤或屬性排列順序的方法[3][4]。此類方法並不
會在嵌入機密資料後使得偽裝文件大小遽增；此外，除非第三者事先知道了嵌入規則，否則嵌入的機
密資料是無法被取出，大幅提升安全性。然而，此方法存需要大量的額外空間來儲存標籤或屬性在不
同排列狀況下所代表的意義。 
一般 HTML 的標籤大多是成對出現，換言之每一個標籤是由開頭標籤與結尾標籤所組成，在開頭
與結尾標籤中的內容為欲呈現的網頁資料。但少數標籤由於其用途並非是作為資料呈現使用，因此會
沒有內文資料的存在，這類的標籤稱之為空元素，例如：<img>、<p>、<br>、<hr>…等等，空元素標
籤原則是單一出現，但如果撰寫成成對出現，經瀏覽器解釋後其結果也不會造成任何錯誤的產生，因
此利用切換空元素之兩種撰寫方式：成對標籤或單一標籤，也可以將二元機密資料順利嵌入為 HTML 
文件中。雖然此類方法[5]可以順利將機密資料嵌入 HTML 文件中，但由於可以使用的空元素標籤選擇
性不多，有著資訊藏密量不高的問題。 
為達到能提升資訊隱藏量又希望不會造成檔案大小擴張的目標，有些研究提出利用植基於字母大
小寫切換的方法來進行機密資料嵌入[6]，此類的方法主要是利用網頁文件之標籤不區分大小寫字母的
特性來進行資訊隱藏的目的，其作法是根據二元機密資料的內容來變更標籤的字母大小寫的狀態。此
類的方法雖然可達成高資訊隱藏量的目的，仍然存有安全性的問題。主要的原因是標籤字母大小寫切
換過於頻繁很容易被第三者察覺文件中的異狀，所以有些學者便針對這類的資訊隱藏法提出了隱藏訊
息分析的偵測攻擊(Steganalysis)[7][8]，只要文件中大寫字母與小寫字母達到某種程度的比率，便判定
一定有某些的機密訊息藏於此份文件中，進而產生竄改或破壞的攻擊行為。 
研究方法 
如圖(3)所示，雲端運算環境架構的智慧型手機資訊隱藏系統(以下簡稱本系統)是藉由智慧型手機
將機密資料或檔案加密並轉換成二進制的加密訊息，再傳送到 Agent Server 去轉換成十進制數值後分
送至系統中的 Cloud Server，接著索引每部 Cloud Server 的機密資料索引資料庫中所相對應的 HTML
標籤屬性排列順序，最後將相對應的 HTML 標籤屬性排列順序儲存為新的 HTML 文件。擷取機密資料
時，只需將嵌入完成的網頁提供給 Agent Server 的擷取程序，系統中的每部 Cloud Server 便會解析網頁
標籤排列順序並透過資料庫取得索引值並轉換成二進制數值傳送到智慧型手機端，最後在智慧型手機
端做 AES 解密即可擷取出機密資料。 
 5 
 
方式並給予索引。Server 端的每部 Cloud Server 會個別將原始網頁 A 與原始網頁 B 指定的標
籤屬性排列順序修改為與 Sd 編號相符的標籤屬性排列順序，另存為新的嵌入網頁 A 與嵌入網
頁 B。 
當兩個嵌入網頁都產生之後，程序會告知使用者完成訊息，而這兩個嵌入網頁在使用網頁瀏覽器
瀏覽時，會跟原始網頁一樣沒有任何差別。擷取程序的部分，使用者登入系統後將嵌入的網頁文件進
行解析取出所嵌入的訊息，再透過 AES 解密即可擷取出機密資料。 
擷取方法如下： 
1. Server 端的每部 Cloud Server 皆有機密資料索引資料庫，並預先建立各種標籤屬性排列順序方
式並給予索引。Server 端的每部 Cloud Server 會個別將藏密網頁 A 與藏密網頁 B 指定的標籤
屬性排列順序對照機密資料索引資料庫，找出與標籤屬性排列順序相符的索引為數值 Sd。 
2. Server 端將 Sd 轉換為 2 進制數值 Sb 並傳送置智慧型手機端。 
3. 智慧型手機將接收到的 Sb 轉成 ASCII 碼後再經由 AES 解密即可取出機密資料 S。 
擷取程序會依照嵌入網頁中的標籤排列順序到機密資料索引資料庫比對索引，找到索引後最後轉換成
機密資料給使用者。擷取程序最後會進行 AES 解密的動作，若 AES 金鑰錯誤將無法正確解密機密資
料。本研究所使用的方法需要先建立機密資料索引資料庫，目的是為了加速藏取機密資料的速度，因
為機密資料數據非常龐大，所以事先利用資料庫儲存機密資料索引，將會加速將來嵌入與擷取的處理
速度。Server 端的每部 Cloud Server 中皆有建立機密訊息索引資料庫，資料庫中將會存放各種標籤的排
列順序與對應的機密資料，如圖(5)。
