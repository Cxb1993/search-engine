Abstract 
 
 We consider the unrelated parallel- 
machine scheduling to minimize total 
weighted flow time. There are N jobs, each 
requiring a single stage of processing on one 
of the M machines. Due to the attribute and 
mechanical structure of the machines, the 
processing time for a job depends on both the 
job and the machine on which the job is 
processed. A sequence-dependent setup time 
is required if there is a switch from processing 
one type of job to another. The required setup 
time depends on the previous job type, the 
current job type, and the machine on which 
the job is processed. In addition, jobs (i.e., 
orders) are from different customers. The jobs 
from primary customers have strict due-date 
constraints. The objective is to minimize the 
total weighted flow time. 
 This type of scheduling problems is 
NP-hard. When dealing with a large instance 
encountered in industry, it may not be possible 
to obtain an optimal solution in a reasonable 
time. In this research, a modified SWPT and 
improvement procedures are applied to 
generate an initial schedule. Three effective 
heuristics, two based on record-to-record 
travel and one based on random descent 
heuristic, are developed to improve the initial 
schedule. Computational characteristics of the 
proposed heuristics are evaluated through 
extensive experiments by using a large-scale 
data set revised from the literature. 
Computational results show that the proposed 
heuristics are able to effectively improve the 
initial solutions. 
 
Keywords: record-to-record travel, setup time, 
total weighted completion time, 
total weighted flow time, 
unrelated parallel machine

3. Simulated annealing and tabu lists 
 
The simulated annealing (SA) method and the tabu 
lists are briefly described in this section. 
 
3.1 Simulated annealing method 
 
Basically, SA is a stochastic hill-climbing search 
algorithm. It consists of a sequence of iterations, where 
each iteration changes the current solution to a new 
solution in the neighborhood of the current solution. 
Once a neighborhood solution is created, the 
corresponding change in the cost function is computed 
to determine whether the neighborhood solution is 
accepted as the current solution. If the neighborhood 
solution results in cost reduction, this solution is 
directly taken as the current solution. Otherwise, it is 
accepted according to Metropolis’s criterion [15]. Three 
parameters need to be specified when applying SA: the 
initial temperature; the temperature cooling rate; and 
the number of iterations to be performed at each 
temperature stage (i.e., the Markov chain length). 
 
3.2 Tabu lists 
 
Since moves not results in improvements may be 
accepted in SA, it is possible to return to already 
obtained solutions. Hence, a tabu list from the tabu 
search [16] is used to overcome the this dilemma. This 
tabu list store attributes that identify specific moves as 
tabu in the later search. By giving an appropriate tabu 
size, the tabu list may effectively prevent obtaining 
cycling solutions. In this paper, tabu lists are used to 
store hub sets searched.

5.5 Heuristic CHLP_ML 
 
Heuristic CHLP_ML is now outlined as follows:  
Step 0. Set the initial temperature (T), Markov chain length, 
tabu size, and stopping criterion. 
Step 1. Set SIGNAL=0 (i.e., to indicate it is a pilot run for 
determining the number of hubs), p=|S| (i.e., to set 
the initial number of hubs to be the pseudo-lower 
bound discussed in Section 4.2.1), and the overall 
cost to be infinite for p=|S|-1. 
Step 2. Generate the initial hub locations and perform the 
procedures (i.e., the capacitated shortest-path 
procedure and improvement procedure) for 
determining the path for each origin-destination pair. 
Step 3. Perform the single-exchange procedure and the 
procedures for determining the path for each 
origin-destination pair. If the solution is improved, 
accept this solution. Otherwise, use Metropolis’s 
criterion [15] to determine whether this solution is 
accepted. If Markov chain length is reached, 
proceed to step 4. 
Step 4. If SIGNAL=0, update the temperature. If the stop 
criterion for a pilot run is not yet reached, return to 
step 3. 
Step 5. If SIGNAL=1, proceed to step 6. If p <|S|, proceed to 
step 5-2. 
Step 5-1. If the resulted reduction of the transportation 
cost is greater than the increase in fixed cost, set 
p=p+1, reinitialize T, and return to step 2 (i.e., to 
increase the number of hubs for another pilot 
run). If p=|S|+1, set p=|S|-1, reinitialize T, and 
return to step 2 (i.e., to decrease the number of 
hubs for another pilot run). Otherwise, set 
SIGNAL=1 (i.e., the number of hubs has been 
determined), p=p-1, and proceed to step 6 for 
refinement run. 
Step 5-2. If the resulted increase in the transportation 
cost is greater than the decrease in fixed cost, set 
p=p+1, SIGNAL=1 (i.e., the number of hubs has 
been determined), and proceed to step 6 for 
refinement run. Otherwise, set p=p-1, reinitialize 
T, and return to step 2 (i.e., to decrease the 
number of hubs for another pilot run). 
Step 6. Update the temperature. If the stopping criterion is 
reached, stop the whole procedure.  Otherwise, 
return to Step 3. 
 
5.6 Random descent heuristic 
 
The initial solution of RDH was generated by randomly 
selecting 40% to 80% of the nodes to be hubs. The capacity 
level of each hub is randomly determined. Each non-hub 
was allocated randomly to h hubs and the capacitated 
shortest-path procedure described in section 5.4.1 was 
applied to determine the path for each origin-destination pair. 
RDH then applied five types of transitions to generate 
neighborhood solutions: 
(1) AddHub: Set a randomly selected non-hub to be a hub 
and the capacity level of the hub is randomly 
determined. 
(2) DeleteHub: Set a randomly selected hub to be a non-hub. 
The non-hubs originally allocated to this hub were 
randomly reallocated. 
(3) ReallocateNode: Reallocate a randomly selected 
non-hub from one hub to another hub. 
(4) RelocateHub: Exchange a randomly selected hub with a 
randomly selected non-hub and the capacity level of the 
hub is randomly determined. 
(5) ResetLevel: Reset the capacity level of a randomly 
selected hub to another level. 
After transition (1)-(5) was performed, the paths for the 
related origin-destination pairs were determined by applying 
the capacitated shortest-path procedure. The algorithm for 
the RDH is stated as follows:

6. Computational experiments 
 
In this section, the AP data set from the literature is 
modified to evaluate the computational characteristics of 
CHLP_ML. The number of hubs that each non-hub can be 
allocated (i.e., h) was set to be 2 and 3, respectively. The 
performance of CHLP_ML is compared with the optimal 
solutions obtained by Cplex and the RDH. Both heuristics 
were coded in C++. 
 
6.1 Data set 
 
The AP data set was introduced by Ernst and 
Krishnamoorthy [20]. The flow was not symmetric (i.e., Wij 
≠ Wji) and Wii ≠ 0. The numbers of nodes were set at 10, 20, 
25, 40, and 50, respectively. For each problem size, two 
types of fixed costs were considered: tight (T) and loose (L). 
For the tight type, the fixed costs were higher for the nodes 
with larger total flows. Two types of capacities were also 
considered: tight (T) and loose (L), which stand for tightly 
and loosely capacitated problems. In order to use this data 
set to test this type of hub location problems, the unit 
collection cost was set to be 5.5, the unit transportation cost 
on the inter-hub links was set to be 3.0, the unit distribution 
cost was set to be 4.5. The fixed costs and the capacity for 
node are then set at three levels.

3. Simulated annealing and tabu lists
The simulated annealing (SA) method and the tabu
lists are briefly described in this section.
3.1 Simulated annealing method
Basically, SA is a stochastic hill-climbing search
algorithm. It consists of a sequence of iterations, where
each iteration changes the current solution to a new
solution in the neighborhood of the current solution.
Once a neighborhood solution is created, the
corresponding change in the cost function is computed
to determine whether the neighborhood solution is
accepted as the current solution. If the neighborhood
solution results in cost reduction, this solution is
directly taken as the current solution. Otherwise, it is
accepted according to Metropolis’s criterion [15]. Three
parameters need to be specified when applying SA: the
initial temperature; the temperature cooling rate; and
the number of iterations to be performed at each
temperature stage (i.e., the Markov chain length).
3.2 Tabu lists
Since moves not results in improvements may be
accepted in SA, it is possible to return to already
obtained solutions. Hence, a tabu list from the tabu
search [16] is used to overcome the this dilemma. This
tabu list store attributes that identify specific moves as
tabu in the later search. By giving an appropriate tabu
size, the tabu list may effectively prevent obtaining
cycling solutions. In this paper, tabu lists are used to
store hub sets searched.

5.6 Random descent heuristic
The initial solution of RDH was generated by randomly
selecting 40% to 80% of the nodes to be hubs. The capacity
level of each hub is randomly determined. Each non-hub
was allocated randomly to h hubs and the capacitated
shortest-path procedure described in section 5.4.1 was
applied to determine the path for each origin-destination pair.
RDH then applied five types of transitions to generate
neighborhood solutions:
(1) AddHub: Set a randomly selected non-hub to be a hub
and the capacity level of the hub is randomly
determined.
(2) DeleteHub: Set a randomly selected hub to be a non-hub.
The non-hubs originally allocated to this hub were
randomly reallocated.
(3) ReallocateNode: Reallocate a randomly selected
non-hub from one hub to another hub.
(4) RelocateHub: Exchange a randomly selected hub with a
randomly selected non-hub and the capacity level of the
hub is randomly determined.
(5) ResetLevel: Reset the capacity level of a randomly
selected hub to another level.
After transition (1)-(5) was performed, the paths for the
related origin-destination pairs were determined by applying
the capacitated shortest-path procedure. The algorithm for
the RDH is stated as follows:

6. Computational experiments
In this section, the AP data set from the literature is
modified to evaluate the computational characteristics of
CHLP_ML. The number of hubs that each non-hub can be
allocated (i.e., h) was set to be 2 and 3, respectively. The
performance of CHLP_ML is compared with the optimal
solutions obtained by Cplex and the RDH. Both heuristics
were coded in C++.
6.1 Data set
The AP data set was introduced by Ernst and
Krishnamoorthy [20]. The flow was not symmetric (i.e., Wij
≠ Wji) and Wii≠ 0. The numbers of nodes were set at 10, 20,
25, 40, and 50, respectively. For each problem size, two
types of fixed costs were considered: tight (T) and loose (L).
For the tight type, the fixed costs were higher for the nodes
with larger total flows. Two types of capacities were also
considered: tight (T) and loose (L), which stand for tightly
and loosely capacitated problems. In order to use this data
set to test this type of hub location problems, the unit
collection cost was set to be 5.5, the unit transportation cost
on the inter-hub links was set to be 3.0, the unit distribution
cost was set to be 4.5. The fixed costs and the capacity for
node are then set at three levels.

在企業的實務運作中，常會依客戶的屬性(例如規模大小或願意成為夥伴的程度等)，將客
戶分為重要客戶與次要客戶。當產能無法滿足所有訂單需求時，企業會先滿足重要客戶的
訂單需求，使重要客戶的訂單都能如期交貨。生產排程在此決策中扮演著關鍵的角色。好
的生產排程不但能滿足重要客戶的訂單需求，並且能降低製造成本。本研究針對非相關平
行機器排程作探討，在考量整備時間是族群順序相依的情況下，使工作的總加權完工時間
最小化。本研究探討的這類型排程問題，是 NP-hard 問題，但當處理實務的大問題時，通
常無法在合理時間內求得最佳解。因此較實務的作法，是發展有效能的演算法。本研究修
改最短加權加工時間法則來產生初始解，並發展改善程序來改善不可行解及初始解，以獲
得好的初始可行解，之後再分別使用紀錄更新法與隨機降坡法來改善初始可行解，使能夠
在合理時間內求得最佳或近似最佳解，以方便業界應用。而為測試本研究之演算法的效
能，我們修改文獻的問題來作測試，以評估演算法的品質和效率。 
這類型的排程問題，在實務的生產環境中經常可見，例如在非相關平行機器組成的彈性製
造系統中，每個工作必須在非相關平行機器上進行一道加工程序，且在加工不同類型的工
作中間，機器必須進行順序相依的整備。射出成型部門亦常使用多種不同的機器去製造各
式各樣的產品，而在製造不同類型的產品間，也需要進行更換模具的整備時間。此外，半
導體及 LCD 的瓶頸製程亦常使用多部功能相似的機器去生產不同的產品。因此發展有效的
解法來求解此類型的排程問題，對提升客戶滿意度及企業的競爭力，會有極大的助益。而
