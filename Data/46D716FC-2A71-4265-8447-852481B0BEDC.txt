3. 證據分析（Evidence Analysis）：此階段分析與犯罪有關的數位證據，例如 Skype或

4. 證據呈現（Evidence Presentation）：檔案刪除後可能無法徹底還原，且電腦與網路
犯罪者經常會想方法隱藏犯罪行為，甚至採取反鑑識(anti-forensics)或加密
(encryption)技術，所以鑑識分析的結果包含不確定性(uncertainty)。此呈現階段
提出鑑識分析後的數位證據，也包含說明文件。 
 
在傳統的犯罪案件中，鑑識人員所收集的證物都是實體且可視的，但數位鑑識所取得的
證據皆為電磁記錄，電磁記錄可能在電源拔除或隨著時間的流逝而使得數位證據發生變動
(evidence dynamics)[7]。數位證據具有容易複製與修改、原始作者不易確定、不易證明
其來源等特質，而傳統實體證據若遭破壞、損毀，幾乎是不可能回復成原狀，但數位資料
可能透過軟硬體設備將資料修復還原。然而不同的還原與解讀工具可能產生相異的數位資
料，易發生不確定性的問題，也連同影響了數位據證的可靠性。因此，一個具有公信力的
鑑 識 工 具 與 採 樣 的 方 法 ， 是 數 位 證 據 能 否 被 法 官 採 信 的 關 鍵 ( 參 見
http://www.cftt.nist.gov/)。因為數位證據具有以上這些特性，因此在數位鑑識的過程
中，鑑識人員必須盡量在不改變或破壞證物情況下取得原始證物，確實所收集的數位證據
來自扣押的證物及在不改變證物情況下進行鑑識分析，以確定證據的有效性。 
 
本三年研究計畫針對電腦與手機鑑識系統進行設計與開發實作[2]，我們提昇鑑識工
具的功能，使鑑識工作的進行更加可靠、迅速。同時，我們將鑑識工具軟體本土化，提供
中文化操作界面的鑑識工具，進行電腦與手機犯罪證據的蒐集、記錄、保存以及事件鑑識
分析。

壹、數位證據的定義 
數位鑑識與傳統鑑識存在許多的相異之處，在傳統的犯罪案件中，鑑識人員所收集的
證物都是實體物質且可視的，但數位鑑識所取得的證據皆為電磁記錄，電磁記錄可能在電
源拔除或隨著時間的流逝而消失。 
國外學者 Casey在「Digital Evidence and Computer Crime」一書中 [6]，提出所謂
數位證據是任何使用電腦傳輸或儲存的電磁記錄，包含：文字、影像、圖片等類型之數位
資料，可用於證明或可用來表明犯罪者與受害者之間重要關聯。NIJ 美國司法機構的電子
犯罪現場調查指南[17]提出數位證據為儲存、接收或傳輸於扣押電子設備有證據效力的資
料或訊息，涵蓋影音、圖像、數字等等檔案格式。 
貳、數位證據的特性 
數位證據非實體物質，其具備下列特性，複製修改容易、不易證實完整性及無法以直
覺直接理解等性質[8]，數位證據不僅限使用者存放於電子設備中之證據，更包含駭客攻擊
和入侵證據。因為數位證據具有以上這些特性，因此在電腦鑑識的過程中，為了使數位證
據具有法律效力，鑑識人員必須在不改變或破壞證物情況下取得原始證物，證明所抽取的
證物來自扣押的證物及在不改變證物情況下進行鑑識分析，以確定證據的證明力。 
參、數位鑑識的模式 
數位鑑識可以區分為 Live Forensics 以及 Dead Forensics  [3]。其中，Live  
Forensics發生於可疑系統正在運作中的狀態下被分析，而 Dead Forensics 則使用專用之
分析系統於可疑系統已被關機狀態下去檢查系統中的資料。現在許多數位鑑識的研究使用
dead-analysis，但這種方法可能會因機器關機或移除連接而使資料遺失。對鑑識分析來說，
揮發性資料的蒐集是非常重要的。因揮發性資料可能包括硬體資訊、已安裝的軟體套件以
及所有程序的狀態等。

The Coroner’s Toolkit (TCT)是一廣泛被使用於數位鑑識之開放原始碼工具，是由

(一) Live-analysis系統開發 
所謂 Live-analysis 是指在電腦犯罪現場進行的鑑識工作，其目標可能是電腦主機或
網路系統等，而鑑識人員必須於第一時間收集「揮發性資訊 (Volatile Information )」，
以避免電腦系統關機或移除電源後，揮發性資訊也隨之消失殆盡，而造成日後重建案件原
貌的困難。 
本計畫將系統揮發性資訊工具整合至 Shell Script (log.sh)中並儲存於 USB裝置上，
當鑑識人員到達電腦犯罪現場時，若該主機是開機狀態，則執行 USB裝置中的 Shell Script，

首先鑑識人員必須將扣押之智慧型手機的外部證據進行蒐證，蒐證後執行本鑑識系統，
開始為建立鑑識檔案，本系統透過 JAVA 程式語言、Android API 及 Linux command  實
作，提供鑑識人員紀錄手機上的時間(date)，鑑識系統提供修改輸入鑑識日期時間(為避免
手機上的時間設定錯誤或誤差，藉以比對開機時間用)，接著鑑識人員須輸入鑑識檔案編號、
鑑識人員名稱、備註事項等資訊，如圖 3-9  建立鑑識檔案所示，系統會自動於鑑識 SD 卡
中建立一鑑識資料夾，並以鑑識日期時間加以命名。
