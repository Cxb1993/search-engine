The interface events can be identified and 
tracked in construction project management 
involved general contractor and 
subcontractors to improve the construction 
process and minimize deleterious interface, 
and promote beneficial interface.  This 
paper proposes a new and practical 
methodology to track and manage 
multi-project interface events by using 
activity-based interface maps approach.  
Using activity-based interface maps, users 
can get an overview of previous and current 
interface events in core multi-project areas 
and take appropriate advanced arrangement 
in interface management.  This research is a 
pilot study to develop a generic model using 
network maps for interface management in 
construction industry, analyzing and 
discussing its components, pertinent 
supplementary techniques, and IM 
implementation status for construction 
projects. This study addresses application of 
interface management in the concurrent 
design and construction and proposes a 
Construction Activity-based Interface 
Management (CAIM) concept and system for 
contractors.  The CAIM system is will 
applied in selected cases study of a 
construction building project in Taiwan to 
verify our proposed methodology and 
demonstrate the effectiveness of interface 
management special in multi-project 
environment.  This research integrates 
present IM-related theoretical literature and 
practical implementation methodologies to 
develop with a generic model for IM 
frameworks integrated with multi-project 
management. Project managers and engineers 
can take advantage of interface events 
information flow and interface activities to 
help managers and engineers to clearly whole 
interface events and activities through 
network maps.  By effectively using 
information and web technologies during the 
construction phase of a project, interface 
management can be managed and controlled 
to benefit future projects.  The combined 
results demonstrate that, a CAIM-like system 
can be an effective tool for interface 
managements in construction projects by 
utilizing the activity-based interface 
management approach and web technology.

二、緣由與目的 
 
本研究根據營建專案特性及多專案管理屬
性，並運用網路技術設計建構 Web-based
營建作業式介面管理系統，嘗試改善過去
專案工程介面管理之嚴重問題。本研究以
高科技廠房興建為研究分析對象，針對多
專案高科技同時設計與施工提出專案介面
管理模組，作為建立作業式介面管理系統
之藍圖參考。利用作業式介面管理概念整
合營建專案資訊管理系統，主要的好處在
於能夠有效率地追蹤及管理同時進行多個
工程專案內部及外部相關之介面內容協
調，有效地追蹤及因應工程專案最新改
變。多專案營建介面管理系統主要提供下
列之主要之目的(1)建構營建工程多專案介
面管理之機制與資訊系統；(2)確實有效掌
握各介面事件處理狀況及訊息分享作為執
行專案介面事件處理追蹤與提昇專案控管
的應變能力；(3)有效將介面管理與專案管
理結合，作為多專案介面整合及協調之功
能提升；(4)提高專案執行生產力與降低錯
誤發生率；(5)整合各專案分工之介面關係
以及提供一個較完整之介面事件之流程與
訊息平台。以目前來說，國內的營建企業
甚少利用多重作業式介面管理模式管理工
程專案，利用介面管理方式進行專案管理
是一種有效的管理方法，不僅可以提升專
案控管效能，甚至可以提前整體專案管
理。本研究將透過相關文獻收集分析及合
作企業訪談，彙整出營建工程在介面管理
之實際需求，模式及預期效益。同時本研
究將以合作之廠商做為系統實例執行對
象，首先對於國內營建多專案同時執行或
同步工程過程進行介面管理需求分析，進
而探討系統應用於合作企業進行工程專案
介面管理可能面臨之問題，並提出適用於
營建企業專案管理之介面管理模式，作為
未來建立營建工程介面管理系統之參考。 
 本研究多專案介面管理方法論的研究
與探討，結合營建業領域之特性，藉由瞭
解多專案介面管理在國內營建業領域之應

Activity-based 介面地圖是本研究所提出一
個新的構想以提昇介面管理之視覺工具。
Activity-based 介面地圖為一介面搜尋之概
念，將介面與專案作業間建立關聯，把介
面利用一種階層性的表達方式進行建立，
讓人們在使用時可依其介面地圖階層及所
提示之關聯性，進而找到相關介面之內
容。Activity-based 介面地圖之作用在於有
效找到專案作業介面事件來源、提供介面
事件的資訊內容、協助介面事件之管理與
處置。Activity-based 介面地圖指的是多專
案、單一專案、作業任務、介面關聯線或
介面事件的地理代表圖。介面地圖可以提
升介面事件分佈的能見度，且支援專案經
理或工程師有效的確認與控管，促使介面
事件負責者將新的介面事件內容，歸類到
相關的類別裡，並促使所屬專案、工作任
務、介面事件、介面負責人等相互連結。 
由於現今並沒有標準或統一之方法來製作
知識地圖，故本研究根據文獻回顧介紹目
前常看到的知識地圖類型。第一種常見知
識 地 圖 的 類 型 也 稱 為 組 織 之 地 圖
(organizational map)，其藉由組織部門間連
接員工之互動。第二種知識地圖之類型係
連接專家或個體之專業技術或知識領域，
在該類型中僅告知使用者組織中所擁有之
技術，對於缺乏或需求之技術則無法涉
獵，本研究將此類知識地圖稱之為指引式
知識地圖。第三種類型係特別與知識領域
有關，用以辨別組織中可用或需要缺乏之
知識，本研究將此類知識地圖稱之為領域
式知識地圖。第四種經常使用之知識地圖
類型係語意式網路(semantic network)，該網
路由節點(知識領域)與連接線(節點間之關
係)所構成，在此類型中可經由連接線上之
說明以瞭解知識之脈絡。 
本研究系統開發工具則是使用網際網
路相關之資訊技術(Information Technology, 
IT)來建構，主要以 HTML (HyperText 
Markup Language)與 JSP (Java Server Pages)
為主、配合 DHTML (Dynamic HTML)與
JavaScript 等技術，只要使用者連上網際網
路便可使用，無地點與時間之限制，如此
協力廠商不但可以減少系統開發或購置軟
體之成本，亦可降低自行開發之風險與降
低資料交換標準之問題。本系統的設計使
用 Java 的 Applet 和 Servlet 程式。Applet 
的全意是指「一種很小的應用程式」，可
附著在網頁(Hyper Text Markup Language 
簡稱 HTML)上，下載到瀏覽器執行。Servlet: 
Servlet 以 Server 端為運算平台，不僅可以
傳送 HTML 的資料，也可以用來開發包含
語音、影像、視訊等多媒體資料，是以
Request/Response 為基礎的 Java 程式。 
本系統架構採三層式架構 (three-tier 
architecture)，包含客戶端(Client)、應用程
式伺服端(Application Server)、資料庫伺服
端(Database Server)。系統運作流程說明如
下：首先，使用者於客戶端使用配備上網
功能之個人電腦執行網路瀏覽器，透過網
際網路連結應用程式伺服端並發出資訊請
求，應用程式伺服端接收到資訊請求後，
會依據使用者帳戶權限及給予條件執行應
用程式，進入資料庫伺服端存取資料，最
後將 HTML 網頁資料回傳呈現於瀏覽器
畫面。系統開發是使用 IT 應用技術與工具
來建構，應用程式伺服端採用 Windows NT 
2003 Server 作業系統、Internet Information 
Service 伺服器軟體與 JSP(Java Server 
Pages)動態網頁開發平台相互配合，在資料
庫伺服端則利用 SQL Server 2005 建構資
料庫。圖 4 顯示 Activity-based 介面管理系
