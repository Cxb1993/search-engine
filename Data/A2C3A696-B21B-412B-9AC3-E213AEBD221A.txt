壹、前言 
1.1計畫背景 
隨著資通技術的快速演進，網際網路已成為每個人生活中不可或缺的一部分。然而網際
網路雖快速便捷，網路安全事件卻也層出不窮。根據統計，每 0.25秒發生一起網路犯罪事件，
每三秒就有一筆個人資料遭竊，上述數據還未包含已在網路上流竄卻還沒被發現的惡意流量。
為了有效監控區域內的資安事件，解決的方法之一即是建置資訊安全營運中心（ Security 
Operation Center）來將入侵警訊統一蒐集並彙整成報表。根據資安業者統計，平均每個月可
偵測到 2億 4 千 5百萬次嘗試惡意攻擊的流量，用傳統的分析方法易因龐大的運算量，使系
統難以負荷而過載。為因應大量且快速變異的惡意攻擊，許多資安防護技術已轉向採用雲端運
算技術來解決海量資料運算的需求，透過快速辨識以及預知新出現的惡意程式，達到保護一般
使用者的目標。  
 
2009年產業科技策略會議以「塑造資安文化、推升資安產值」為主軸，決議未來政府每
年將額外投入至少 5億元，改善資通訊環境安全、提升網路應用安全；每年 4.2 億元發展資
安產業推動計畫，推動重點技術與應用，培育頂尖的研究人才。經濟部與國家科學委員會未來 
4年將配合 iTaiwan 計畫投入 20億元，推動雲端服務（Cloud Service）為主的大型資安重
點應用，打造雲端服務資安重鎮與建置資安虛擬攻防平台，形塑台灣資安運籌的國際地位。 
 
近年來駭客行為模式已逐漸改變，攻擊對象已由廣泛傳播，轉型為針對特定目標或區域，
而且攻擊行為混合多種攻擊技術與手法，不易偵測。其次，惡意程式（Malware）的行為模式
亦從早期製作潛存於資通訊設備中的病毒、木馬程式，轉變成以取代或刪除檔案、阻絕服務
（ DDoS）與癱瘓電腦為目標的惡意程式。惡意程式變種快速，亟需雲端運算進行分析。此外，
匿蹤技術日趨成熟，資安工作由消極的單點掃毒，轉為積極誘捕防禦。因此，目前缺乏網路骨
幹等級的資安防禦架構，資安設備僅點狀佈建，無法提供全面性之防禦，雲端運算虛擬化世界
將使資安防駭工作更為複雜。 
 
現階段雲端資訊安全研發面臨諸多瓶頸，諸如：（1）傳統網路安全、系統安全問題對於
雲端運算環境仍造成威脅；（2）資安與雲端運算緊密結合，增加資安難度；（3）雲端環境虛擬
化後衍生安全防禦弱點，尚無有效機制可因應；（4）雲端建置多以應用服務為出發，僅針對特
定服務內容，進行資安防護；（5）雲端虛擬環境大多受限於商業軟體架構，無法有效掌握資安
可能漏洞，增加資安難度；（6）傳統實驗平台受限於硬體資源，無法進行大規模測試、比對。
本計劃嘗試提供部分解決方案，以期降低這些瓶頸造成的傷害。

1.3計畫架構 
1.3.1核心工作一 
多層次資訊架構與資安研究平台維運：主要是將持續維運前期的「資訊安全研究平台」，
以持續提供有效的安全防護、資訊及相關的研究工具給學研界，來研究不斷更新的資安攻
擊手法。 
 
1.3.2核心工作二 
互動式探測檢定與主動式社交工程防禦機制：主要將透過互動式訊問，來提高 bot 可疑
名單的正確性，並且可主動防禦社交工程攻擊，減少使用者被欺騙的機會。 
 
1.3.3核心工作三 
針對 DDoS 攻擊之骨幹網路全域聯防系統：主要將利用骨幹網路的部分 netflow 資訊，
來針對 DDoS 攻擊做分析及阻擋，減緩 DDoS 攻擊的損害。 
 
1.3.4計畫最終目的 
本計畫最終目的主要分成四項：  
（1）維運資安關鍵技術與分析平台，整合多重資安資訊與分析技術，給予學研界資安研發
團隊實用的支援、工具與經驗 
（2）運用雲端安全實驗平台於 Botnet 探測，提高通報名單有效性 
（3）結合骨幹網路與資安技術，提供學研網全面性之安全防禦，有效防制 DDoS 攻擊事件
之擴散 
（4）建置雲端安全實驗平台，提供學研界資安研發 / 攻防之用  
（5）透過合作，進行技術交流與資安人才培育

2.1.2重要成果說明 
1.  建置於台灣學研網路之誘捕網路與偵測系統進行資訊收集，截取真實網路流量與系統日誌，
目前建置超過 6,000 個 IP 位址之偵測系統。 
2.  提供分散式搜尋引擎架構，依需求自訂搜尋條件，搜尋約 20,000 種各種惡意程式樣本，
並在經過個資暱名化處理後，提供下載研究使用。 
3. 採分散式的運作與分析為基礎，結合搜尋引擎的運作機制，針對各偵測點每秒所產生的大
量資料進行索引，運用搜尋指定條件等方式，針對大量的資料進行資訊的探勘，與資訊安
全維運中心完成整合，並將其分析成果與技術，應用於資訊安全維運中心之分析平台。 
4. 開發「暱名化」的處理程序，將原始紀錄中與使用者相關的資訊，包括使用者的 IP 位址、
通訊的行為以及傳遞的明文內容等，先進行隱暱資料的處理。 
5. 針對目前資訊安全維運中心所擁有之資訊進行外洩防護機制與架構的設計，將其研發成果
應用於台灣學研網路之資訊安全維運中心使用。 
6. 可提供學研界資訊安全研究人員可以取得研究所需要惡意程式資料，並透過 Data Set 提
供完整的攻擊態樣，以建立符合真實狀態之研究資料。 
7. 可協助國內產業提供資訊安全相關軟硬體設備研發所需要之環境，並協助驗證資通安全產
品成效。

2.3.2.1 流量資料收集 
NetFlow記錄內容詳盡，並且以 NetFlow方式記錄也不致於使整體記錄資料量太大，因此
使用 NetFlow 分析通訊內容資料，可以同時兼顧效能與分析的精確性。Flow 來源是 TWAREN 研
究網路的所有路由器，設定路由器輸出 NetFlow資料，為了取得更即時的流量數據資料，利用
Perl 程式語言每 5分鐘產生一個流量資料檔，資料中每條紀錄代表一個 NetFlow 資料，每台
設備一天平均產生 288個文件檔。 
 
2.3.2.2 匯整分析 
本模組以能夠同時多工執行，提高處理效率與分析速度為前提，利用多執行緒來實現研究
目的，並以開放原始碼的 Linux 系統作為系統設計及開發的平台，以提升系統穩定性，本系統
主要使用具有跨平台特性的 JAVA及相關的 Library來撰寫開發。藉由解析出攻擊告警訊息，
統計時間區間攻擊告警訊息統計紀錄，以提供網管人員攻擊次數時間趨勢。由於 NetFlow的資
料是編成二進位(binary)的形式儲存，而不是以純文字的方式．無法直接看懂裡面的內容，透
過 JAVA開發程式解析取出資料。定時統計告警訊息，比對 Source IP 和 Destination IP ，
並加入 count欄位，以計算彼此連線數， 並根據告警訊息比對 NetFlow取出 NetFlow 記錄內
容佐以提供網管人員更進一步攻擊資料，並根據歷史攻擊資料及 TWAREN安全政策同時參考國
際安全組織 CVE (Common Vulnerability and Exposure) 所提供的弱點資料庫(vulnerability 
database)，來進行調整設定臨界值(Threshold)與其比較過濾，並透過臨界值的變數與攻擊事
件，以判斷告警訊息的真假，即時提供網管人員欵似攻擊資料。由於不同的網管人員可能對各
種攻擊訊息的緊急程度有不同的見解，因此，依據 TWAREN 安全政策及根據攻擊訊息規則的不
同，劃分了 3個不同的緊急等級: 
●緊急(Emergency)：需要立即應變處理事件，立即對網管人員發出告警通知；如可能取得 root 
權限或系統當機之事件。 
●警戒(Alert)：雖未立即發生損害，但具有潛在攻擊性，如 DDos attack。 
●通知(Notification)：不正常的事件，但不一定是攻擊行動，如 ping 型事件。 
透過臨界值的變數與攻擊事件分析判斷，將超過臨界值則會自動處理將攻擊者 IP 位址納入疑
似攻擊名單，使攻擊者 IP暫時無法連線存取。

般資安偵測設備難以應付骨幹大流量封包，通常只能布置在小區域網路的出入口，並不能
全面偵測網路，僅能達到「點」的功用，當 DDoS 攻擊發生時，儘管資安防禦設備能偵測到

核心工作一 :多層次資訊架構與資安研究平台維運 
1.資安研究平台可提供學研界資訊安全研究人員可以取得研究所需要惡意程式資料，並透
過 Data Set 提供完整的攻擊態樣，以建立符合真實狀態之研究資料，更可推廣至產業界，
協助國內產業提供資訊安全相關軟硬體設備研發所需要之環境，並協助驗證資通安全產品
成效 
2.目前資安平台已提供申請資訊安全研發平台之學研單位，可透過平台取得研發偵測機
制、防禦機制以及前瞻研究所需之研究資料，縮短學研單位取得研究資料的時間，提高資
訊安全研發平台的使用效率。未來將持續透過大範圍且分散式的資訊搜集，以提供符合台
灣學研網路真實情況的研究資料，且透過資訊安全知識庫，可建立各種不同類型資料之間
的關聯性，來提供完整且真實的研究資料。並藉由研發成果進行偵測技術與機制的導入，
協助各學研單位將所研發之成果，應用於實際之台灣學研網路環境之中。 
 
核心工作二 :互動式探測檢定與主動式社交工程防禦機制 
1. 透過互動式探測檢定與主動式社交工程防禦機制可有效判別是否有使用者正在使用此
電腦，進而採取不同之因應策略，使用者更可經由客戶端工具，避免已遭受 bot 攻擊之電
腦持續攻擊其他電腦，可抑制 botnet 成長速度。 
2. 此機制未來可與各大 SOC 合作，藉由此系統來加速「通報流程」，並可依各種 bot 不
同攻擊後所產生的特性，如：流量、系統執行碼等等，以更多面向判斷是否遭受 bot 之攻
