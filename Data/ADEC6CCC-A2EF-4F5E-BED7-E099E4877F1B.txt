 
研究摘要(500 字以內)： 
本計畫之主要目的，在於研究利用 Zigbee 無線聯網要求的技術，達成用戶端自動化之應
用功能，其中將利用 Zigbee 晶片，發展週邊之通信介面、資料處理系統、電源系統，而構
成 Zigbee 通訊模組，建構家電設備耗電量之量測，並提供遠端負載控制之電路。本計畫針
對先進讀表基礎建設 (Advanced Metering Infrastructure, AMI)系統，建構一套含智慧型電表
之示範系統，並與共同主持人陳朝順教授所發展之需量反應控制模組整合，並進行現場自
動讀表功能與需量反應功能之驗証。 
本計劃之讀表系統將根據所蒐集之電力參數作負載組成推估，用戶負載模型推估係以所
收集到的所有抽樣用戶之用戶耗電資料為基礎加以推估。電表下載後的資料，經過異常資
料處理可得每小時一筆之耗電量循序檔。以單元平均法依每月同一小時用電量統計加總平
均，推導出各類型抽樣用戶工作日每小時平均用電量，根據各類型用戶每小時負載佔比建
立用戶負載資料庫。本系統另一功能是透過 Zigbee 通訊模式作直接負載控制，此功能可避
免用戶因用電量超過所簽訂之契約容量而被罰錢。 
電力公司為維持供電可靠並降低營運成本，實施配電自動化與負載管理己成為必然之趨
勢。尤其以臺灣近年來電力供應不足，更為當務之急；另一方面，為配合國內外電力公司
民營化與自由化後的競爭環境，而整合應用智慧型電表及雙向通訊等先進科技產品，並建
立結合負載控制、自動讀表、負載調查及電價與用戶資訊傳遞等功能的整合性自動讀表系
統，為達成此任務中不可或缺之技術。本計畫將可推廣自動讀表系統之應用範圍，使配電
系統之分析能達成自動化之目標，並有效改善其準確性，透過 Zigbee 模組提供者（華亨科
技）之合作，將可提升本案實務應用之能力。 
 技術研發成果說明： 
本計畫將運用華亨科技股份有限公司之 Zigbee 通訊單元及電子式電表，進行裝置及測
試。整個系統的設計包含了 ZigBee 通訊單元、數位電表單元、及負載控制單元、電源電路
單元以及資料處理單元。其中 ZigBee 在這整個系統擔任的角色是資料的傳輸，包括從數位
電表接收到的電力參數，例如電壓、電流，實功率等，以及命令碼的傳輸。數位電表單元
的主要功能為量測各家電的電力參數值，而負載控制單元，則是透過輔助電驛進行直接負
載控制，為了避免卸載及復載產生火花，將配合零交越電路，電源供應器單元則是提供整
個系統設計的供電來源。其測試項目包含：通訊性能、用戶負載組成及負載控制等各項功
能測試，以培養系統建置及操作經驗，並設計一友善之人機介面，收集各個電表之電力資
訊加以顯示及完成資料庫建置。

整體自動化讀表系統架構圖如圖一所示，為一多階層網路系統（multi-level network）主
要包含電子式電表、Zigbee 通訊模組、電能管理系統用戶端末單元、伺服主機及網路遠端

並建立電力參數的資料庫；並經由網路卡與網際網路連線至遠端電腦，已完成網際網路遠
端監控電表或負載以達成電力監控之目的。

圖三 智慧型電表架構 
本計畫之智慧型電表的電能統計運算晶片將使用由 ANALOG DEVICE 公司所研發的
ADE7763 運算晶片。ADE7763 是配有串列埠界面和脈衝輸出的電能統計運算晶片，它包含
兩階調變類比轉換器、數位積分器、基準電路、溫度感測器和信號處理需要執行精準的數
位電能量測校準，以及在電流和電壓頻道上 RMS 和峰值測量。 
ADE7763 可以與各種電流感測器連接，包括低電阻分流器和電流變壓器。此外，在晶片
內部的數位積分器提供與電流感測器直接連接。這直接連接無需外部類比積分器，而且這
解決方案提供了長期的穩定，還有電流和電壓通道之間精確的相位匹配。要將積分器打開
和關掉的時候，需要選擇電流感測器。針對電力參數量測之部分，主要是透過電能運算晶
片進行負載耗能之計算。其中，所計算出之各項電力參數，諸如電壓、電流、實功率、虛
功率、視在功率、功率因數、頻率與用電度數等，則必須透過取樣電流與電壓信號始得運
算出。 
四、 負載控制單元 
針對電熱水器或抽水馬達等負載之控制，即可直接透過輔助電驛進行直接負載控制時，
其負載電流相當大，若在控制時沒有考慮其啟閉之時間點，容易因負載電流在控制接點兩
端產生電弧造成接點壽命降低，因此在進行此類型負載控制時，其啟閉之控制技術將顯得
相當重要。針對此一控制技術，設計零交越偵測電路，並在電流於零交越時，啟閉負載，
如此可延長接點之壽命。 
此一負載控制模組，整合各種通訊模組介面，可提供 ZigBee 無線通訊之方式，進行遠
端負載之控制。當通訊模組接收到卸載指令時，則微處理器開始偵測電壓及電流零交越信
號，並於零交越點時進行卸載；同理，當通訊模組接收到控制指令時，則微處理器同樣地
也開始偵測零交越信號以進行控制之動作。架構如圖四所示。 
ASP 的內建 Request 物件主要之功能為取得從用戶端傳回資料給 Web Server、Request 物件
主要之功能為重送資料給用戶端瀏覽器：ADO 為 ActiveX 伺服器元件，適用於 Internet 的
資料庫存取物件群，若執行中的 Script 程式使用了 ADO 物件，則 Web Server 會根據 ADO
物件所設定的參數來啟動對應的 ODBC 驅動程式，而啟動 ODBC 驅動程式以後，程式可以
直接利用 ADO 物件來存取資料庫，或藉由 ADO 物件來下達 SQL 指令，進而達到存取資
料庫的目的。本文採用資料庫 Microsoft Access，使用 ASP 透過 ODBC 結合資料庫，開發
開放式網際網路資料庫，目前僅應用於自動讀表方面，而用戶端使用瀏覽器，即可瀏覽或
存取資料庫。 
六、 需量反應控制模組 
本系統將與共同主持人中山大學陳朝順教授所發展之需量反應控制模組整合，其功能包
含了待機電力管理、用電資料回報及需量反應控制，而主站電腦則有負載狀態、電力參數
及總用電監視。依目前台灣家庭的電表主要是由機械表所構成，對電力資訊系統的建構是
一大阻礙，而且以目前國際上已具有商品化之電表來說，其可量測出之瓦數皆為數瓦以上，
對於家電產品待機電力管理而言，無法滿足其量測之需求，因此本計畫為了滿足其多功能
之需求，所設計出之自動讀表系統將會具有以下功能： 
(1)  即時量測與雙向通訊功能 
(2)  具足夠之記憶體容量，紀錄用電歷史資料 
(3)  具有更精確的量測精度 
(4)  具自動讀表之功能 
(5)  具需量反應功能，根據不同卸載群組執行卸載動作 
(6)  提供使用者介面(User Interface, UI)提供使用者了解用電資訊 
在電力公司執行需量反應時，針對負載類型以及控制方式，將家庭中消費性電子產品加
以分類，例如電視機、錄放影機、電風扇、收音機、音響、洗衣機、溫冰熱開飲機等歸納
為非重要負載；電腦、冰箱為可卸除之重要負載；保全系統為不可卸除之重要負載，此外
也進行上述負載之待機電力量測。電力公司於供電不足時直接卸除用戶不重要之負載，以
降低發電成本；在非重要之負載卸除後未能達到預期之目標可再進行可卸除之重要負載。
本計畫所提出之需量控制策略也將配合時間電價進行家庭用電管理，例如：預設一個單位

 技術特點說明：

動讀表系統，進行通訊測試用戶負載組成推估，及負載控制等目的。本計畫可延伸合作廠
商華亨科技在 Zigbee 系統之能力，而與配電系統分析及配電自動化作整合，因此本計畫將
可提升自動讀表系統之應用面。
