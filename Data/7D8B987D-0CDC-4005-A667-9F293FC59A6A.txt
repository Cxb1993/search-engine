二、 前言 
前瞻無線網路系統如WiMAX[1]、LTE(Long Term Evolution)[1]等，有更高的頻譜效益，
能提供更多樣性的服務及資訊給更多不同的用戶(Clients)。資料廣播(Data broadcasting)使用
於非對稱無線網路系統(Asymmetry Wireless Networks)[2]是一種很有效率的資訊傳遞方法。
在眾多不同的用戶中，對於某些熱門資訊例如交通資訊、氣象、及時新聞發佈、流行音樂
及數位電台廣播等的需求幾乎是相同的，為了減少移動裝置(Mobile Devices，簡稱MD)對這
些資訊重覆提出要求(Request)，進而避免資訊在無線網路下載通道中重複傳送，這些使用
率較高的資訊應週期性的經由基地台(Base Station)的下載通道(Downlink channel)廣播，而
在此基地台內的MD 只需自動的將接收頻道調整至該廣播通道，讀取出其想要的資訊即可，
而較少使用的資訊可以經由上傳通道(Uplink channel)對系統提出要求，再進一步從下載通
道獲得資訊。 
節能(Power Saving)在無線網路環境中是重要議題[3，4]，MD 通常只配備一低電量的
電池，而一般節能的作法，不外乎在啟動模式(Active mode)和睡眠模式(Doze mode)中作適
當切換。MD 處於啟動模式時會監聽廣播通道中正在廣播的訊息是否是所需，由於啟動模
式的耗電量遠大於睡眠模式，因此盡可能使MD 進入睡眠模式，而再最適當時機回到啟動
模式，是節能的不二法則。有二個重要參數用以評估資訊廣播系統的效能。一為調諧時間
(Tuning time)，另一為讀取時間(Access time)[5，6]。調諧時間表示一MD 開始從廣播中尋
找欲接收的訊息至正確的接收完畢所需進入活動狀態的時間，而讀取時間表示MD 開始接
收訊息至正確的收到訊息所需花費的時間。因此，調諧時間經常用來評估使用某一廣播技
術時MD 省電的效能，而讀取時間則用以評估MD 取得一訊息所必頇等待的時間。通常調
諧時間及讀取時間是不可兼得的。

I. Pinwheel Scheduling Algorithm (PSA) 
PSA將無線資源假設為環狀資料結構，依照服務訊息受歡迎程度高低，依據將訊息安
排進廣播環中；如圖二所示，將受歡迎較高的服務訊息，以最接近正多邊形的方式安排。
因為無線裝置(MD)可能在任何時間點存取服務資料，以最接近正多邊形的方式安排服務資
料，可以降低使用者的平均等待時間。 
PSA排程執行流程如下: 
i. 統計廣播服務受歡迎程度；並且依據受歡迎程度高低，分配可用資源個數。 
ii. 對廣播環中的無線資源進行編號。 
iii. 依據受歡迎程度依序選取服務安排進入廣播環中。 
iv. 安排服務資料至演算算式(1)之所有資源位置。 
v. 對尚未使用的無線資源重新編號。 
vi. 重複第三、四、五步驟，直至將所有訊息安排入廣播環中。

五、 結果與討論 
本計畫修改 PSA與 ID2 排程演算法，並將兩項排程演算法應用於 LTE廣播排程；最後
以程式模擬使用者存取由 PSA和 ID2演算法所完成的廣播串列，統計讀取服務的讀取時間
和調諧時間。 
從模擬結果顯示，ID2 演算法在讀取時間以及調諧時間的表現，無論在何種 DS 情境之
下，皆略勝 PSA一籌；但是在讀取熱門訊息所需要的時間，則是 PSA的表現較為優異。因
為 PSA分配較少的無線資源給受歡迎程度較低的服務資料，並且在資料搜尋時優先尋找受
歡迎程度高的服務，導致無線裝置讀取受歡迎程度較低訊息的讀取時間被延長，拖累系統
的整體效能表現。 
另一方面，PSA設計索引資料 Extra Key，雖然消耗的部分的無線資源，可是由模擬結
果得知，Extra Key能有效的改善系統的平均讀取時間和平均調諧時間表現。無線資源的使
用率以及系統整體效能的平衡點，則需要由系統使用者依據實際情況評估。

圖四、e-MBMS 控制訊息架構 
目前 LTE 系統對於 e-MBMS 的控制訊息架構如圖四，
分別包含廣播控制通道(BCCH)、群播控制通道(MCCH)、
與動態排程資訊(DSI)。其中，廣播控制通道是透過系統資
訊區塊#13(SIB13)[4]，由下載分享通道(DL-SH)傳輸；而
動態排程資訊(DSI)與群播通道(MCH)則由系統分配的無線
資源傳輸[3]。無線裝置由系統資訊區塊取得群播控制通道
(MCCH)的位置資訊，經由此資訊擷取群播控制通道的調
變與編碼機制(Modulation and Coding Scheme，MCS)、群
播通道子訊框分配模式(MCH sub-frame Allocation Pattern，
MSAP)等。 
一個單頻網路區域可以同時支援多個廣播群播服務，
而不同的廣播群播服務可能基於其服務的特性，有不同的
服務品質(Quality of Service, QoS)、區塊錯誤率(Block Error  
Rate, BLER)等需求。因此一個單頻網路區域可能會有多個
群播通道(MCH)；系統將有相同服務品質(QoS)、區塊錯
誤率(BLER)需求的服務安排在同一組群播通道(MCH)中，
不同的群播通道使用不同的調變編碼機制(MCS)避免資源

II.  THE CONSTRUCTION OF INDEX CHANNEL AND 
DATA CHANNEL 
 
II.I  The structure of index channel and data channel 
 
Figure 1 shows the structure of the index channel 
and data channel. Each slot in index channel consists of Key, 
Extra Key, Forward, Backward and Message fields. Tdata 
and Tindex  are respectively the time required for mobile de-
vices to read the message content and the index information, 
The function of each column is explained below: 
 Key field: record the key of to-be-broadcasted mes-
sage. 
 Extra Key field: Extra Key field records the keys of 
those messages which index information follow in the 
next one, next two and next B slot 
 Backward field: record the distance from the broad-
casting index slot to the nearest index slot of most 
popular message.  
 Forward field: record the distance from the broad-
casting index slot to the nearest index slot of next 
popular message. 
 Message field: record the distance from the slot of 
index information of message i in the index channel 
to the nearest data slot of message i in the data chan-
nel. 
If the broadcasting index information is recognized 
as desirable by extracting the key field, the location of de-
sired message can be found in the data channel by extracting 
the content of the message field; otherwise, the mobile de-
vice refers to the key1, key2,…,keyB in the extra key field to 
determine whether the index information of desired message 
is allocated in one of the next B slots. If the mobile device 
cannot locate the desired message in the extra key field, it 
refers to the next field that contains the extra information 
and subsequently switches into the doze mode, and the exact 
time slots in which it must return to the active mode for the 
next reception is already determined. After matching the key, 
mobile device relies on the record of message field, switches 
into doze mode then returns to active mode in exact time 
slot to receive message from data channel. By referring to 
these fields, mobile devices locate the desired messages 
quickly by using the WIA. This strategy provides short re-
sponse and active times for mobile devices to receive popu-
lar messages. Although mobile devices use more time to 
receive unpopular messages, according to the index infor-
mation, it consumes less energy because it remains in the 
doze mode during most of the access period. The design of 
the extra key field, in which the keys of these messages in 
the next slots are prearranged, improves mean active time 
and response time, especially in searching for unpopular 
messages.

The manner in which a mobile device receives a de-
sired message is discussed as follows: after synchronizing 
with the system clock, a mobile device extracts the key and 
extra key fields (key, key1, key2,…,keyB) of the broadcasting 
index information. If key is not the desired message, the 
mobile device compares the key of desired message with 
the key1, key2,…,keyB in the extra key field to determine 
whether the index information of desired message was allo-
cated in one of the next B slots. If the mobile device cannot 
locate the desired message in the extra key field, it reads the 
backward field that records the distance from qi to the near-
est q0 (the most popular message must be searched for next), 
it subsequently enters the doze mode and begins counting 
according to the content of the backward field and returns 
to the active mode when the broadcast of q0 begins. If key 
of q0 and the keys in the extra key field are not the desired 
messages, the mobile device returns to the doze mode and 
subsequently acquires the nearest q1 (according to the for-
ward field), which is the next most popular message. Even-
tually, qi may be located if it is broadcasted by the base 
station, mobile device subsequently enters the doze mode 
and begins counting according to the content of the mes-
sage field of qi and returns to the data channel to acquire 
message when the broadcast of di begins.

V.  CONCLUSION 
 
In this study, we proposed a WIA method to improve 
the performance of a data broadcasting system. The WIA 
method provides a new scheduling technology for the 
broadcast system and a new information search method for 
mobile devices. By establishing the index channel based on 
the windmill concept, the WIA uses special indexing fields 
in each time slot to guide mobile devices to rapidly deter-
mine the nearest position of the desired message in the data 
channel. The design of the data channel also conducts 
scheduling according to the windmill concept. Popular 
messages are uniformly broadcasted in the data channel at 
higher frequencies. This design can attain a minimal mean 
response time and mean active time  
We compared the WIA, PSA, and ID2 methods. The 
numerical analysis results indicated that when accessed 
messages of various types possess skewed access character-
istics, the proposed WIA method achieves a superior per-
formance regarding the mean response time compared to the 
WSA and ID2 methods. The WIA performance regarding 
the mean active time is also prominent.

MECHANISM TO IMPROVE THE PERFORMANCE OF WIRELESS

and GUO-CHENG LI

Abstract: 
In this study, we designed a data broadcasting method 
called a windmill indexing algorithm (WIA) This WIA pro-
vides new scheduling technology for broadcast systems and a 
new information searching method for mobile devices that 
simultaneously conserves mobile device energy and maintains 
favorable access latency of broadcast messages. The WIA con-
structs index and data channels based on the windmill concept, 
using the index channel to guide mobile devices to rapidly 
locate the nearest position of the desired message in the data 
channel. Consequently, a minimal mean response time and 
mean active time can be attained. The results of numerical 
analysis verify that when accessed messages possess skewed 
access characteristics, the proposed WIA method performs 
exceptionally.  
 
Keywords: 
Mobile Computing; Wireless Broadcast System; Active 
Time, Response Time,  Energy Saving 
 
I.   INTRODUCTION  
 
To reduce the repeated transmission of popular infor-
mation in wireless network download channels, where lim-
ited wireless bandwidth resources are provided, data broad-
casting is an efficient information transmission method [1]. 
Recently, the research focus of academia and the industry 
has been to simultaneously pursue the quality of service 
(QoS) provided to users and energy conservation goals [2]. 
However, the balance between these aims is not easily at-
tained. To provide increasing multimedia broad-
cast/multicast services (MBMSs), the international standard 
organization, that is, the 3
rd
 Generation Partnership Project 
(3GPP), modified Long-Term Evolution (LTE) Release 9 
documents to further support evolved multimedia broad-
cast/multicast services (e-MBMS) [3]. 
The metrics of the response time and active time are 
often used to evaluate the performance of broadcast sys-
tems [4]. Response time refers to the speed with which a 
mobile device accesses the requested data. It represents the 
responsiveness of the system. Conversely, active time refers 
to the duration for which the mobile device remains active. 
It measures the energy that is consumed by the mobile de-
vice in the active mode. An efficient broadcast system must 
achieve both low response time and low active time. Gen-
erally, active time may be reduced by air indexing [5]. The 
basic concept is to interleave the index information with the 
data in the broadcast schedule, to assist the mobile device in 
locating the data. By following the links in the index struc-
ture, the mobile device alternates between the active and 
doze modes until the requested data arrive. However, most 
air indexing schemes were designed for flat broadcasting, in 
which all data items are broadcast at the same frequency [6, 
7]. This reduces responsiveness when mobile device ac-
cesses are not uniformly distributed among data items.  
Non-uniform access (or skewed access) exists in data 
broadcast systems. Uniformly distributing varying broad-
cast information wastes resources, which not only hinders 
mobile devices from rapidly obtaining popular information, 
but also leaves non-popular information idle and continu-
ously broadcasting in the broadcast channel. In our previous 
study, we developed a windmill scheduling algorithm 
(WSA) [10].WSA exploited a small amount of extra infor-
mation to assist a mobile device in locating its desired mes-
sage was placed in front of each broadcast message that 
was disseminated in a specific time slot. However, this al-
gorithm had a significant disadvantage, namely, the large-
scale repeated broadcasting of popular messages. Although 
the WSA increased energy conservation and message ac-
cess latency performance, it reduced the channel use effi-
ciency. When message types increase, employing the WSA 
method rapidly increases the mean access time. Thus, we 
must increase the number of broadcast channels to mitigate 
the rapid decline in performance.  
In this study, we propose a windmill indexing algo-
rithm (WIA). The WIA method establishes an index

II.II  The winwheel index algorithm (WIA) 
 
This section focuses on using WIA, the proposed al-
gorithm, to organize messages and their index information 
in data channel and index channel, respectively, in the most 
symmetrical manner. As shown in Fig. 2, the index channel 
and the data channel are considered two slotted rings. When 
using the WIA method, the index information and message 
content of various message types can be uniformly distrib-
uted to the index and data channels and formed into various 
regular polygons based on popularity levels. Messages with 
higher popularity can form regular polygons with more 
sides. A mobile device may acquire a desired message at 
any time, and therefore, the mean number of waiting slots is 
minimized if the broadcasted messages form a regular pol-
ygon in the ring. 
Consider an index channel with C time slots, in which 
the index information of one type of message must be 
broadcasted n times; by using WIA, the index information 
form a regular n-sided polygon. To broadcast K types of 
index information (C0, C1, C2, …, CK–1) in C time slots, the 
WIA is used to form polygons (C0, C1, C2, …, CK–1) that are 
as regular as possible. If C is sufficiently large, each poly-
gon is a regular polygon. Consequently, the index infor-
mation that are placed in the time slots are disseminated 
cyclically, quantitatively, and symmetrically by the base 
station.

IV.  Numerical results 
 
 To evaluate the performance of the proposed 
method regarding energy conservation and message access 
latency, we used numerical analysis to compare the follow-
ing three methods: WIA, PSA [8], and the index tree with 
two fixed fanouts (ID2) [9]. To conduct a fair comparison 
of these three methods regarding the change of the mean 
active time and mean response time with increases in the 
number of message types, we assumed that these three 
methods possessed the same broadcast cycle and number of 
broadcast channels. Because the WIA and ID2 methods 
both had an index channel and a data channel, we set the 
broadcast channel number of the PSA as 2. We hypothe-
sized Tindex (the slot time in the index channel) as 1 milli-
second and Tdata (the slot time in the data channel) as 5 or 
10 millisecond, enabling the index channel and the data 
channel to possess sufficient and equivalent number of time 
slots to actualize the scheduling method of WIA, PSA, and 
ID2.  
 The proposed mechanism exploits the skewed-
access characteristic in allocating various amount of index 
time slots to various types of messages, and uses the WIA 
to improve energy consumption and response time. A large 
degree of scenes (DS) implies that the access frequency 
markedly varies between various message types. In the nu-
merical analysis, DS was defined as equal to 1 to 3 to repre-
sent the access frequency among the disseminated messages 
(that is, A0, A1, A2, … are equal), linear (for example, A0, A1, 
A2, A3, … are 1, 2, 3, 4, …), and exponential (for example., 
A0, A1, A2, A3, … are 1, 2, 4, 8, …), respectively.

conjunction with the DS. The WIA and PSA mechanisms
