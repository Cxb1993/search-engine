研究摘要 
晶城環保服務有限公司基於過去清潔公司使用人工的方式來進行清潔工作指
派，無法確保每次人工的安排方式是否最佳，或是否符合成本利益。因此希望能藉本
工作流程指派系統，達成排程效率的提升，並減少不必要的金錢及時間花費，確保每
次工作指派的最佳化。

1-2 研究目的 
提昇企業效能不是只有電腦軟硬體間的架設，真正專業的是能了解企業所需，以
及為其量身訂做解決方案，還要有整合原有設備，為企業成本做到有效利用及節省的
最佳化能力。 
在派工系統使用之前，因需先建立顧客資料庫、清潔人員資料庫、清潔用品資料庫、
員工標準工作時間等，以上這些類別因先建立成電腦化檔案，以便未來派工系統完成
時，可以立即使用於實務工作上。 
在商業市場中，中小企業因規模小，個體而言較為弱勢，因此需要企業及政府間
的支持與照顧，以振興營業水準及競爭能力，並且與較大型的企業有較良好互動的產
業分工體系。 
因在產生合約前，有許多的報價單以及報價人員，在無記錄時易混淆，人事上也不易
管理，人工處理也易疏失。 
提供廠商與業主之間溝通平台 WEB 介面為較可行方式，業主端無須另行安裝軟體，可
直接由瀏覽器登入廠商所建立之網站，進行清潔工作之排程、帳務，無須再透過人力從
中斡旋。

影響-決定施工人員的因素： 
1. 人員分配：不同的施工項目，由專門的技術人員處裡。 
2. 人員休假：施工人員月休8天，是依畫假的方式來休息。需規定每一位相同技術
的人員，休假時間盡量不能重複，並參考時間配合度來安排工作。 
3. 經驗值：參考機動項目表，了解有哪些技術人員曾經清潔過相似或同樣的場所，
以便分派有經驗並熟悉此場地的人工作，以提升效率。 
4. 通勤時間：在分派工作時，並非所有的工作地點所需時間都一樣，因此，可能一
天有兩個工作量以上的話，要考慮到下一個施工地點的安排以及需要將交通時間
也考慮在內。 
5. 人員配合度：有時會遇到突發狀況，例如工程突然增加需加班、或大量工作量導
致員工無法休假等，這時需考慮到員工配合度的高低，以便能在派工上隨時應變
突發狀況。 
6. 客戶指定清潔人員：如顧客想指定技術人員來施工，則需在不影響工作安排的情
況下，優先滿足顧客需求。

4.藥劑特性：有些施工，可能需要等待時間。如：除蠟需等待蠟乾。 
5.場地清潔頻率：清潔場所 1個月清潔一次和半年清潔一次，因清潔頻率不同，
乾淨的程度上也會有差異，因此較不頻繁的場所清潔時間會比較長。 
6.隔場的地點：對於非整天的工作場次，要盡量安排鄰近的工作地點作為場次的
銜接，此舉可減少來回車程所耗費的時間。 
7.客戶要求：如：客戶要求施工人員只要男生／女生。如：有哪一個區塊要特別
髒需要處理。 
8.星期五晚間及週末假日案子較多：一般企業星期五晚間及週末假日為非上班時
間，故施工時間安排於此時段的情況為較多。 
9.緊急場次：臨時追加且有急迫性，如：施工後顧客反應不佳，需要臨時加強的
場次。

4.JDBC(Java Data Base Connectivity)-資料庫連接 
 JDBC 是 Java 資料庫連接 (Java Data Base Connectivity)技術的簡稱，是為各種常用
資料庫提供無縫聯接的技術。JDBC 在 web 和 Internet 應用程式中的作用和 ODBC 在
Windows 系列平臺應用程式中的作用類似。ODBC(OpenData Base Connectivity)，稱為
開放式資料庫互聯技術，是由 Microsoft 公司宣導並得到業界普遍回應的一門資料庫連
接技術。JDBC 現在可以連接的資料庫包括：xbase、Oracle、Sybase、Aceess 以及 Paradox
等。   
   SQL(Structure Query Language)，它是一種標準化的關係型數據庫訪問語言。在 SQL
看來，資料庫就是表的集合，其中包含了行和列。SQL 標準雖然也還處在不斷變革之
中(任何一門標準如果已一成不變了，那只能說明這門學科已經停止發展了)，但其基本
內容相對穩定。JDBC 定義了 Java 語言同 SQL 資料之間的程式設計介面。  
 它為 Java 程式提供了一個統一縫地操作各種資料庫的介面，在編程時，可以不關心
它所要操作的資料庫是哪個廠家的產品，從而提高了軟體的通用性。只要系統上安裝
了正確的驅動器組，JDBC 應用程式就可以訪問其相關的資料庫。  
 
5. CSS(CSascading Style Sheets)- 程式語言 
CSS(Cascading Style Sheet)，使用 CSS 可以讓製作網頁時更輕鬆地調 整整個樣式，
以往都會在網頁中利用字形、大小 、顏色來美化版面，但是這些版面美化標籤(tag)往往
使得整個 HTML 體積變大且不易維護，尤其是一旦要更改時，所費時間往往就形同重新
製作一樣，相當不經濟且沒有效率，如今使用 CSS 之後，便可以達到精簡網頁與創造一
致性的版面的目標，CSS 主要增加了更多的樣式定 義方式來輔助 HTML，以往 HTML
文件的標籤放在不同的網頁或同一網頁的不同位置都必須各自設定其顯示格式，但透過
CS 只要修改定義標籤的樣式則 其它相同標籤的顯現格式都會跟著改變。 
使用人員及設計者都可以使用 CSS 來決定文件的顏色、字體、排版等 顯示特性。CSS
最主要的目的是將文件的結構（用 HTML 或其他相關的語言寫的）與文件的顯示（CSS）
分隔開來。這個分隔有許多好處： 
1. 頁面的可讀性被加強 
2. 頁面的結構更加靈活 
3. 頁面的結構可以更簡便 
     HTML 文件中的每一個 class 或 id 都可以有自己的顯示特徵，而且每一個沒有 id
特性的 HTML 結構也可以有自己的顯示特徵。這些結構有的是 HTML 自己需要的，有
的是專門為 CSS 設置的。
