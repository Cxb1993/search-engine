 II 
Abstract 
At this study, we were created a cloud compute web service for estimate the 
terrain of coast area in Taiwan with Geographic Information System (GIS) concept 
and a variogram model parameter database which was created by analysis the data of 
different sediment properties from the project “The Study of the Local Parameters 
for the Geography Estimate System of the Near-shore Area” (Project No: 
NSC98-2221-E-019-058-MY2). With this web service, we also collect the terrain 
history data in systematically. First, we analysis the data of the terrain of the coast 
area with Try-Error and Least Square Method for verify and calibrate the variogram 
model parameter of database. Second, deploy the web service on the Google App 
Engine (GAE) platform for anyone who needs the service to estimate the terrain or get 
the history data and provide new terrain for update the database. Finally, deploy the 
Application Programming Interface (API) library on internet for the user to download 
the library and develop there estimate method at future.

雲端運算之特點能便利地隨需求透過網路、伺服器、儲存裝置、應用程式
與各類服務存取設定好的共享運算資源。運用管理工作或服務供應商互動模
式，進行快速配置和發佈，雲端模型提升了服務可用性。「雲端運算」的概念
係由 Sun Microsystem (1983)所提出之「The Network is the Computer」，而後 Eric 
Schmidt (1993)提出「網路的速度如果能跟上電腦處理器(CPU)，電腦就可以擴散
到網路上，進而讓網路成為電腦」，這是早期類似「雲端運算」的想法。Eric Schmidt
更於 2006 年提出，「使傳統使用者手邊執行之資料服務與架構，移動到伺服器
上，即稱為“雲端運算”」。德國 Rajkumar Buyya 等(2008)提出雲端伺服器服
務為未來平台的趨勢，但未針對雲端伺服器服務的發展，進行進一步的探討及
說明。Chrisstof Weinhardt 等(2009)以三階層來區分雲端運算，並對供應商提出
計價模式之方式使用。李等(2009)採用開放式服務閘道平台技術(Open Source 
Gateway Initiative, OSGi)，其為標準化及開放性標準，且能跟 Google 所提供的
Android(An Open Handset Alliance Project)平台相容及溝通，讓車載資訊結合行動
裝置創造更多的服務應用情境。鄭和陳(2010)運用雲端運算為基礎，以公路監理
資訊服務平台為研究標的，針對雲端運算做一深入淺出的探討，規劃設計新一
代公路監理資訊服務架構，並進行跨平台、跨網路的資訊服務，達成雲端化的
目標及願景。 
本計畫基於 98 年國科會計畫「區域性海灘地形推估系統建置之研究」(計畫
編號:97WFA2000168)成果，結合雲端運算技術，將研究成果透過網際網路的發
布，發揮雲端運算的優勢，並可達成擴充空間地形資料庫規模，執行成果可做
為日後檢討推估參數及推求區域平衡斷面之資料庫工具。

第三章 雲端應用及系統架構 
3-1 雲端應用 
雲端服務可概分成三部分：軟體即服務(Software as a Service, SaaS)、平台即
服務(Platform as a Service, PaaS)、架構即服務(Infrastructure as a Service, IaaS)，與
四種部屬模式：公用雲、私有雲、社群雲、混合雲；以及五項的基本特徵：高度
彈性，計算服務、隨需應變自助服務、網路使用無所不在、資源匯整，如圖 3-2
所示。 
SaaS：屬於最高層，是一種以租賃形式提供企業使用的應用軟體服務，企業
透過 SaaS 服務平台可以自行設定所需要的功能，相關的資料庫，後
續的軟硬體維護等，都由 SaaS 服務供應商提供。透過多重用戶
(multi-tenancy) 的方式，將完整的應用做為服務隨需提供。 
PaaS：屬於中層，企業可以根據廠商所釋出的開發工具與 API 等，針對自己
的需求客製開發，甚至是開發其他的應用軟體。 
IaaS：屬於最低層，為整個網路中提供了核心計算資源和網絡架構的服務，
將基本儲存與運算功能做為標準化服務提供的一種方式，相較於下列
兩種服務，使用者不能管理或修改任何雲端基礎設施，但能控制操作
系統的選擇儲存空間及部屬的應用。

四、建議 
參與本次國際性學術研討會，瞭解全球不同國家地區有各自面對的問題及
解決問題的思維與新方法，可提升研究及促進工程技術的發展，未來更可為借
鏡。參與國際性研討會，透過問題的討論，不同觀點的腦力激盪與意見交換，
分享彼此研究經驗，是最有效率的學習方式，建議貴會應多多鼓勵國內年輕學
