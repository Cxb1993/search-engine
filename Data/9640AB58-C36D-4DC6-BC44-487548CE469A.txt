研究摘要(500 字以內)： 
 
本計畫擬將結合業界資源規劃開發一套具行動定位技術之校園入口網站系統，包含系
統開發以及初期之內容建立，並將率先於龍華科大內推行系統試用，供全校師生透過行動
電話訂閱相關學習內容及緊急救難時靠手機發出訊號，透過行動電話基地台及本計畫開發
之系統就能回傳定位找到受災者的下落。 
校園入口網站的開發包含三大重點：首要為「行動學習系統設計開發」，此部份需整
合現有行動通訊系統相關技術如：3GPP、GPRS、XHTML、Java Tech、Content Repository 
Management System(CRMS)等相關技術建構一方便且人性化的行動電話學習平台。其二為
「數位學習內容(Content)」，此部份著重於Content 的產生、編輯、製作以便適合所開發之
行動學習系統及滿足使用者學習的需求。其三為LBS(Location-Base Service)是透過一組定位
技術獲得行動終端的位置資訊（如經緯度座標資料），提供給行動用戶本人或他人以及通
信系統，以實現各種與位置相關的業務。

技術研發成果說明： 
 
行動服務內容方面，無線應用已經脫離純粹語音服務，必須與多種數據加值服務、多
媒體服務、行動商務服務等結合。對內容供應商(軟體供應商)而言，面對如此多樣化的上網

而且回應速度很快，通常首次定位只需一秒鐘， 但缺點是定位較不明確。 
A-GPS 輔助參數定位 
第三種方法是採輔助GPS（Assisted-GPS；A-GPS）定位技術，即利用蜂窩網路提供輔
助資訊來幫助衛星作參考點進行定位，這是目前相當受到重視的一項技術方案。A-GPS是
目前唯一通過美國E911規格的定位技術，StrategyAnalytics預計至2008年A-GPS手機出貨量
將超過1億5000萬支。 
A-GPS的定位過程為：A-GPS不僅在手機端接收定位資訊，在系統網路端所加裝的GPS
接收器也會接收定位資訊，並將此資訊傳送至手機，此時手機再同時計算由網路端傳來的
輔助參數資訊與自衛星接收而來的訊號，從而獲得準確的所在位置。 
A-GPS定位技術直接改善了定位的可用性、靈敏性、精確度和定位耗時，具有比小區
ID、E-OTD或TOA/TDOA/OTDOA更高的定位精確度。它能將開機第一次定位時間由GPS
的60 秒縮短為6到8秒，之後每次定位搜尋時間也從40秒縮短為0.1秒，精準度更是大為提
升，從原本500公尺的範圍，準確縮小到5到15公尺。 
雖然手機內需安裝A-GPS晶片才能執行此項功能，但利用輔助參數來提供終端對衛星
的定位跟蹤時所需要的資料，能夠大幅減少終端運算上功耗，最多可以減少終端GPS晶片
60％的功耗。 
混合型定位方案 
由於A-GPS 的主要資料還是來自GPS，當在一些環境中（比如高樓間、室內、地下等）
GPS無法發揮作用時，行動終端就得使用其他的網路定位技術，如Cell ID、AFLT（針對
CDMA的E-OTD）等才能定位。因此，混合型的定位解決方案就是將不同的技術加以整合，
提供一個更為可靠、靈活和穩定的解決方案。 
混合技術最常見的實施模式是將A-GPS與小區ID組合在一起，增強了覆蓋範圍和平均
定位精確度。A-GPS和小區ID的組合還整合了二者具有的漫遊優勢，可用於擁有大量傳統
手機用戶的網路之中， 其中小區ID可用作傳統手機的定位技術，同時也可在A-GPS性能不
佳的環境中使用，如圖2所示。

2. 「行動校園入口網站的開發建置」：此部份需整合現有行動通訊系統相關技術如：
3GPP、GPRS、XHTML、Java Tech、Content Repository Management System (CRMS)…
等相關技術建構一方便且人性化的行動電話學習平台。而網站系統之內容，此部份
著重於Content的產生、編輯、製作以便適合所開發之行動應用系統及滿足使用者獲
得服務的需求。本計劃將透過產學合作邀請業界菁英透過論壇方式整理出數位學習
內容，並以文字、圖片、影像等方式呈現，製作上需考量頻寬、手機上記憶容量之
問題，並完成「內容資料庫管理系統」(Content Repository Management System; 
CRMS)。  
行動校園入口網站系統設計方面，由於 XHTML 模組化之特性，因此 XHTML 可輕易
利用標記標籤(markup tags)將向量圖形、多媒體、數學、電子商務和其他多種內容做結合，
確保網頁內容提供者(Content providers)可於更為廣泛的之平台上，提供更有價值之內容；
此種模組化設計方式亦反映出「一體適用」(one-size-fits-all)之觀念，將不再適用於多樣化
瀏覽器充斥的世界，如多媒體桌上型電腦之瀏覽器網頁，受限於大哥大手機之記憶體，即
無法以相同的方式應用於手機瀏覽器之上。結合使用了 Java 程式語言，利用 Java 跨平台的
特性，使得大部份之手持行動裝置，皆可順利的瀏覽與使用系統所提供之服務。 
行動服務內容方面，無線應用已經脫離純粹語音服務，必須與多種數據加值服務、多
媒體服務、行動商務服務等結合。對內容供應商(軟體供應商)而言，面對如此多樣化的上網
裝置，如何滿足各種裝置的顯示及內容需求而又不增加經營成本，是刻不容緩需要解決的
問題。為解決上述諸多問題，希望透過此一平台的運作，大幅降低內容供應商及軟體供應
商的開發成本，更能增加手持裝置的個人化調適能力。  
伺服端儲存網頁內容及各種手持裝置可用的物件(包括軟體、鈴聲、音樂等)。當手持裝
置連上網路時，伺服端與用戶端雙方經由協定溝通，伺服端可以判斷出手持裝置的能力及
使用者設定等資料，包括軟體版次、支援的網頁格式(HTML, XHTML or iMode cHTML)、
圖檔格式等。再由伺服端進行網頁及圖檔格式轉換(Transcoding)、分頁調整、版面調整等工
作，使得一份來源網頁資料可以根據不同設備的特徵進行調適，降低內容製作與維護的成
本。
