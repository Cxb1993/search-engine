第一章 0B簡介 
 
隨著電腦與網路科技的進步，資訊教育在學校變成一門重要的課程，而資
訊設備的購置、安裝、應用與管理等等也是學校重要工作之一，尤其在病毒肆
虐的網路環境，常常也是學校沈重負擔之一，如何以最低成本建置一套電腦網
路教室，同時還能降低管理負擔，另外還能達到節能效果，節省整體電力消耗，
幫學校節省電費。建置後端伺服器與前端精簡型電腦（Diskless PC）取代傳
統的一般有硬碟的電腦是目前一可取方案，在本計畫之中，精簡型電腦的作業
系統後端伺服務器是採用 DRBL (Diskless Remote Boot in Linux)[1]。 
 
精簡型電腦已經有一些個別建立在 Windows或是 Linux based的系統上，
可是大多是其中單一的系統，或是只是單純開機時安裝系統模擬單機執行，鮮
少有多重系統可供選擇，更難見到能夠管理跨越 Windows與 Linux系統平台的
個人帳號，而進一步達到帳號管理並且分享檔案。 
 
這個整合型計畫，其目的為解決這種精簡型電腦網路教室的建立與管理問
題，讓希望建立這種教室的學校能夠方便地安裝整合 Microsoft Windows 與
Linux based系統的教室環境，並且能用整合的帳號與密碼，將這一類教室的
資源提供給師生，並且能方便達到系統管理、帳號管理、教學支援等等目的。
其中帳號整合系統整合 Windows 與 Linux 帳號，網路檔案系統將整合 SAMBA
與 LDAP，提供各帳號的網路硬碟供給學生在不同教室都可以使用。 
 
本研究主要是在提供一個帳號整合系統，整合多台 Windows Server 與多
台 Linux Server 系統的帳號，學生只要記住一個帳號，不管在哪一間電腦教
室都可以正常登入使用電腦，存取自己帳號下的私有檔案。因為此整合計劃中
主要包括三個分部：(1) 帳號系統的整合，多個帳號系統造成管理者額外負擔，
帳號整合後，學生只需記著一套帳號，就可遊走各電腦教室；(2) 精簡型電腦
本身不配置硬碟：(3) 整合不同 Server 、不同作業系統帳號系統，成為單一
帳號系統，學生只要記住一個帳號資料可以在任何精簡型電腦教室使用，同時
也可以存取自己帳號底下的資料與檔案。

 在帳號管理方面： 
 
對於重度管理者而言，使用 command line 較容易對針對目前所需的需求去做指
令動作的。有任何的問題及需求，亦可直接做判斷。針對大量重複性的工作，亦可
編輯成 script 檔案做執行。如需異地管理的話，亦可以遠端登入，作系統管理工作。 
 
但對於一般的使用者，特別是未來中小學教路教室管理人員，其技術層次未必那
麼高。對於系統規劃、網路設定等技術，不一定熟悉各指令的意義與操作方式。其
指令方式並不人性化。 
 
例如，如需在本系統架構中，在傳統 Linux 要新增使用者，其需要下達的指令步
驟包含： 
建立機器帳號→新增系統帳號→修改帳號群組→新增 LDAP 資料 
→修改 SAMBA 密碼。

2B第四章  總結與未來展望 
 
在這個數位網路上資訊爆炸，電腦元件趨向輕薄短小，行動裝置功能性越來越
強，而數位上網設備趨於個人化、多樣化的時代，我們現階段的嘗試讓學校內的使
用者在使用資訊設備時，能夠不受時間與地點的限制，透過網路存取屬於使用者自
己的帳號與檔案。未來更期望這種校園內的資訊整合使用方式能容納更多樣的使用
者與資訊設備平台。 
 
跨越 Windows與 Linux使用者網路的帳號管理系統是本專案的目標特色。使用
者無論使用一個 Windows或 Linux 精簡型無碟電腦連接上網路，都可以使用單一帳
號登入，方便而安全的存取個人的資訊。未來我們預期這樣的方便性，在使用者族
群上將不僅限於校園內，而將擴充到商業、家庭、與個人；在資訊設備上也將不僅
限於精簡型電腦或智慧型終端機，更將包括一般電腦工作站、手持式行動上網設備、
資訊家電、影音娛樂設備、遊戲機、醫療保健管理設備、安全監控設備、倉儲管理
設備、商業自動化設備、電子化學習設備、整合式資料擷取設備等等。 
 
現在蔚為風潮的低價學生電腦 AMD OLPC(One Laptop per Child )[13]、Intel 
Classmate、或是也將在台灣能夠購置的 ASUS eee PC[14]，更是以連網為主要功能，
卻只提供小容量的固態硬碟 SSD (Solid State Disk) 作為架構，執行 Linux或是
Windows 的類精簡型筆記電腦。預期將在近期普及，大量進入校園後，將會快速提昇
校園中多系統的帳號與檔案系統服務需求。許多與學習相關的網路功能，如網路剪
貼簿、網路筆記本、通訊錄等等，都可以在這種平台上實現。 
 
更多的設備將會以不同的作業系統平台呈現給使用著，如人手一機的行動電話、
遊戲機、個人影音設備等進入校園與家庭中，也是由多樣的作業平台融入個人的數
位生活中，更增加了我們這樣的多平台帳號整合系統的重要性。 
 
除了應用層面之外，技術層面上我們也期望未來能做到自動備份、負載平衡、
與擴充/縮減、虛擬主機等等功能。這些技術上的功能是我們的系統進入被校園或企
業組織大量使用後，將必頇面臨的課題。
