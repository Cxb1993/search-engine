到相當多的運用領域，拉邦符譜應可以用來電腦
動畫人形動作編排領域。本研究的目的，將探討
以拉邦的符譜系統，作為動作編排的基礎，將盡
力保留原拉邦符譜的原則，設計以動作編輯器、
肢體動作產生器、電腦角色等，同時進一步應用
於人型機器人的動作操控可行性研究。 
三、研究目的 
為達到實現逼真生動的人物角色動畫。本研
究之開發的目的有下列四點：(1)拉邦符譜在其舞
蹈藝術領域上有其地位與價值，若能將其數值化
並僅作簡化修正而得到更好的應用方向，將會使
得曾以拉邦符譜所記錄的舞蹈資料得以被快速
重現。以數位方式重現舞譜中所記載的演出，將
具有數位典藏的意涵，應是相當值得進行的研究。
(2)介面需以動作目的為導向，而非資料導向：以
動作目的導向，可以抽象出命令的層次，讓不同
型態的四肢骨骼共用相同的命令，做出相仿的動
作。拉邦符譜本身僅對動作加以闡述，演釋的內
容則由舞者表現，此即符合以動作目的導向為出
發點的設計。(3)資料適應能力：一份動作資料，
必須能在不同的肢節長度與架構中所使用。也就
是動作描述的資料層次不應受不同的肢體所限
制。在一段動作中，必須能夠快速方便的插入另
一個動作描述，或加強細部的描述。(4)介面簡易、
人性化、易讀：文字敘述雖然是一個好的方式，
但經過設計且加以適當理解後的圖形符號，可以
在較短的描述資料當中，傳達更多的資訊。此外
圖形必須適切且易於理解，為有系統的安排，以
達到最佳的效果。 
四、文獻探討 
Labanotation 在西元 1920 年，由歐洲相當
重要的現代舞專家 - Laban Rudolf[9]所設計的
一種記錄肢體動作的系統。在”Labanotation”[1]
一書中所闡述之拉邦記譜方式，是一門科學的符
號設計方法，可由簡單的描述方法到相當複雜的
肢體行為，適用於各種動作描述的領域。拉邦符
譜使用抽象的符號以描述下列的幾種資訊：1.位
移的方向、2.位移的肢幹部位、3.位移的高度、
4.操作此位移的時間。以目前對拉邦符譜的的認
知上，其具有相當能力細節程度，可以表示動畫
與機器人肢體的細節。[7,8,15,16] 
耶魯大學在此其一篇研究報告[10]「Dance: 
A Declaractive Language for the Control of 
Humanoid Robots」中指出 labanotation 有其一定
程度的不明確度，其部份記譜方式與符號系統的
衝突與晦暗不明的原則。耶魯大學在其研究的方
向偏向於捨棄使用拉邦符譜，而另行設計其文字
描述語言「Dance」，這並非排除拉邦的可用性，
而是改採重新設計的一種功能性系統。由於認定
拉邦在其舞蹈藝術領域上有其地位與價值，若能
將其數值化並僅作簡化修正而得到更好的應用
方向，將會使得曾以拉邦符譜所記錄的舞蹈資料
得以被快速重現。 
由於拉邦符譜所記錄的動作資料，可以是細
緻的肢幹角度，亦可以是粗略相對位置資訊。為
了配合此一輸入的彈性，在動作產出的系統，必
須有能力以 Animation blending[6] 的方式，來結
合兩個不同的肢幹姿態之間的差距。另外再如重
心的轉移與肢幹位置的直接指定，則需要反向關
節[2,3,18]的計算來得到關節的角度輸出。 
骨架動畫，是以肢幹、關節所組成的階層架
構，操作其關節角度與肢幹的位置，得到肢幹相
對於根座標系的空間轉換，再用以影響 mesh 的
位置轉換，形成動畫[17]。浙江大學電腦輔助設
計與圖學關鍵實驗室[5,12]便以 Mocap 系統擷
取動作，經過數位化並以人力賦予拉邦符號的對
應，再以資料庫比對方式，人力反覆修飾資料庫
的定義以產生對應於拉邦符號的 Key Pose。以該
方式所產出的 Key Pose 缺點：(1)人力時間成本
過高、(2)動作無法完全對應、(3)邏輯上缺陷 
五、研究方法 
整個系統的組織構想，主要分成介面與資料
轉換兩大類：1.使用者操作介面：主要提供使用
者操作的前端程式。包括有拉邦動作編輯器
[11,13]、3D 動作模擬平台、伺服機操作介面。
2.資料轉換模組：資料轉換模組為將產生的資料
在不同的目的程式間做轉換，以提供其它模組使
用。包括符號轉譯模組、骨架動畫輸出模組、伺
服機對應模組。使用者藉由「拉邦符號編輯器」
編輯拉邦舞譜，產生動作的流程，再經由符號轉
譯，產生動作關鍵姿勢。3D 動作模擬平台則根
據得到的動作資訊，產生連續的動作，以供骨架

轉矩=力場與力臂法向量的點積 x 常數 
其幾何上意義為：力場向量於力臂法向量的
投影長度乘以一常數，由於力場方向不變，力矩
會根據旋轉角度而變化。當力臂法向量與力場方
向垂直時，即力臂已指向力場方向，則力矩為 0，
力臂不再旋轉。因為力矩與角度成正比，只要取
最簡單的公式如： 
角度=前次角度＋力矩 x 時間變化量 
則可得到下一個時間點角度的數值。 
   
圖3：力臂與力矩關係： (左)同方向；(右)不同方向 
為了得到自然的運動軌跡，可以進一步加入
重力的模擬。以力矩分析圖 4 來觀察當一個向前
的力場，施於手腕使手腕得到施力 F 時，沿著下
臂產生分量 Fp，以及垂直於力臂的分量 Fv，重
力施於力臂中心的力為 Fg，上下臂長皆為 R，
Fp 在上臂產生的垂直分量為 Fvu。 
 
圖 4：力矩分析 
所得到的下臂轉矩 Tl，及上臂轉矩 Tu 分別為： 
Tl = R x Fv + 0.5 x R x Fg x cos 
(下臂與水平線夾角) 
Tu = -R x Fvu + 0.5 x R x Fg x cos 
(上臂與水平線夾角) 
得到的力矩，即可由當下的姿勢，計算出終
點得到的 key pose。而過程中力臂的演進，即是
兩個動作之間的軌跡。在多個關節的狀況下，此
法仍然可以得到相當自然的結果。

二、與會心得 
 
ARS Electronica 2010 Festival and Conference 國際學術研討會，與國外相關單位保持密切
的交流與合作，瞭解目前最新科技研究的成果，有助於我們在互動科技相關的領域與國際研
