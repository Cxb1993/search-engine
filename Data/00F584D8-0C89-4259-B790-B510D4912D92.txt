Abstract 
As an emerging model of communication and computation, Peer-to-peer systems are under intensive 
study. In unstructured P2P systems, queries are flooded among peers. Each peer acts as both a client who 
requests information and services and a server who produces and/or provides information and services. The 
flooding-based search mechanisms cause a large volume of unnecessary traffic. It brings great stress in the 
Internet infrastructure. Moreover, various forms of abuse and attack have been observed in current P2P 
system. The most common ones are free loaders and denial-of-service (DOS) attacks. Existing P2P search 
algorithms generally target either the performance objective of improving search quality from a client’s 
perspective, or the objective of reducing search cost from an Internet management perspective. Most 
existing work of designing and optimizing search algorithms in unstructured P2P networks address the 
trade-off between the two performance objectives. In contrast, our goal in this project is to attempt to 
achieve both objectives. We proposed a series of different mechanisms for P2P systems. They aimed to 
reduce the unnecessary traffic, improve the search efficiency, and create an incentive structure. 
 
Keywords: Peer-to-peer, flooding, distributed approach, search efficiency, super-peers, cache, topology 
mismatch, load balance.

1.2 P2P網路的拓樸架構 
拓撲架構是指分散式系統中各個計算單元之間的物理或邏輯的互連關係，節點之間的拓撲架構
一直是確定系統類型的重要依據。目前互連網路中廣泛使用集中式、層次式等拓撲架構。Internet
本身是世界上最大的非集中式互連網絡，但是 90年代所建立的一些網路應用系統卻是完全屬於集
中式的系統，許多 Web 應用都是營運在集中式的伺服器系統上；而在此架構下，將面臨著過量儲
存負載、DOS（Denial-of-Service）攻擊、網路頻寬限制等一些難以解決的問題。P2P 系統主要採
用非集中式的拓撲架構，其中又以非架構化拓樸 P2P系統（Unstructured Topology P2P System, 簡
稱 Unstructured P2P）最為普及，如下圖三所示。Unstructured P2P是在疊蓋式網路（Overlay Network）
上採用隨機圖的組織模式，網路中的節點可以自由地加入或離開，同時節點分支度（Degree）服從
Power-law法則[8]，從而能夠較快發現目的節點，面對網路的動態變化表現了較好的容錯能力，因
此具有較好的可用性；同時可以支援複雜查詢，如帶有規則表達式的多關鍵詞查詢、模糊查詢等，
採用這種拓撲架構最典型的範例便是 Gnutella。準確地說，Gnutella 不是特指某一款軟體，而是指
遵守 Gnutella協議[9]的網路以及客戶端軟體的統稱。目前基於 Gnutella網路的客戶端軟體非常多，
著名的有 Shareaze、LimeWire和 BearShare等。

1.3 Flooding搜尋演算法之問題探討與相關研究 
q Flooding搜尋機制既有性質與隱含問題 
在 Unstructured P2P網路中，隨著節點的不斷增多，網路規模不斷擴大，透過傳統 Flooding模
式搜尋目的節點的方法將造成網路流量急劇增加，從而導致網路中部份低頻寬節點因網路資源過載
而失效，所以在初期的 Gnutella的網路中，存在比較嚴重的分區、斷鏈現象。另外，因網路拓撲連
接方式是完全隨機之緣故，同樣的 Query 在節點與節點間之相連邊（Edge）將可能重覆傳送，造
成不必要的頻寬浪費，如下圖五所示，當節點 S 發出 Query至整個網路，其箭頭代表 Query 的發
送流向，而 Edge上有兩條相反方向的 Query即代表 Query重覆發送，若網路規模更加擴大，其頻
寬浪費之嚴重情況可想而知，如此將大大地降低整個 P2P系統的效率。而因為 Unstructured P2P系
統擁有節點成員可以隨意加入與離開之特性，使得 Flooding 的搜尋路徑在若干時間後可能產生斷
裂或遺失之現象，造成擁有所搜尋資源的節點在回回傳 Query Hit的訊息時，因路徑遺失而無法將
該訊息順利且正確地回傳到原始發出 Query的節點，如此一來，將使得原先的 Query成為毫無意義
的查詢，徒增頻寬與系統資源的浪費。 
另外，因Unstructured P2P是屬於疊蓋式網路（Overlay Network）的架構，其網路的連接是Logical 
Topology，而實際上的 P2P 網路裏的節點連接方式則是 Physical Topology，在此情況下，將產生
Topology Mismatch的問題，而使得 Unstructured P2P系統架構所隱含的問題更加嚴重；如圖六所
示，其圖（a）為圖五之部份網路，而在實體網路裏，其實際的連接方式如圖（b），因此圖（a）的
拓樸連接方式若能像圖（c），則實際 Query的傳輸路徑將簡潔許多，即能提高搜尋的效率。

算能力和儲存共享以及基於 P2P技術的協定與服務共享平台等進行研究發展與改進探討，使得 P2P

針對前述 Flooding 搜尋機制之相關問題與缺點，後期許多學者在 Flooding 方法的基礎上作了
許多改進[21, 22, 25, 27]：首先，為了減少重覆 Query搜尋的網路頻寬浪費，Random Walk方法[71, 
21, 29]採用只發出 k個 Query給隨機挑選的 k個相鄰節點之策略；而 Iterative Deepening[15, 25]則
是策略性地循環遞增 TTL，來減少 Flooding的搜索半徑；[64, 16, 40]則是研擬出不同的 Query方式，
試圖維持 Flooding 的概念與精神，達到更有效率的搜尋方式。上述幾種方法雖然能夠達到降低搜
尋頻寬的目的，但卻可能會帶來 Response Time過長、Search Scope變小、Query Hit減少之缺點。 
[18, 44]則是採用節點群聚現象的特性，針對特定的搜尋行為以及所提供分享資源的類型進行
Clustering的動作，將 P2P系統裏的節點區分成若干個 Cluster，當節點欲尋求資源時，則針對資源
類型在所屬 Cluster 裏進行搜尋的動作，以期能快速地找到所需資源，同時能減少不必要的頻寬浪
費；但在 Unstructured P2P的環境裏，因節點能夠隨意地加入與離開，將使得節點的群聚現象容易
產生變動，使得 Cluster不易維持正確和即時的狀態，因而造成 Query Hit的狀況不佳，或者過於頻
繁進行 Clustering而產生 Overhead過大，相對地系統的 Performance將無法達到預期表現。 
另外，[67, 73, 63, 22, 30, 32, 35, 39, 41, 43, 48]等研究則是導入快取記憶體（Cache）的概念，
其暫存的資料可能包含資料的索引或者實體內容，每個系統中的節點利用自身的儲存空間，暫存之
前其他節點發出 Query查詢的 Query Hit資訊，當下次該節點又收到同樣的 Query查詢，則可透過
自身暫存的資料，來快速回覆相對應的資訊，如此可避免不必要的 Query轉發動作；但此方法大多
局限於儲存空間的限制，或者只暫存相鄰節點的 Query查詢，同時因 Unstructured P2P系統中節點
可任意加入或離開之特性，將大大地影響到此方法的效果。 
[60]則是從 Query Response的角度進行探討，研擬出一套演算法來降低因節點任意加入或離開
而產生 Query路徑遺失或斷裂之現象所造成的影響，讓 Flooding 搜尋方法可以確保其在 Query以
及在 Query Hit 後產生 Response 之整體流程的完整性，不受系統節點變動而產生無謂 Query的現
象，讓需求節點與目的節點之間能夠確實互相連接，以維持系統應有的搜尋效率；但是在 Flooding
搜尋機制下，不必要的頻寬浪費往往在 Query搜尋時所造成的情況最為嚴重，Query Response的路
徑遺失或斷裂現象所耗費的額外頻寬比例不高，因此，該方法所提昇的效率對於整體系統搜尋效能
而言，所改善的程度可能不大。 
接下來，[28, 27, 49]等研究則提出超級節點（Super-peer, Super-node, Ultrapeer）的概念，此概
念現今被應用於 Morpheus[50]，KaZaA[51]，以及目前的 Gnutella，而其精神主要是由 P2P 系統中
挑選出具備某些特性的節點來擔任 Super-peer，其他非 Super-peer的節點則與這些 Super-peers相連
成為樹葉節點（Leaf-node），每個節點欲發出 Query查詢時，則先透過與該節點相鄰的 Super-peer
進行查詢的動作，如果查詢不到，再透過該 Super-peer 進行 Query Flooding 的動作，換句話說，
Super-peer 即具備代理伺服器（Proxy）或索引伺服器（Index Server）的角色，利用此架構來提昇
Query查詢的效率，同時降低 Flooding時的頻寬浪費；但是，隨著 P2P系統的網路規模不斷擴大，
Super-peer 的數量也勢必隨之增加，此時仍然需要一套有效率的搜尋方法以及如何選定節點擔任
Super-peer的選擇策略，否則，將會失去設置 Super-peer的原始意義，而 Flooding的缺點與影響同
時也將必然存在。

解決 Logical Topology與 Physical Topology所產生的 Topology Mismatch問題，來改善 Flooding方
法在進行 Query查詢時所產生的多餘路徑與重覆轉發 Query查詢等現象，進而達到降低頻寬浪費之

q 於 Unstructured P2P系統中設計更高效率的搜尋演算法 
我們採用機率統計模型來進行Flooding演算法的改進：首先，我們針對Flooding搜尋法的Query
查詢方式進行觀察，如下圖七所示，虛線代表 Query 查詢，而其虛線上的編號代表由 Query Peer
發出 Query查詢時，透過 Flooding方法搜尋時 Query的轉發次序（Hop）。我們可發現到，每個節
點與相鄰節點之間，存在其相依關係，亦即某節點若需要服務，該節點需向其相鄰節點尋求服務，
若相鄰節點無法提供服務，則繼續向其相鄰節點提出尋求，如此遞延下去，直到找到可提供服務的
節點為止；反之亦然，目前提出服務請求的原始節點，可能以後也會因相鄰節點需要服務，而成為
被相鄰節點請求服務的對象。因此，我們可透過機率模型，來求出最有可能滿足其服務的 Query
尋求路徑，換句話說，即是節點在發出或轉發 Query查詢時，透過機率模型機制來計算其相鄰節點
的權重（Weight），從而挑選出適合的相鄰節點來進行 Query查詢的動作，如此可避免不必要的頻
寬浪費，同時又能維持搜尋的效率。而其Malicious Nodes的 DOS攻擊問題，將同時被考量進入機
率模型搜尋機制中，透過此機制被過濾出來，亦即經過相關的判斷與計算，Malicious Nodes 所被
求出的Weight值將會很低，系統裏的其他節點將不會向這些Malicious Nodes進行 Query查詢的動
作，使得這些Malicious Nodes將無法佯裝成服務提供者，如此可有效避免系統效率之降低。

分利用：若某節點提供分享或服務的情況愈頻繁，則該節點的 Weight 值將愈高，而相鄰節點則根
據此 Weight 值的高低來進行相對程度的服務或 Query 轉發，亦即該節點若需要尋求服務而發出
Query查詢，而其相鄰節點則因其Weight值較高而提供較多的服務（如資源分享，Query轉發次數
等）。如此一來，將形成一套鼓勵及回饋的機制：節點提供愈多資源分享與服務，將也能從相鄰節
點得到愈多資源分享與服務。最後，透過此相依關係的應用，我們發展設計之機率統計搜尋機制則
能同時解決 Free Loaders和 DOS攻擊等問題：Free Loaders因其只分享其他節點資源而不提供服務
之特性，其Weight 值勢必將非常低，如此一來，便可抑制 Free Loaders所造成的系統負擔，使其
無法扮演資源消耗者的角色，因為其相鄰節點將不會為其提供資源或服務，除非 Free Loaders能和
其他節點一樣提供分享與服務，否則將無法存活於系統中；而Malicious Nodes的 DOS攻擊則可能

將形成一個階層式的架構，此架構區分為二階層（2-tiers Structure），分別為 Super-peer 階層與非
Super-peer階層。當某個節點需尋求服務時，則先向所連結的 Super-peer發出 Query查詢的動作，
透過該 Super-peer轉發 Query查詢至 Super-peer階層，因 Super-peer階層裏的節點提供較多資源之
緣故，如此將擁有較高之機會可滿足需求節點的 Query 查詢，若該 Query 查詢無法在 Super-peer
階層裏尋求到所需資源，則需求節點可以再重新發出 Query 查詢至整個 P2P 系統，如此一來，將
可降低不必要的 Query轉發之頻寬浪費；另外，當需求節點在重新發出 Query查詢至整個 P2P 系
統時，若此時該 Query查詢由整個 P2P系統得到 Query Hit的回覆，即意謂在非 Super-peer中，可
能存在某些節點擁有擔任 Super-peer 的能力，此時將可試圖判斷此滿足 Query 查詢的節點，加入
Super-peer階層中，提昇系統搜尋的效率。

當系統架構改進完畢後，接下來便是進行 Cache概念導入的研究與探討。傳統的 Unstructured 
P2P系統架構中，當節點需尋求資源而發出 Query查詢時，透過 Flooding搜尋方法將能夠探訪系統
中所有節點以尋求可提供服務的目的節點，若導入Cache的概念，讓每個節點能夠根據以往的Query
尋求資訊，暫存相關的資料內容，則當後續的時間裏，節點收到 Query查詢時，將先行查詢暫存區
裏的資料，若暫存區存放的資料與 Query查詢內容相符合，便能夠提供立即且迅速的服務，同時可
避免 Query轉發的動作，如此一來，將可節省 Query查詢的時間以及不必要的頻寬浪費，提昇系統
的查詢效率。以往的相關研究[19, 30, 32, 35, 39, 41, 43, 48]針對 Cache的導入應用大多都是讓系統
中的每個節點額外提供暫存區來存放相鄰節點透過該節點得到 Query Hit的查詢內容，其存放的資
料內容是隨著 Query 查詢的次序來儲存，並沒有根據暫存資料的使用頻率來進行相關的判斷與處
理，同時因為 Unstructured P2P系統的所有節點可以任意加入或離開之緣故，將造成節點裏暫存區
的存放資料可能無法發揮其最大效用。本計畫中，我們則針對此缺點提出一套 Cache暫存的改進機
制，透過 Cache暫存之機率統計模型，來挑選出合適的資料內容來進行資料暫存的動作，同時，針
對透過節點本身收到的 Query Response 來源，進行節點位置索引存取之處理，如此一來，則能夠
充份利用 Cache暫存區的空間，同時讓暫存的資料提高 Query Hit的機率，以達成導入 Cache概念
的目的。 
 
圖十二 Cache暫存的架構示意圖 
如上圖十二所示，系統中的節點皆提供額外的暫存區來儲存相關資料，但因暫存空間有限之緣
故，因此所存放的資料內容必需要儘量能夠提供需求節點 Query Hit的回應，如此才能真正達到設
置 Cache暫存區的目的。因此，在我們發展設計的 Cache暫存之機率統計模型，則提供一個判斷機
制，讓暫存區存放重要且有效的資料，使其皆能夠儘量達成 Query Hit以提昇系統效率，同時降低
DOS攻擊的潛在危機；除此之外，我們同時採用資源分享異質性（Resource Sharing Heterogeneity）
的概念，亦即系統中的節點對於資源尋求擁有若干的共同性，而 P2P 系統中不同的資源需求共同
性，將產生異質性的差異，透過此差異性，系統中所有節點在性質上將能夠區分成不同群聚
（Cluster），透過每個節點的 Cache暫存，將能夠區分出群聚，同時找到自身所屬的群聚，因此，
我們在每個節點中，透過發展設計的 Cache暫存機率統計機制，則擁有共同資源需求的其他節點資
訊暫存起來，形成節點位置的索引，當某個節點需尋求資源而發出 Query查詢時，則首先透過節點
位置索引的查詢，向索引中的其他節點進行資源尋求的動作，因其具有資源分享共同性之緣故，將
能夠提高 Query Hit之機率，以減少 Query轉發之動作與次數，如此一來，Cache概念的導入將能

步驟十：由隨機產生之不同拓樸連結中，分析若以架構演化機制來改進 Unstructured P2P系統架構，
其演化的週期與頻率設定對於整體架構與效率之影響。同時探討 Weight 的 Threshold 值訂
定對於 Super-peer階層實際效率改進之影響。並實際套用 Unstructured P2P架構演化機制，
透過不同的系統環境與參數（系統規模、系統拓樸、系統演化週期與頻率、Super-peers 之

及變化。 
步驟十一：於不同系統規模以及不同之隨機拓樸 P2P系統進行架構演化，驗證 Free Loaders的抑制
性與改善效果。同時模擬不同的 DOS攻擊於不同系統規模以及不同之隨機拓樸 P2P系統之
演化架構，同時驗證其 DOS攻擊的破壞程度與改善效果。最後，評估 Load Balance於傳統
Unstructured P2P系統的影響性，同時探討演化架構所考量 Load Balance機制對於整體系統
效率之改善程度。 
步驟十二：分析於傳統 Unstructured P2P系統中，節點所擁有資源之分佈性及其對於系統之貢獻性。
同時導入Cache暫存概念，並探討不同的暫存策略對其系統效能之影響。並分析Unstructured 
P2P 系統中，節點透過所擁有資源之共同性所產生之群聚現象，探討其分佈情況以及其群
聚性對於系統搜尋效率之提昇程度。 
步驟十三：發展 Cache 暫存機率統計機制，訂定暫存資料選定策略，在有限的暫存空間，儲存有用
的資訊；以及如何適當地存放零亂的暫存資料，使得在實際進行 Query 查詢時，能夠快速
有效地查詢暫存區的資料。同時延伸 Cache 暫存機制的架構，利用資源分享異質性來區分
出節點間的不同群聚，並發展不同群聚的節點位置索引存取策略，訂定節點如何利用群聚
位置索引來找尋可用節點，以提昇搜尋效率。 
步驟十四：分析不同 Unstructured P2P系統規模中，其 Cache暫存區的最適空間設定，以達到最大空
間使用，在系統效能與額外空間耗損情況下訂定最佳平衡點。同時實際採用 Cache 暫存機
制，透過不同的系統環境與參數（系統規模、系統拓樸、Cache 暫存空間大小、節點資源
群聚現象等），驗證其方法的效率，觀察系統中 Response Time與 Traffic Cost之改進與變化。 
步驟十五：整合 Flooding搜尋改進機制與 Unstructured P2P架構演化機制、Flooding搜尋改進機制與
Cache暫存機制、Unstructured P2P架構演化機制與 Cache暫存機制，透過不同的系統環境
與參數（系統規模、系統拓樸、節點分享資源多寡、Super-peers 數量、Cache 暫存空間大
小、節點資源群聚現象等），驗證其機制整合的整體效率，觀察系統中 Response Time 與
Traffic Cost之改進與變化，並利用各機制獨立執行之效能分析結果，進行單獨機制與整合
機制之效能比較與探討。 
步驟十六：針對 Flooding搜尋改進機制、Unstructured P2P架構演化、Cache暫存機制進行整合，驗
證其機制整合的系統整體效率，觀察系統中 Response Time與 Traffic Cost之改進與變化。
