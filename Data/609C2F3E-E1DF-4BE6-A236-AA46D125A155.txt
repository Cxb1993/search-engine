3. Introduction 
 
伴隨著產業與市場全球化而來的，是無限的機會與挑戰。許多公司都被迫重組或改變
他們的運作模式，來因應新的企業模式。其中，為了支援企業資訊的分享與人員的溝通，
並協調人員及各部門的分工，善用資訊及通訊技術，以達到分球化 e-collaboration的目標，
是企業面臨全球化挑戰時的一項重點[1]。目前已被廣泛採用的 e-collaboration的技術包括
有網頁介面的計畫討論專區、群組式的電子郵件、即時訊息（instant messaging）、群體行事
曆、文件及知識管理系統、桌上型視訊會議系統、工作流程（workflow）管理系統等。 
 
E-collaboration相關技術的發展，有兩種走向。一種是原先以視訊會議設備起家
的廠商，進一步拓展市場到公司的日常工作流程中。另一種是以資料庫處理能力見
長的廠商，將產品由原先的 ERP（Enterprise Resource Planning）進一步擴展到企業
的日常通訊活動中。Oracle在 2006年宣告進入 Voice over IP市場，是此一類型的代
表[2][3]。 
 
4. VoIP 
 
在過去數年裡，通訊科技中最廣為社會大眾所採用的，首推行動電話與網際網路
（Internet）。行動電話提供使用者一個全新的體驗，讓他們隨時隨地都可以利用電話與客
戶、同事、家人、朋友保持聯繫。而相對的，有別於行動電話在設計之初以語音服務為主
的模式，Internet在一開始就希望提供多種的服務。它的彈性允許使用者創造各種全新的應
用方式，這項優點是 Internet在過去數年中迅速成長的一大主因。因此，傳統的 Internet終
端設備通常是一般用途的電腦（不像行動電話有專屬的、經過認證的手機），而且是連接在
固定式的網路上。這使得它在本質上與行動電話有著極不相同的特性。 
 
最近發展的趨勢，是上述兩項科技的結合。網路電信（IP Telephony）及相關技術已經
有愈來愈多的個人甚至企業用戶採用，市場上的產品也愈來愈多。許多人相信，在未來的
幾年內，IP（Internet Protocol）就會取代現有的交換機，成為電信服務所主要使用的技術。
許多未來的無線電話系統，也將以 IP技術為主，來提供相關的服務。行動電話手機的功能
也將更一般化，由現有純語音的功能，加入影像、網頁瀏覽、電子郵件、及線上遊戲等。 
利用 IP網路來傳送聲音，希望將電信話務的傳送媒介，由原先的 circuit-switching 
network 改為使用 packet-switching network，這個 Voice over IP 的理想，從 1995 年 H.323 
[4]開始，一直到後來的 Session Initiation Protocol (SIP)[5]，Media Gateway Control Protocol 
(MGCP)[6]，已經行之有年。早期由於網路頻寬有限，在有限的頻寬下並未能有很好的應用
成果。而若需具備足夠的頻寬，昂貴的專線租金又令許多使用者望之卻步。近年來，由於
網路頻寬也如同積體電路一般，依循著 Moore's Law，以每十八個月增加一倍（或更多）
的速度飛快地發展。中小企業以及SOHO族都能以極為低廉的費用取得相當高的頻寬。利
用網路進行多媒體的語音互動，所需要的網路基礎環境已經初備。不僅許多國際電話在越
洋的線路部份都已經改用 VoIP 的技術進行話務的傳送，甚至最近在 Internet 上引起大家

上圖為網路電話平台的架構示意圖，網路電話系統的各個元件藉由 SIP 通訊協定彼此互
動。關於這些元件的敘述如下。 
 
 Call Server: (圖1(a))在網路電話的平台中提供基本的通話控制功能，並以SIP Proxy 
Server 的身分處理SIP的要求與回應。此外，它也當作儲存SIP使用者聯絡資訊的 SIP 
Registrar。 
 PSTN Gateway: (圖1(b))支援網路電話平台與公眾電話交換網路之間的通訊，它允許 
VoIP 的使用者藉由企業內部的電話交換機 (PBX, 見圖1(c))直接或間接地聯絡到其他
公眾電話交換網路使用者，以與傳統市話或行動電話的使用者通話。 
 SIP網路電話是一個以硬體或軟體來執行的終端設備(圖1(d))，它提供了如撥號、應答、
拒答、掛斷、轉接等基本的電話功能。在 VoIP 系統裡，目前已成功地有桌上型電腦、
筆記型電腦(使用有線或無線網路)和PDAs(使用無線區域網路)等終端設備上的SIP網路
電話軟體。另外使用者也可視經費，採購符合 SIP 通訊協定的 audio/video IP phone 硬
體設備(圖1(d))，以整合到此一系統中。 
 
5. Cyberhood ERP Software 
 
本計畫中所參考的 ERP系統，是國尊科技所開發的 Cyberhood軟體[16][17]。Cyberhood

的 e化平台，內含完整的MAIL SERVER以及 DB Server，安裝 Cyberhood於伺服主機後，

7. Choices of Software 
 
VoIP Client的部份，原本廠商考慮採用現成的 X-Lite軟體[18]。這套軟體電話可安裝
在Windows 98/ME/NT/2000/2003等作業系統上，支援 SIP通訊協定，同時也支援
G.711u+a/iLBC/Speex等 codecs，在 Internet上的通話品質佳，是個相當好用的軟體。但在
使用之前必須先經過安裝的動作，是它比較麻煩的地方。因此本計畫利用 RAT[19]加上
eXosip[20]另行開發專供 Cyberhood使用之 VoIP Client，以支援其 nomadic user之需求。另
外，使用者也可以採購支援 SIP協定的硬體電話，以取代軟體電話使用。 
 
VoIP Server是本階段的重點，本計畫採用 iptel.org以 GNU General Public License
提供之 SIP Express Router (SER)軟體[21]，與 Cyberhood的 AD Server (Authentication 
& Dispatching Server) 進行整合。SER支援RFC 3261的SIP標準，可同時扮演 registrar, 
proxy server, redirect server的角色，同時可透過內建的MySQL模組去查詢資料庫，
在 FreeBSD及 Linux上都十分易於安裝。本年度我們主要進行 SER與 Cyberhood的
整合，發展出將 VoIP服務導入 ERP系統的模式。 
 
8. 計畫成果自評 
本計畫成功設計出一軟體架構，將網路電話之服務以 loosely-couple的方式添加到現有
之 Cyberhood軟體。成果以會議論文之方式發表於: 
 ChingChen Chang and Quincy Wu, "Design and Architecture of a Portable 
User Agent in SIP Collaboration Systems", in Proc. 3rd International 
Conference on Internet and Web Applications and Services (ICIW 2008), 
pp.273-278, Athens, Greece, June 8-13, 2008. 
本篇論文並獲得 IARIA (International Academy, Research and Industry Association)
的最佳論文獎(Best Paper Award).
