說明：  
訊號控制器可使資料輸入介面與非接觸 IC 讀卡器相連接，透過控制器讀寫並存取非接觸
IC 讀卡器，可控制相關設備，藉以管理大型資料庫資料，將資料庫與非接觸讀卡器控制器結
合，可達到自動化管理之目的。 
本系統開發架構採網路連結，透過標準乙太網路連結，所有控制器皆可透過網路直接存
取資料庫，並透過資料庫與資料庫連結相關學生、職員、教師…等基本資料庫作即時存取。
因連結至網路即時存取資料庫，所以所有控制器皆可透過網路設定登錄人員資料，並可將權
限透過網路下放給各控制器管理者自行管理，並作線上新增、修改、刪除、查詢等功能。 
本計畫認為若能將校園事務機透過訊號控制器與非接觸 IC 讀卡機作連接，將可作有效的
控管，除可大量減少廢紙的產生並可降低紙張、碳帶、滾軸、及維護的成本外，甚至用電量
也會降低；故經評估後，認為有迫切開發此系統之需求。 
為求務實致用，本計畫經產學雙方同意，以醒吾技術學院相關校園 E 化建設為基礎，作
為校園事務機導入 RFID 應用之前置研究標的，希經由實務情境規劃、系統分析、設計、建
置等步驟，期望能協助企業完成系統雛型，以作為後續商業化之依據。

合作企業簡介： 
微程式創立於 1995 年，為一「專業身份識別產品與技術提供者」，致力於 RFID 及 SSO 技術研究

並創立自有品牌 IDU。 
微程式具備豐富的產業實務導入經驗，並發展出符合國內在地的應用解決方案，輔導系統整合廠
商(SI)跨入身份識別領域，提升各大產業在 RFID 技術及 SSO 領域的導入及應用能力，徹底提昇產業國
際競爭力。 
「資訊系統加值服務」為本公司 84 年創立時鎖定之營業項目，隨著各種資訊系統，產品（服務）
生命週期的起落，行銷重心也跟著循序轉移。早期切入「硬體經銷」門檻較低，轉而「硬體整合」「客

系統使用情境流程說明 
1. User Login 
使用者透過 windows 統一認證系統，輸入 Username / Password 登入之後，使用者列印每一份文件，
都會帶有該使用者相關資訊。 
2. Print Job 
使用者透過 Application 列印文件，列印功能透過 OS 的 LPD Server 及 Print Driver 產生文件（PCL6、
PostScript…等印表機語言），透過網路印表伺服器（Print Server）將此份文件傳輸到 Print Queue
暫存，等待使用者靠卡取回文件。 
3. Print Queue 
透過 MP PrintQueue Server 接收來自 Client 端印出的文件，將列印文件儲存管理，等待使用者靠卡
後放行至 MFP，若在設定時間內未被放行之文件，系統會自動移除該份文件，以節省儲存空間。 
4. User take Paper 
使用者透過 RFID 卡片去感應讀卡機，讀卡機取得使用者身份 ID 及剩餘張數，當使用者權限驗證
通過後，通知 Print Queue 會將文件送出到讀取卡片之印表機。 
5. Print/Copy Data Receive 
將 User 感應卡片資料紀錄，即時傳遞給 Print Server 處理。 
6. Print/Copy Accounting Server 
接收使用者資料及使用記錄，通知 Print Queue Server 放行並統計列印張數產生報表。 
7. Inform  
並將印表機資訊及使用者資料送給 Print Queue Server，通知User所在位置，放行文件列印至該MFP。 
8. Print Job  
收到靠卡使用者位置通知，再放行文件列印至選用的印表機。  
9. Inform 
並將印表機資訊及使用者資料送給 Print Queue Server，放行文件列印至該 MFP。 
10.  Print Job 
收到靠卡使用者位置通知，再放行文件列印至選用的印表機。
