  
研究摘要(500 字以內)： 
 
數位生活已隨著科技技術的進步，逐漸的成為未來的發展趨勢。其中，普遍存在
(Ubiquitous)的裝置與技術，能讓使用者隨時隨地，存取數位內容，更是未來數位生活能否
普及的重要關鍵。若能結合生活周遭的裝置，建立起安全監控、居家照護、自動控制等智
慧型系統，著實能為生活或工業環境帶來更高的便利性。然而，隨著技術的發展，周遭裝
置的功能日漸強大，許多裝置(如 PDA, Home Gateway, Set Top Box, PlayStation)自成一格的
提供特定人士特定的服務，但彼此之間卻無法相互整合以提供更完整的服務。 
 
SOA(Service Oriented Architecture)服務導向架構具備的快速整合、易變動、跨平台特
性，正是企業用來推動系統整合的一個重要方法，讓不同平台間的服務，可以被整合以形
成更有意義的應用。但 SOA 的整合，多以企業內的大型伺服器為對象，要應用於小型裝
置將遭遇許多問題，如介面規格、頻寬、電源及可靠度等。 
 
因此，若能利用現有的環境，發展適用於小型裝置的 SOA 執行平台，將可用來整合
各種裝置，以建立更完整的數位生活或是工業應用，這對於推動智慧型系統在國內的應用
發展將有所助益。

研究子題一：智慧型異質介面探索機制 
為連結不同的標準的裝置，本計畫必須研究各種智慧型裝置的標準並開發各種標
準的轉換器，使得本研究之系統能連接不同標準上的裝置。以 UPnP 為例，UPnP 利
用了包括 IP、TCP、UDP、HTTP 以及 XML 在內的網路通訊協議，並可透過橋接方式來
支援操作非採 IP 協議的裝置。其運作原理錯誤! 找不到參照來源。包括： 
 Discovery 
當某樣裝置加入到特定的網路中之後，UPnP 的 discovery 協議將可容許該裝置
在網路中宣稱自己服務控制點的存在，而當該控制點被網路認可並加入後，UPnP
的 discovery 協議將可容許該控制點搜尋在該網路中的其他裝置並進行配對。這方
面的協議主要是遵循目前的 SSDP 機制來進行。 
 Description 
當控制點發現網路上的裝置之後，控制點對於該裝置的能力與特性還處於一
概不知的狀態，在這種情況之下，當然也就無從控制起，因此就必須要透過偵測
該裝置的 XML 描述語言，來描述裝置本身包括型號、製造商、序號以及該裝置所
能提供的服務等等特性。描述內容也包括了執行狀態、資料類型以及事件特徵等
等，作為控制點判斷該裝置的有效參考資訊。 
 Control 
當控制點接收到裝置的描述（Description）之後，該控制點便可以開始嘗試對
該裝置送出指令，利用 SOAP 協定，並透過 XML 來表達。而接收指令之後的動作
效果，則是可以從 Description 中對於服務運作狀態的描述來取得回饋。 
 Event Notification 
當 UPnP 描述中包含了一連串的動作、回應以及變數時，服務本身也會公布這
些變數的變化。因此在控制點方面也會取用這些描述。服務本身會利用送出事件
訊息來傳達這些更新。同樣的，這些事件訊息也是基於 XML 的事件描述格式。控
制端與服務端為了要進行相互的狀態確認，事件的產生與傳送都會傳達到所有參
  
與規則(Rules)物件轉為中繼形態，以進行跨標準的訊息交換。執行後的結果，則再透
過轉換器轉換為異質標準特有的資料格式，以傳送回遠端特定標準的裝置上。透過此
中繼資料型態的轉換機制，可降低不同訊息內容轉換的複雜度與連線效能評估的負
擔。

圖 5 SSO 解決方案 
由於本系統的執行環境為 Ubiquitous 環境，較不具穩定的系統，可供儲存其他裝
置的認證資料。因此，本研究擬改變 Proxy-based true SSO 的方式，來進行本系統的
安全保護。在 OSGi 裝置要加入系統前，將設定可使用該裝置的角色資料於該 OSGi
裝置中，該資訊將以系統私鑰進行簽署，以防遭到竄改。當使用者要使用某裝置時，
需輸入一次帳號密碼，以取得該使用者的角色資訊，該帳號密碼將與本機端的使用者
資料進行比對，若成功認證後，即可以該角色，存取其它裝置。 
此方式的使用前提，在於 OSGi 裝置上需設定使用者資料與該 OSGi 裝置的存取權
限，然而，對於裝置數量較大的系統可能會有設定及維護繁複的情況發生。為來將視
情，是否加入統一的認證裝置以降低系統維護成本。
