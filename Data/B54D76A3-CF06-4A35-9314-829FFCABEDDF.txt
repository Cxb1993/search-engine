中文摘要 
本研究計畫目標主要是研發在同儕網路(peer‐to‐peer  networks)上提供影音串流的關鍵技術(key 
technologies)並建立以此為基礎的服務導向式平台，讓大型異質系統內的同儕應用程式可透過此平台取
得所需的評估資訊，冀能最佳化整體效能，並提升系統的可靠度與穩定度。 
在此背景下，本計畫彙整台北科技大學、台灣大學、交通大學及中央研究院等六個研究團隊，共
同展開為期三年的『高韌性同儕網路及其在即時多媒體上之應用』研究計畫。即時多媒體應用的發展，
仰賴有良好的服務品質。服務品質的提昇，則與連線節點效能有相當大的關係。因此，我們期望能建
立適用於異質網路的節點效能評估機制，讓同儕節點在取得資源時，能夠視該節點的需求，取得其他
節點的效能資訊。具體的研究主題包括下列五大關鍵技術之研發。 
(1) 監控代理人與通用訊息傳送架構 
(2) 異質同儕網路的傳輸品質監測技術 
(3) 異質同儕網路傳輸品質評估技術 
(4) 異質同儕網路節點整合 
(5) 平台開發軟體工程技術 
我們以理論和實作並行方式，藉由資訊平台技術的創新應用，建立一個兼具廣泛性監測和即時性
支援的『高韌性同儕網路之多媒體應用平台』，實現異質同儕網路  整合的先導性研究。 
研究的過程中，我們分三年，環繞著五大關鍵技術進行研究與開發： 
第一年研究，著重於同儕網路相關技術的研讀與建立監控代理人及通用訊息傳送架構。監控代理
人執行於同儕節點上，負責取得節點間的效能資訊。而通用訊息傳送架構，讓訊息能透通(Transparent)
於同儕應用程式與超節點之間，兼顧高效能與高延展性等考量，作為後續研究的基礎。 
第二年研究內容，著重於超節點如何建立最佳連線的關鍵技術上。包含前端同儕節點的評量機制
到後端最佳連線節點的分析等，並進一步以服務導向架構進行分析與設計，建立可重複使用的軟體服
務。 
第三年研究主題整合一、二年研究成果與其他子計畫產出，完成整體系統平台的建置，並針對即
時多媒體的應用，進行相關的效能評估，以完成同儕網路之最佳配置。 
研究成果有助於解決同儕網路連線的不穩定性，克服即時多媒體應用所遭遇的問題。如此一來，
將可讓即時多媒體廣泛應用於一般生活，配合手持式裝置的研究，未來將能以手持式裝置隨時觀看即
時資訊，如電影或電視等，除了讓民眾的生活更加便利外，也能滿足國家對於寬頻通訊、網路多媒體、
數位內容的發展期望，貢獻棉薄之力於提昇國家高附加價值新興產業的研發能力。 
 
關鍵字：同儕網路、服務導向架構、服務整合、多媒體應用程式、即時多媒體、效能評估

一、前言 
本研究計畫目標主要是研發在同儕網路(peer-to-peer networks)上提供影音串流的關鍵技術(key 
technologies)並建立以此為基礎的服務導向式平台，讓大型異質系統內的同儕應用程式可透過此平台取
得所需的評估資訊，冀能最佳化整體效能，並提升系統的可靠度與穩定度。 
同儕網路是一項正在現實世界中蓬勃發展，且引領許多資訊科技研發方向的重要議題。其具無中
央控管、需自我組織、跨階層傳遞的路由機制以及動態拓撲等特性。相對於理論完整、技術成熟的傳
統主從架構(standard client/server model)，同儕網路中每位成員都可身兼伺服端與客戶端的功能，而成
員本身又具有相當大的異質性，它可能是個人電腦、工作站、有通訊設備的 PDA，甚至專門為戰場或
救援任務設計的裝置。 
這種異質性一方面增加了許多應用上的彈性與便利，但另一方面也大大提高了系統和軟體設計上
的難度。具體實作具備服務導向架構的連線平台。於在發展的初期，同儕網路最大宗的應用是檔案共
享(P2P file sharing)，例如著名的 Napster, Gnutella，而隨著網通與多媒體相關技術的發展，以及像 Skype, 
Youtube 等公司的成功，我們認為端對端影音串流(P2P audio and video streaming)將受到大眾的歡迎並
有潛力成為同儕網路上最重要的應用之一。相對於純檔案共享的同儕網路，影音串流有許多本質上的
不同，因此本計劃將對這些獨特議題進行深入探索，並提出從理論到實作面的解決方案。 
如上所述，我們需掌握這股趨勢可能帶來的工商業需求，並盡力在研發水準趕上世界頂尖機構。
鑑於同儕網路、影音串流和無線通訊都與跨階層設計的良窳緊密切合，因此我們也需納入上述因素對
系統可靠度或穩定度帶來的影響。在此總目標下，本計畫發展「異質同儕網路之服務導向效能評估與
監測機制」，用以評估與監測同儕網路之效能，適時地調整網路組態之配置，以達到同儕網路應用的
最高效率。

夠視該節點的需求，取得其他節點的效能資訊。當後續網路狀態有所改變時，仍不斷的更新連線資料，

當異質的同儕應用程式要取得資源時，監控代理人將連線至最近的超節點，上傳該節點的資訊，
並根據該應用程式的需求，詢問超節點以取得其他節點的效能評估資訊。在超節點接收到由 Peer 端傳
送而來的訊息後，啟動系統內的效能評估流程，根據其他同儕節點上傳的監控資料與使用者需求，呼
叫子系統所提供的服務。進行跨同儕網路的搜尋(Search)、發現(Discovery)與評比(Evaluation)等功能，
組合並產生使用者所需資訊，建立節點評估資訊，如反應時間最快、資源擁有度最高等連線列表，並
將該資訊傳送至同儕節點，作為其連線參考。若超級節點上的資訊不足，可與其他超級節點連線，交
換其他同儕的資訊。 
 
當同儕應用程式在接收到連線列表後，可依據連線建議列表，連線至其他同儕，取得必要的資源
與服務。同時，當同儕節點連線其他同儕取得資源時，監控代理人仍不斷的與超節點連線，將該同儕
節點實際連線時的服務品質更新至超節點，讓超節點能隨時更新連線資料，以根據該資料，作出最為
適當的連線建議，詳細的運作流程圖請見錯誤! 找不到參照來源。。

四、研究方法 
要達到異質同儕節點效能評估與監測，本研究規劃於異質同儕應用軟體上，安裝監控代理人程式。
該監控代理人負責取得其他同儕節點的效能資訊，於同儕節點上進行初步的訊息過濾，以減少訊息數
量及轉換格式。接著將處理後的資料傳送至超節點中，以進行資料彙整。超節點接收其他異質同儕節
點的效能資訊後，進行效能的評估，並根據節點需求，建立最佳連線列表。當超節點的資訊不足以進
行評估時，則啟動超節點間的訊息交換機制，與其他超節點交換更多的效能資料。在超節點建立最佳
效能列表後，將傳送至監控代理人，由代理人將取得的列表整合至原有的應用程式中。 
 
除此之外，上述監控代理人與超節點的互動，需以服務導向架構進行開發。採用服務導向技術的
原因除了達到跨平台的應用外，也期望能進一步加強監控代理人與超節點的功能，作為服務導向架構
在同儕網路上的應用框架。 
 
系統的整體架構如錯誤! 找不到參照來源。所示，同儕應用程式(P2P Application)上安裝監控代理
人，透過 Common Message Bus 與超節點交換訊息。超節點則是根植於原有的同儕軟體，在其上加入
更多的功能。包括訊息處理程式(Event Handle Module)、節點效能評量模組(Evaluation Module)、節點
狀態管理程式(Peer Status Manager)、評量規則(Evaluation Rule)與效能資料管理程式(Repository Manager)
等核心系統模組。最後透過服務整合技術來組合個別獨立的功能模組，以建立完整的系統功能。

為了完成上述目標，本計畫於三年執行期間分別完成下列研究項目。第一年著重於同儕節點上，

圖 9  監控代理人基礎架構 
 
Message Receiver： 
廣義訊息接收器的資料來源包含兩個部份，第一部分來自於超節點，如回應的同儕評估資訊或要
求監控代理人執行特殊功能的訊息。第二部分則來自於同儕應用程式，在同儕應用程式要求其他同儕
資訊或更新同儕狀態時之要求。針對第一部份的訊息，本計畫以 JMS API 錯誤! 找不到參照來源。設
計訊息接收的 Client，以符合傳送架構中 MOM(Message Oriented Middleware)的設計。針對第二部份的
訊息，則以 API 的型式，接收同儕應用程式的呼叫。 
 
Message Parser： 
 訊息處理器則在於解譯接收到的訊息，取出其命令、參數與相關資訊。在本監控代理人的設計上，
由於訊息以 XML 格式存在，因此訊息處理器為一 XML 解譯器，該解譯器會被用來驗證(Validate)訊息
格式的正確性，並將 XML 物件轉換為監控代理人的物件格式，以進行後續的訊息處理。 
 
Command Dispatcher 與 Command： 
 命令分派器針對訊息處理器送達的訊息物件進行處理，根據其訊息內容，喚起(Invoke)不同的指令
執行者(Command)進行處理。為了降低指令派送的負擔與提高未來 Command 的擴充性，我們以 Design 
Pattern 錯誤! 找不到參照來源。中的 Command Pattern 進行此架構的設計。如錯誤! 找不到參照來源。
所示，不同的功能都被實作為 Command 物件，命令分派器則以同樣的介面存取不同 Command 物件，
啟動系統功能。在系統的擴充上，可透過不同 Command 物件撰寫、切換與組裝來達成。

 物件關係(Relationships)：開發者需要在系統整合場景或者不同的後端系統中存取相同的資
料。保持系統物件同步的一個常見的問題就是使用不同的關鍵字表示同一物件。物件關係服
務在這些不同的系統物件之間建立關聯。通常在將一種系統物件格式轉換成另一種格式時，
從系統物件映射中存取這些關係。 
 選擇器(Selectors)：可以使用選擇器元件動態選擇和呼叫相同介面的不同服務。例如，不同的
同儕節點類型可能啟動不同的服務。。 
 
服務元件(Service Components) 
 系統流程(Processes)：系統流程元件實現了與服務流程執行語言相容的流程引擎。可以開發和
部署業務流程，它支援長時間執行和短時間執行的系統流程以及可伸縮的基礎設施中的補償
模型。 
 人工任務(Human Tasks)：任何人工需要使用者的介入來完成某功能，以進行下一步驟的動
作。如可能需要使用者的決定，來確定是否需進行進一步的搜尋。 
 狀態機(State Machine)：狀態機提供流程建模的另一種方式。通過這種方式，可以根據輸入
條件和流程來改變流程的狀態，例如在取得評估資訊的一連串服務呼叫時，可視需求變換至
不同的處理模式。 
 規則(Rules)：業務規則用以將系統中的許多判斷依據等，以外部資源檔的方式呈現，此方式
的採用可讓系統的調整更具彈性。 
 
軟體開發與建置方法 
在本計畫中，包含了許多軟體程式的開發與設計。如同儕節點的監控代理人與超節點的建置等，
皆需要開發相關程式以達成必要之功能。為了能設計具高品質的應用程式，本計畫以軟體工程(Software 
Engineering)的方法論為設計綱要，導入如 UML(Unified Modeling Language)、軟體開發程序(Software 
Process)與專案管理(Project Management)等軟體工程(Software Engineering)技術，逐步完成系統的各項
功能。相關的研究方法如下： 
 
統一流程（Unified Process）

Use Case Driven：指團隊在執行整個流程時，都是從 Use Case 開始所有的工作。這樣可以讓開發者在
每次的工作迭代，以客戶的需求為出發點，避免迷失在專案的規模中。Use Case 是用來找出並記錄功
能性需求的一種技術。換句話說，它的功用在於找出對使用系統的參與者有價值的需求，並且能產生
有價值及看的到的結果；例如，建立連線、取得資源等。Use Case 也可以幫助我們開發使用者介面，

系統的開發中能提高整體軟體的品質。

生的結果，使其成為可執行的系統。  
 
測試(Test)：測試工作要驗證物件間的互動，驗證軟體中所有元件的正確性，確保所有的需求已被正確

中進行測試，以期能盡早地發現缺陷，以降低修改缺陷的成本。測試類似於三維模型，分別從可靠性、
功能性和系統性能來進行。

學術研討會，於 6 月 28 日至 6 月 30 日在日本東京(Tokyo, Japan) 北陸先端科學技術學院 (Japan 
Advanced Institute of Science and Technology, JAIST)舉行。 
本研究計畫旨在研發同儕網路(peer-to-peer networks)上提供影音串流的關鍵技術，並建立已此為基
礎的服務導向式平台，使得大型異質系統內的同儕應用程式可透過此平台取得所需的評估資訊，以最
佳化同儕網路整體效能，並提升系統的可靠度與穩定度。 
在計畫執行過程，計畫主持人研究資訊服務管理技術  (Information Technology Service 
Management，ITSM)，並且將研究成果撰寫一篇「A System-Oriented Management Framework for Telecom 
Operation Support Systems」研究論文，投稿於 2010 年的 ICSSSM 國際學術會議，獲得該會議接受發表。
因此，計畫主持人於 2010 年 6 月底前往日本東京，參加這場探討「服務系統與服務管理」領域相關研
究的學術盛會。

二、 與會心得 
首先，感謝國科會補助出國參加國際研討會，使申請人有機會參與這次
IWCMC 2010國際學術會議。筆者過去一段時間執行國科會「高韌性同儕網路及
其在即時多媒體上之應用」，以服務導向技術為基礎，研究大型異質同儕網路監
控與效能評估技術，最佳化同儕網路整體效能，並提升系統的可靠度與穩定度。 
近年來，無線網路通訊技術蓬勃發展，各種前端智慧型裝置亦趨於成熟，筆
者將過往在電腦通訊網路上之研發成果，進一步地往行動通訊網路應用之領域發
展，並將研究成果投稿於此次國際學術會議。 
本次會議聚集了世界各地之學者，共同討論無線與行動通訊網路發展的技術，
其中更有十多位來自台灣學者，由此可見台灣在該領域上之投入。在會議上，可
看到許多學者提出各種無線以及行動通訊網路之應用，並且探討各種通訊模型可
能遭遇的瓶頸，以及相關解決方案等。透過學術會議過程與其他學者進一步地交
流，激發彼此之創意，獲得許多新穎的觀念，對於日後尋求新研究議題勢必有相
當大的幫助。

具備無線操作功能的數位內容應用系統
DIGITAL CONTENT APPLICATION SYSTEM WITH WIRELESS OPERATION INTERFACE
國立臺北科技大學 陳英一
本發明揭示一種整合無線感測技術、影像辨識技術與內容應用機制來達成客制
化資訊服務的方法、裝置與應用系統。藉由使用者身上所配帶之無線感測標籤
，系統可以感測使用者身份，進而針對不同的使用者提供對應的服務內容，達
到客制化的資訊服務。在使用者操作系統時，系統能透過影像感知元件判斷使
用者動作，讓使用者能以非接觸的方式操作系統，而不需透過鍵盤、滑鼠或觸
控面板等傳統裝置。除此之外，本發明包含內容應用系統，可讓多個前端操作
裝置同時連線取得所需的數位內容，亦可讓數位內容編輯者連線，以管理系統
內之數位內容。
The present invention discloses the mechanisms employed in achieving
the goal of information customization by integrating wireless
sensing, image recognition, and content management technologies.  It
contains methods, devices and an application system.  By recognizing
identification tags carried by users, the system can provide
customized digital content by user type.  The invention makes use of
several specific components in accomplishing its task.  An image-
sensing component detects user actions and transforms them into
computer events, while the system is in use.  Thus the user can
control the system without making contact with devices such as a
keyboard, mouse or touch panel.  The concept also includes a content
application system.  This system primarily provides digital
information to many operations devices simultaneously.  As a
secondary function, the content application system can be employed in
資訊服務業
本發明整合無線感測技術、影像辨識技術以提供一種非接觸式數位內容存取方式。系
統可藉由使用者身上所配戴之無線感測標籤，感測使用者之動作，再藉由影像辨識的
分析，將使用者動作轉換成數位訊號，進一步改變所呈現之數位內容，提供一種新的
使用者體驗方式。
目前智慧型手機的應用從鍵盤走向觸控、在電視遊樂器的領域也有Wii提供全新的使
用者體驗。由此可知，近年來各種新穎的使用者體驗模式成為創新應用的指標，讓人
們以更直覺的方式操作系統是當下的潮流。本發明提出一種非接觸方式讓使用者與系
統進行互動，提供一種全新的使用經驗，由上述內容評估本發明可進一步進行技術轉
移。
