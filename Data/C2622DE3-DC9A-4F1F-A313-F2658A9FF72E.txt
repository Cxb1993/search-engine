研究摘要(500 字以內)： 
 
本研究主要的目的，在於發展一軟體與醫療及運動用途之脈搏測量手錶相配合，接收
來自此裝置之資料，讓使用者能夠於個人電腦上規劃與管理其健身計畫。此系統藉由 USB
裝置(USB IrDa Dongle)以紅外線的方式對目標裝置傳送/接收資料。對於此運動手錶而言，
使用紅外線傳輸能夠使裝置不必安插額外的線路，更能讓手錶有一定程度的防水能力，讓
這樣的裝置可以應用在醫療等其他領域。本計劃預計完成項目如下：1.在 PC 端開發一有
能力收發 IrDA 訊號之應用軟體。2.接收資料後在 PC 端分析並產生更有意義之資料供使用
者參考，並讓使用者規劃其健身計畫。3.程式撰寫和測試。4.系統的整合與製作。5.系統
驗證與評估。

系統軟硬體部分研究人員將學習到： 
趨。在主機(host)端，最新推出的PC幾乎100%支持 USB；而在外設(device)端，使用USB
連接埠的設備也與日俱增，例如數位相機、掃描器、CCD攝影機、印表機、鍵盤、滑鼠等等。 
 
USB設備之所以會被大量應用，主要具有以下優點： 
1、可以即插即用，讓用戶在使用外接設備時直接在PC上將USB裝置插上使用。 
2、攜帶方便。USB設備大多以輕薄見長，在想要隨身攜帶裝置數據時，支援USB的
裝置會是首要之選。 
3、標準統一。有了USB之後，各種外接設備統統可以用同樣的標準與PC連接。 
4、可以連接多個設備。USB在PC上往往具有多個連接埠，可以同時連接幾個設備，
最高可連接至127個設備。 
在 PC 端，一切裝置都由 windows 負責管理，紅外線裝置亦不例外。IrDA 即架構在
WinSock 之上。WinSock 規格定義了一套供應用程式設計者與各家網路軟體廠商共同遵守的
應用程式界面(API，Application Programming Interface)，即 WinSock API，它定義在
一個檔名為 WINSOCK.DLL 的 Windows 動態連結程式庫的頂端，WinSock 的應用程式可呼叫
WINSOCK.DLL 內的函式與網路溝通，各家網路軟體廠商則根據 WinSock API 的定義撰寫
WINSOCK.DLL，此檔可利用已安裝在系統的協定存取網路。 
 
在 WinSock API 的定義中，有 30 個插槽函式引自 BSD，其中的 23 個是與網路存取有
關的函式，這些函式包括插槽的操作、數值轉換等，其中，插槽的操作有分為區塊式
(blocking)與非區塊式(non-blocking)，區塊式操作的函式將不限定長的時間直到工作完
成後才回到其呼叫者，例如 rev()，一旦呼叫它之後，呼叫者將一直等待直到由彼端收到
資料為止，非區塊式則不如是，呼叫者在呼叫之後可立刻進行下一道指令，不須等待函式
工作完畢，在 BSD 模型中，除非程式設計者特別指定，否則插槽內定的操作模式為區塊式，
但在 WinSock API 則強調使用非區塊式，因為在 Windows 的非強制性(non-preemptive)環
境下，非區塊式操作為優，WinSock 規格建議：除非確有必要，儘可能不要使用區塊式操
作。 
 
另外的 7個是資料庫函式，它們提供網路的各項資訊，包括取得系統的主機名稱、協
定名稱與編號、某網路位址的主機名稱、某主機的網路位址等等，實際的資料庫規格可由
提供 WINSOCK.DLL 的廠商自行定義。 
 
在 Windows 特有的延伸方面，WinSock 定義了 16 個函式，這些延伸函式皆是 Windows
訊息導向(message driven)的，並且以異同步方式處理網路事件，儘管這組延伸函式並非
BSD 插槽模型早已有的，但為了配合 Windows 的程式環境，WinSock 規格仍建議使用它們。
