(GUI) 設計出專屬的圖形化顯示功能，透過圖形介面讓使用者能更直覺的操作，容易上手。本
研究針對發電調度模擬系統建構圖形使用者介面，軟體係利用 MATLAB 為設計基礎，並搭配 
SIMULINK 功能及使用者介面發展環境 (Graphical User Interface Development Environment, 
GUIDE)。使用者可由系統模擬觀察其調度效能，評估頻率調節的肇因及量化其成本，並可由即

調度人員了解系統性能並配置合理之備轉容量，以提升良好之供電品質。 
 
二、研究背景 
A. 系統動態 
    電網中因為時變負載的擾動以及電路的切換效應，所以電力系統的運轉環境會不停地變
化。當擾動來源為發電機機組跳脫或是劇變負載所產生的電力不平衡時，瞬間的電力不平衡，
首先會由發電機組轉動慣量的動能所補償，其結果將導致系統頻率受到改變，進而影響到頻率
敏感性負載。如果系統頻率偏移量遠離調速機的死區 (deadband)，則調速機將立刻反應來增加
或是減少發電機組的電能輸出。此調速機的控制動作稱之為 (系統的) 一次控制。在一般的干
擾下，當頻率偏離額定值時，一次控制機制將會動作來阻止頻率的繼續變化。而後續的控制則
是要將頻率回歸至額定值。其二次控制機制可以是人工或是自動操作模式。一個自動的二次控
制模式也被稱為自動發電控制(AGC)[1-2]. 
 
B. 頻率響應 
  假若有一電力不平衡的量為∆PG存在於電力系統之中。在二次控制對於頻率偏差產生動作
之前，分散於電力系統中的頻率響應特性試圖將脫序的頻率穩定下來。系統穩定脫序的頻率範
圍，端看於頻率特性值的大小，也就是說對於不平衡時的系統響應，越高的頻率響應特性值，
將導致較小的頻率偏移量。 
  當調速機組的一次控制試圖去維持負載發電量平衡時，系統的頻率響應量(-10βΔF)是受到
頻率偏移量、發電機組調速機特性還有頻率敏感性負載的阻尼所影響。頻率響應特性值，-10β，
可由調速機的速率下垂特性，Req，和負載阻尼係數，Deq，來表示之。在控制區域ｉ，其頻率響
應特性值表示如下，

五、應用頻率響應特性值於偶發事故備載規劃 
獨立電力系統中發生偶發事故或擴大偶發事故時會使系統頻率遽降，調度人員必須規劃足
夠備轉容量因應無法預期偶發事故，備轉容量規劃多寡必須適當，規劃過多則不符合經濟效益；
規劃過少可能造成頻率繼續下降。在台灣現行的系統頻率在 59.70 Hz 以下時為偶發事故。 
在偶發事故備轉容量的瞬間備轉容量(Instantaneous Reserve,IR)又可分為兩大部份：一為快速瞬
間備轉容量(Fast Instantaneous Reserve, FIR)，最主要目的在使事故後最低頻率快速回復至可接
受的頻率安全範圍；另一為持續瞬間備轉容量(Sustained Instantaneous Reserve, SIR)，其主要目
的在接續快速瞬間備轉容量使頻率回復至正常運轉範圍。利用 RLS 演算法即時估測頻率響應特
性值，並應用 β規劃其備轉容量及負載卸載量[8]，其規劃備轉容量分為： 
(1) 快速瞬間備轉容量之規劃

六、GUI 設計環境軟體 
圖形化使用者介面 (Graphical User Interface，GUI) ，是圖形化的溝通程式介面，由介面可
以方便達到特定的控制功能，而介面是由視窗、按鈕、鍵盤動作等物件所構成的，並藉由介面
物件呼叫MATLAB與SIMULINK來進行運算處理動作。另外，MATLAB具備有高度的數學運算
能力與各種領域的工具箱，與GUI結合程式將會非常實用。在GUI 設計的過程中，先利用GUIDE
將物件佈置完成，並設定物件屬性、操作的方法、及物件與物件間的關係，最後才撰寫各物件
相對應的程式碼 [9-11]。 
    GUI 完成設計後，若要在其他 GUI 使用必須把參數儲存 MATLAB 工作空間中，透過工作
空間做變數傳遞，在模擬 SIMULINK 中時 GUI 欲要做即時傳遞資料，本研究利用 SIMULIN 中
MATLAB Function 把資料即時放到函式下即可做傳遞；如圖 2 所示 SIMULINK 模擬後因在 GUI
中為一函式，資料只是在函式中做運算並不會存到 MATLAB 工作空間裡，藉由 SIMULINK 中
To File 資料存成檔案，再把檔案呼叫出其變數。
