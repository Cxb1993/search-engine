 系統架構: 
 
本計畫中，我們提出基於 facebook遊戲開發平台，主要我們可分為客戶端(Client)
與伺服器(Server)兩大部份，而在伺服器裡又主要分為我們平台的部份與 Facebook
平台兩大部份。主要的設計架構如上圖所示。 
各元件說明如下： 
 客戶端(Client) 
以 Flash作為遊戲客戶端，在這裡為了簡單化客戶端的功能，主要的功能都被搬
移到伺服器，而遊戲客戶端主要的功能為使用者介面與畫面呈現，並與伺服器進
行連線。 
 遊戲伺服器(Game Server) 
作為遊戲即時資訊存放，管理玩家的資料，遊戲狀態，轉送玩家間的資訊，廣播
全域資訊(global message)，為整個遊戲平台的核心。而本計劃的核心架構也是
以它為主，架構圖如下圖所示。 
 
Game Server 架構圖 
 網頁伺服器(Web Server) 
提供高效能的 HTTP Service服務，架構方式為 Apache 加上 PHP 的 module開發，
為了效能考慮使用了 Fast-CGI的架構來加速 PHP在 apache上的執行效能。主要
功能為提供使用者下載遊戲程式、會員資料取得，與 Facebook 平台通訊，遊戲
資訊的管理。 
 資料庫伺服器(Database Server) 
主要功能為儲存遊戲分數、儲存會員資料、儲存遊戲資料。當玩家們大量的分數
更新的需求產生時，一律先送達 Data Base Server 來作安全性檢測並作記錄，
遊戲相關 API: 
遊戲伺服器依性質不同主要成下面幾個類別： 
 房間管理類別：主要負責房間的生成，移除，查詢的功能。並透過負載偵測的機
制動態的進行房間的配置。 
 遊戲底層指令類別：登入房間，離開房間，廣播，聊天，玩家列表維護等遊戲底
層的指令。 
 遊戲指令類別：遊戲各別設定的遊戲指令，依不同遊戲有不同的指令產生，這也
是遊戲擴充的主要部份。 
 Facebook 通訊類別：提供與 facebook 平台的通訊，如發佈分數到塗鴉牆，邀請
朋友玩遊戲等指令。 
 
技術特點說明： 
本計畫中設計並實作了基於 facebook遊戲開發平台，其內包含了： 
 
 提供了開發平台架構。 
 伺服器端的設計與實作。 
 提供了與 facebook 的通訊方式設計。 
 提供俱有高彈性、擴充性、高效能遊戲平台。 
 透過新增遊戲模組來縮短開發時程。 
 
技術研發 
基於上述平台的特點，本技術可以有降低遊戲營運廠商的開發成本，與縮短開發
時程，讓廠商能準時提出新的遊戲產品，增加競爭力。 
 
可利用之產業及可開發之產品： 
本計畫可應用社交網路遊戲的開發，廠商能依平台提供的功能很容易的與
Facebook平台作通訊，並專注於遊戲模組的新增，不必關心太多遊戲底層與平台的問
題，大幅減少開發與測試的成本。若廠商不需要社交平台的通訊功能，本台平也很適
合開發網頁遊戲(Web Game)類的線上遊戲開發。
