社群及行動安全技術之研究與應用－子計畫三:整合社群網路與行動裝
置之安全雲端服務 
 
     主持人 王勝德 
摘要 
 由於網際網路之高便利性及行動裝置的普及化，行動裝置與雲端計算之結合已被廣泛地應用，而在這樣的情境
下，使用者不需自備特定硬體資源，亦不需安裝軟體，只要所使用的裝置能連上網際網路並使用瀏覽器的介面，
即可進行雲端服務的操作。然而雲端計算是架構在網際網路的技術之上，對於使用者來說，透過網路來存取雲端
服務時安全及隱私保護就更是顯得重要。例如行動裝置將使用者的社群網路帳號密碼透過無線網路傳輸出去，即
使所傳輸的是加密過的網路封包，使用者的帳號密碼仍然可被他人攔截並解析，如此一來雲端服務的使用者便可
能遭受危害，使得使用者的隱私資料遭到竊取。因此本文嘗試使用一組使用者自訂的信賴碼，並藉由零知識證明
的方式，進行安全驗證使用者的程序，以驗明想要存取雲端服務的人為真正具有帳號的本人，這樣一來除了可以
為安全雲端服務設立一道基本的防護界線之外，也可以有效抵禦中間人攻擊。 
關鍵詞 
雲端計算、社群網路、行動裝置、零知識證明、隱私保護、安全、信賴、中間人攻擊

 圖一：NTUSquare 平台示意圖 
 
「NTUSquare」的 LBS 概念圖如圖一： 
整個系統由 Location Based Service Platform、Subscribers 與 Service Providers 所組
成。Subscribers 代表生活在台大校園廣大的教、職員和學生族群，Service Providers
代表校園內各式店家，或是想提供各式服務的組織單位(例如:學校單位、社團組
織或系學會的宣傳活動)，Location Based Service Platform 則是 NTUSquare 的主
題，它是一個雲端應用程式主體，主要負責存取和搜尋雲端 Google App Engine
資料庫，並將搜尋或計算結果回傳給 Subscribers。雖然目前 NTUSquare 上資料庫
的資訊，是以台大校園內的各式餐廳等美食資訊為主，但其實資料庫的內容不限
定只存放美食資訊，因為隨著資料庫資訊的擴充與擴展，NTUSquare 平台的生活
資訊也將趨於更多元和完善。 
 
[系統設計] 
NTUSquare 的分層架構如圖二所示，是以 Subscribers、LBS Platform 與 Cloud 
Services 三個構面所組成。其中箭頭的方向表示系統的資料與服務流。 
外，為了讓 Web 應用程式的開發者方便進行開發，Google App Engine 也針對
Python 的開發者提供 webapp 架構，讓開發者可以專心在 Web 應用程式的功能
設計上，而由 webapp 架構負責處理 CGI 標準的運作細節。依循 webapp 架構所
設計的 Web 應用程式主要有三個部份： 
1.一或多個 RequestHandler 類別，可處理要求和建置反應。 
2.一個 WSGIApplication 實例，會根據 URL 將連入要求傳送至處理常式。 
3.一個主要常式，使用 CGI 配接程式來執行 WSGIApplication。 
 
另外，Google App Engine 根據 Google 基礎架構，提供幾種好用的服務，其
中的一項服務為「使用者」服務，它可以讓我們的 Web 應用程式與 Google 使用
者帳戶整合在一起。從我們的服務平台架構來看，雲上的 Web 應用程式是放在
Google App Engine，端點的 Android 應用程式是在 Google 所提供的 Android 平台
上執行的，因此我們便考慮藉由使用者的 Google 使用者帳戶，來做為登入我們
服務時所使用的使用者帳戶，這樣一來我們便可以不用要求所有的使用者在使用
進階服務前，都要先針對我們的服務做特定帳戶的註冊和開通動作，而且這樣的
做法相信對於使用 Android 平台的大部份使用者來說，會是一種使用者親和性
(user-friendly)的行為。除了上述 Google App Engine 對開發 Web 應用程式提供的
好處之外，Google App Engine 還提供 Datastore 讓開發者來建立資料儲存空間(例
如：Database)，而 Google App Engine 也針對其基楚架構提供好用的 API(例如：
Data Modeling API)，藉此來幫助 Web 應用程式處理資料的散佈、複製以及負載
平衡，並且也讓我們的 Web 應用程式可以獲得強大的查詢引擎與交易功能。

[結論與建議] 
綜觀業界現有的 LBS 服務有以下發展的趨勢，包括有： 
 「定位追蹤」結合相片定位功能：近年來大多數的電信業者與 LBS 行動
定位裝置製造商，已先後推出相片定位服務，使服務趨勢從原有位址追
蹤，逐漸演進至結合相片定位的功能，藉由地圖、相片與文字的方式紀
錄生活旅遊經驗，滿足用戶重視社群分享的訴求。 
 「生活資訊搜尋」範圍日趨擴大：目前 LBS 業者紛紛投入多功能搜尋服
務的開發，不再侷限於特定搜尋功能，加強擴充各式各樣生活資訊的搜
尋服務，甚至提供「關鍵字查詢」的服務，讓用戶以更方便的使用方式，
獲得更全面性的所在地週遭生活訊息。 
 「導航與交通訊息」可靠性與完整性：LBS 業者除了持續精進導航技術
以外，更致力於提昇「即時交通訊息」的準確性，此外，更結合行人「大
眾運輸路徑規劃」的服務，讓用戶不論是在開車或是徒步行走時，均能
掌握可靠的位址與交通訊息，快速抵達目的地。 
 「特定使用族群需求功能」向多角化發展：針對特定客群需求，研發具
有「社群遊戲」、「環保概念」，以及「健康訴求」功能的服務，讓 LBS
相關應用服務更為多樣化，除了提供一般性的 LBS 服務以外，各種特定
功能訴求的服務，將滿足不同市場區隔客群的需求，促使 LBS 應用服務
功能逐漸朝向多種利基市場方向發展。
